# Epic Plan 1.8: Output Directory Override (`--output-dir`)

**Estimated Durations**: 1 day

## Objective

Add a `--output-dir` flag to `atcr review` and `atcr reconcile` so the entire review directory tree is written to an arbitrary path instead of `.atcr/reviews/<id>/`. This enables external orchestrators (skills, CI pipelines, wrapper scripts) to direct atcr's output to a location they control without post-hoc file moves.

## Context

Epic 1.0 establishes the review directory at `.atcr/reviews/<id>/` with a `.atcr/latest` pointer â€” the right default for interactive CLI use. But external consumers need to place output in a specific location: the existing `execute-code-review` skill writes to `${SPRINT_PATH}code-review/multi-agent/`, and CI pipelines may want output under an artifacts directory. Without `--output-dir`, these consumers must move files after the run, which is fragile (partial writes, lost crash-state) and defeats the purpose of the deterministic directory layout.

## Problem Statement

1. `atcr review` always creates the review directory under `.atcr/reviews/<id>/`. There is no way to redirect output to an external path.
2. External orchestrators that need output at a specific location must either (a) move files post-run (losing crash-safety guarantees) or (b) symlink `.atcr/reviews/` (fragile, platform-dependent).
3. The `.atcr/latest` pointer is updated on every run, which is wrong when an external consumer directs output elsewhere â€” the "latest" pointer should track interactive runs only.

## Proposed Solution

1. **New flag `--output-dir <path>`** on `atcr review`:
   - When specified, the review directory tree (manifest.json, payload/, sources/, reconciled/) is created at `<path>` instead of `.atcr/reviews/<id>/`.
   - The path must not already exist (or must be empty) â€” fail with exit 2 if it contains files, to prevent accidental overwrites.
   - The `.atcr/latest` pointer is NOT updated when `--output-dir` is used.
   - The `<id>` is still generated (for manifest.json) but is not used for path construction.

2. **`atcr reconcile` inherits the output directory** from the review directory it operates on. No separate flag needed â€” reconcile discovers its paths from the review directory's manifest.

3. **`atcr report` reads from the review directory** â€” no change needed. It already takes an id-or-path argument.

4. **Validation**: `--output-dir` and `--id` are mutually exclusive (the id is meaningless when the path is explicit). `--output-dir` with a relative path resolves against CWD.

## Success Criteria

- [ ] `atcr review --output-dir /tmp/test-review` creates the full review tree at `/tmp/test-review/` (manifest.json, payload/, sources/).
- [ ] `.atcr/latest` is NOT updated when `--output-dir` is used.
- [ ] `--output-dir` with a non-empty existing directory fails with exit 2 and an actionable message.
- [ ] `--output-dir` and `--id` together fail with exit 2 (mutually exclusive).
- [ ] `atcr reconcile` operates correctly on a review directory created with `--output-dir`.
- [ ] Existing behavior (no `--output-dir`) is unchanged â€” `.atcr/reviews/<id>/` and `.atcr/latest` work as before.
- [ ] Unit tests cover the flag parsing, path validation, and the `.atcr/latest` skip logic.

## Task Breakdown

1. CLI: add `--output-dir` flag to `atcr review` command, wire into the review directory creation logic.
2. Review directory: when `--output-dir` is set, create the tree at that path; skip `.atcr/latest` update.
3. Validation: mutual exclusion with `--id`, non-empty directory check, relative path resolution.
4. Tests: flag parsing, path creation, latest-pointer skip, validation errors.
5. Docs: CLI help text, README usage example for external orchestrators.

## Out of Scope

- Changing the review directory layout (the tree structure is unchanged).
- Output filename aliases or format compatibility â€” external consumers read atcr's actual output format.
- `atcr reconcile` or `atcr report` getting their own `--output-dir` flags (reconcile inherits; report reads in place).

## Dependencies

- Epic 1.0 review directory management (task 7) â€” complete, in code review.

## Clarifications

### Recorded 2026-06-12 (rubber duck â€” /execute-epic)

**Key Decisions:**
- `--output-dir` is added to `atcr review` ONLY. `atcr reconcile` and `atcr report` already accept an explicit filesystem path via their `[id-or-path]` positional argument, so an orchestrator passes the same path there â€” no new flag on reconcile/report.
- The generated `<id>` is NOT persisted into `manifest.json`. It already surfaces in the `atcr review` stdout line (`cmd/atcr/review.go:100`) and in `fanout.ReviewResult.ID`. No consumer reads an id from `manifest.json`; reconcile/report resolve their working directory purely by path. The `Manifest` struct is unchanged.

**Scope Boundaries:**
- IN scope: `--output-dir` flag on `atcr review`; path validation (non-empty existing dir â†’ exit 2; relative path resolved against CWD; mutual exclusion with `--id`); skip `.atcr/latest` update when `--output-dir` is set; unit tests; CLI help + README docs.
- OUT of scope: a `--output-dir` flag on `atcr reconcile`/`atcr report`; changing the review tree layout; adding an `ID` field to `manifest.json`; output filename aliases.

**Technical Approach:**
- Add `OutputDir string` to `fanout.ReviewRequest`; thread from `runReview`.
- In `PrepareReview`, when `OutputDir != ""`: scaffold the trio (`payload/`, `sources/`, `reconciled/`) at the path via a new helper that creates parents (MkdirAll) and rejects a non-empty existing directory (`os.ReadDir`, any entry â†’ error â†’ exit 2), then SKIP `WriteLatest`.
- Mutual exclusion (`--output-dir` + `--id`) enforced in `runReview` via `cmd.Flags().Changed()` and returned through `usageError()` so it maps to exit 2.
- Relative `--output-dir` resolved to absolute via `filepath.Abs` at flag-parse time.
- One-shot (`--fail-on`) composes for free: it reconciles `result.Dir` in-process, which is the output-dir path.
