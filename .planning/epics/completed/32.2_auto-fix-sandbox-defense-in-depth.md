## Tasks

### Task 1 — Require explicit `noSandbox` on the host dispatch (`cmd/atcr/autofix.go`)

**Source TD:** `internal/verify/autofix_exec.go:28` (LOW, maintainability) —
*"ResolveAutoFixSandbox returns (nil, nil) when enabled==false, and runAutoFix
treats a nil sandboxBackend as run-on-host. The fail-closed guarantee is coupled
to validateAutoFixBackend hard-coding enabled=true; a future caller passing
enabled=false would get a silent host fallback with no warning and no error."*

Add an explicit `noSandbox bool` to `autoFixBackend`, set it **only** on the
warned `--no-sandbox` opt-out path in `validateAutoFixBackend`. Change the
dispatch in `runAutoFix` (`cmd/atcr/autofix.go:377`) so host execution requires
`be.noSandbox`; a nil backend without the opt-out flag **refuses** (fail closed)
instead of silently running validation on the host.

**Why the dispatch, not the gate (clarification, 88% confidence):** the gate
already hard-codes `enabled=true` at its single call site and is the only place a
nil legitimately arises today (the warned opt-out), so a gate-only check adds
nothing. The real risk is a future/test caller constructing `autoFixBackend`
directly and bypassing the gate — only a dispatch-side check catches that.

**Blast radius:** inverts the AC 01-03-pinned nil→host zero-behavior baseline
(`TestRunAutoFix_NilSandboxUsesHostPath`) and requires updating ~6 host-path test
constructions currently relying on implicit-nil zero values. Revisit AC 01-03 as
a conscious decision when this lands.

### Task 2 — Fold `context.Canceled` into `TimedOut` (`internal/sandbox/docker.go`)

**Source TD:** `internal/verify/sandboxvalidate.go:37` (LOW, performance) —
the sandbox path relies on the backend's own timeout rather than layering a
ctx-deadline backstop like the host path.

**Clarification (92% confidence):** no adapter-level change — the adapter has no
independent signal to arbitrate a ctx error against an opaque `Backend.Run`
result. The disambiguation belongs **inside** each `sandbox.Backend`. In
`internal/sandbox/docker.go` (which currently checks only
`context.DeadlineExceeded` at ~docker.go:232), also fold `context.Canceled` into
the `TimedOut` classification, mirroring `internal/verify/localvalidate.go:127-128`'s
host-path belt-and-suspenders handling. The naive adapter backstop was explicitly
rejected as unsafe at the Sprint 32.0 Phase 2 gate (TD-004).
