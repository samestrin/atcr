## Current State

Two divergent `writeRadarSection` implementations render the disagreements radar:

1. `internal/reconcile/disagree.go:413` — `writeRadarSection`, used by the reconciled report.md path
2. `internal/report/disagree.go:49` — `writeRadarSection` (delegates to `writeRadarItems` at :58), used by the archival/display path

Both render the same `DisagreementsFile` structure but with different escaping helpers (`esc`/`codeSpan` in reconcile vs. `escTrunc`/`joinReviewers`/`reviewerOrUnknown` in report). The duplication risks divergent escaping and behavioral drift.

## Pain Points

### Divergent Escaping
- reconcile uses `esc()` + `codeSpan()` from emit.go
- report uses `escTrunc()` + `joinOrNone()`/`joinReviewers()`
- Same data structure, two rendering pipelines — a bug fix in one won't reach the other

### Import Constraint
- `internal/report` already imports `internal/reconcile`
- `internal/reconcile` → `internal/report` would be circular
- The shared renderer must live in `reconcile` (or a new shared package), not in `report`

### Intentional Differences
- archival `esc` vs display `escTrunc` (verbatim vs truncated)
- `joinOrNone` vs `joinReviewers`/`reviewerOrUnknown` (different reviewer formatting)
- These must be captured as explicit parameters, not silently erased

## Proposed Solution

Extract a shared `WriteRadarSection` function in `internal/reconcile` parameterized by:
- A truncate/verbatim text renderer (function parameter)
- A heading prefix format (function parameter)
- Reviewer join strategy (function parameter)

`internal/report` calls the shared renderer with its display-oriented parameters. `internal/reconcile` continues to use it with its archival parameters.

## Scope

### In Scope
- Shared renderer in `internal/reconcile` with parameterized escaping/formatting
- `internal/report/disagree.go` calls the shared renderer
- Preserve intentional differences via parameters
- Update tests for both call sites

### Out of Scope
- Changes to the DisagreementsFile data model
- Changes to call sites in `cmd/atcr/report.go` or `internal/mcp/handlers.go` (separate TD rows)

## Clarifications

### Recorded 2026-06-22 (rubber duck — /execute-epic)

**Key Decisions:**
- The shared renderer takes exactly TWO parameters: a heading prefix (string) and a free-text body renderer (func(string) string). The "reviewer join strategy" parameter from the Proposed Solution is dropped.
- Reason: `reconcile.joinOrNone` and `report.joinReviewers` are byte-identical (both return `(none)` on empty input, both join with `, `), and `report.reviewerOrUnknown` matches reconcile's inline `(unknown)` fallback exactly. Parameterizing a non-difference would be speculative surface, contrary to the epic's own Risk-table guidance ("start with 2-3 function parameters; extract more only if needed").

**Scope Boundaries:**
- IN scope: exported shared renderer in `internal/reconcile` (section header + per-item loop), `internal/report` call sites (`writeRadarSection` + `RenderDisagreements`), removal of report's now-dead helpers (`formatScore`, `reviewerOrUnknown`; `joinReviewers` is RETAINED — still used by the main findings render at `render.go:155`), tests for both packages.
- OUT of scope: `cmd/atcr/report.go`, `internal/mcp/handlers.go`, the `DisagreementsFile`/`DisagreementItem` data model.

**Technical Approach:**
- Fixed-vocabulary fields (Kind, Severity, File, Disagreement, Skeptics, Reviewers, position reviewer/severity) route through reconcile's `esc`/`codeSpan`/`joinOrNone` unconditionally; free-text body fields (Problem, Detail, position Problem) route through the injected text renderer.
- reconcile passes `esc` (verbatim/archival); report passes `escTrunc` (500-rune cap). This is byte-output-preserving — existing golden tests are the parity oracle.

**Original Questions and Answers:**
1. Q: AC4 lists "reviewer formatting" as an intentional difference to preserve via an explicit parameter, but `joinOrNone`/`joinReviewers`/`reviewerOrUnknown` are byte-identical across packages. Keep a reviewer-join parameter (B) or drop it (A)?
   A: Drop it (Option A). Use exactly two parameters: heading prefix and text renderer. The only genuine behavioral difference is the free-text escaper (`esc` verbatim vs `escTrunc` 500-rune cap).
