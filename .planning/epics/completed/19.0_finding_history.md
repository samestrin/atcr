# Epic Plan 19.0: Finding History

- **Estimated time**: 2-3 days
- **Tasks/Components**: 4/2
- **Execution**: execute-epic --strong

## Objective

Persist findings from every `review` run and let a user query a package's finding history over time.

## Context

Teams (and solo users) want to see whether a package's findings are trending up or down across runs. `cmd/atcr/review.go` already produces findings; nothing durably records them per run. This adds a keepable history store and query command.

## Problem Statement

There is no way to answer "what findings has `internal/registry` accrued over the last 30 days?" without re-running review against old commits.

## Task Breakdown

1. Append one JSON record per finding to `findings-history.jsonl` at the end of a `review` run (hook the existing output path in `cmd/atcr/review.go`), with run timestamp, package, severity, and finding id.
2. Add `internal/history` reader that loads and filters records by `--since` (duration) and `--package` (path prefix).
3. Add `atcr history` cobra command in `cmd/atcr/history.go` exposing `--since` and `--package`.
4. Render filtered results as a markdown table (count by severity per package).

## Acceptance Criteria

- [ ] Two consecutive `review` runs append two record batches to `findings-history.jsonl`.
- [ ] `atcr history --since 30d --package internal/registry` prints a markdown table filtered to that package and window.
- [ ] Empty/absent history exits 0 with a "no history" message, not an error.
- [ ] `go test ./...` passes; new code covered by `cmd/atcr/history_test.go` + `internal/history/*_test.go`.

## Components Touched

- `cmd/atcr` (new `history.go`, hook in `review.go`)
- `internal/history` (new package)

## Out of Scope

Dashboards or charts; cross-repo aggregation; retention/rotation policy.

## Dependencies

- `cmd/atcr/review.go` output path (exists).

## Clarifications

### Recorded 2026-07-04 (rubber duck — /execute-epic)

**Key Decisions:**
- Persist the **pool-merged** findings (`<reviewDir>/sources/pool/findings.txt`, 8-column per-source format, parsed via `stream.ParseSource`) on **every** review run — hook the append after `fanout.ExecuteReview` returns in `cmd/atcr/review.go` (~L297-325), before the conditional in-process reconcile at L340. Reconcile runs only conditionally, so gating history on it would break AC1 for a bare `atcr review`.
- Finding `id` = sha256-truncated-hex of `File + "\x00" + Line + "\x00" + Problem` (Severity deliberately excluded), mirroring `internal/debate/emit.go` `itemID`. Severity is mutably re-settled by debate/verify (`internal/debate/emit.go:122-124`); keying on it would mint a new id on re-settlement and defeat trend tracking.
- History ledger lives at `.atcr/findings-history.jsonl` (repo-level, append-only, always — independent of `--output-dir`, which is a one-off redirect that leaves `.atcr/latest` untouched). `.atcr/` is the project's existing persistent workspace root.
- `--package` filter uses a **separator-aware** prefix check: `path == pkg || strings.HasPrefix(pkg, path + string(os.PathSeparator))`, mirroring `internal/reconcile/discover.go:154-156`, so `internal/registry` does not spuriously match `internal/registry2`.
- Record shape: `{ts (RFC3339 run timestamp), package (filepath.Dir of file), severity, id, file, category}`.
- `--since` accepts day/week/hour/minute units (`30d`, `2w`, `48h`, `90m`); Go's `time.ParseDuration` rejects `d`/`w`, so a small custom parser handles those.

**Scope Boundaries:**
- IN scope: JSONL append hook in `review.go`; `internal/history` reader + `--since`/`--package` filters; `atcr history` cobra command; markdown table (count by severity per package); empty/absent history → exit 0 with "no history".
- OUT of scope: dashboards/charts; cross-repo aggregation; retention/rotation policy.

**Technical Approach:**
- Read pool findings with `stream.ParseSource`; derive `package` and `id` per finding; append one JSONL line per finding under an append-only writer.
- `internal/history` exposes a record type, an `Append` writer, and a `Load`+filter reader keyed on `--since` (duration) and `--package` (path prefix).
- No new third-party dependencies; stdlib + existing `internal/stream`.

**Original Questions and Answers:**
1. Q: Findings source & which runs persist — pool merged, reconciled-when-available, or reconciled-only?
   A: Option (a) — pool merged, every run. Hook after `ExecuteReview` returns, reading `result.Dir/sources/pool/findings.txt`; reconcile runs only conditionally so gating on it would break AC1 for a bare `atcr review`.
2. Q: Finding id — hash of `severity|file:line|problem`, or a different key?
   A: Drop Severity; hash `File+Line+Problem` only (sha256-truncated-hex, `debate.itemID` pattern). Severity is mutably rewritten by debate/verify, so keying on it defeats trend tracking.
3. Q: History file location — `.atcr/findings-history.jsonl` (repo-level, always) or under `--output-dir`?
   A: Confirmed — `.atcr/findings-history.jsonl`, always, independent of `--output-dir`.
4. Q: `--package` semantics — `filepath.Dir(file)` + path-prefix match, or exact match?
   A: Confirmed, with separator-aware prefix check (`path == pkg || HasPrefix(pkg, path+sep)`) to avoid `internal/registry` matching `internal/registry2`.
