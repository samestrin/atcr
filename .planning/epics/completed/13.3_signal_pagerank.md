# Feature Request: Deterministic PageRank Signal

- **Estimated time**: 4 days
- **Tasks**: 2
- **Components**: 1
- **Execution**: execute-epic

## Context

The current reconciler calculates "Confidence" via simple vote counting. If three models flag an issue, it gets a "HIGH" confidence rating.

## Problem Statement

Flat vote counting is naive. "3 models agree" produces a flat confidence score regardless of whether those 3 models frequently hallucinate or are highly authoritative for that specific codebase or language. It treats an agreement between three weak open-source models identically to an agreement between Claude 3.5 and GPT-4o.

## Proposed Solution

- Build an agreement graph for the current review run. Nodes are models; edges represent "Model X agreed with Model Y on a finding".
- Run deterministic PageRank over this graph to calculate the relative "authority" of each model within the context of the current run.
- If models agree frequently on high-confidence issues, their isolated findings gain inherited weight.

## Technical Constraints

- Must be calculated entirely offline and deterministically per run.

## Non-Functional Requirements
- Graph processing must execute in under 5ms.

## Out of Scope
- Global PageRank across historical repository runs (requires a persistent database; this is strictly per-run for now).

## Dependencies

## Clarifications

### Recorded 2026-06-28 (rubber duck — /execute-epic)

**Key Decisions:**
- Reuse the existing `Confidence` enum `{LOW, MEDIUM, HIGH, VERIFIED}`; PageRank authority promotes isolated findings between existing tiers (MEDIUM→HIGH). No wire-schema change, no new fields, no downstream consumer breakage. `VERIFIED` stays reserved for the adversarial verify stage.
- Promote-only: a single-reviewer finding from an above-baseline-authority model is promoted MEDIUM→HIGH. Authority NEVER demotes. Rationale: demotion below HIGH silently excludes a finding from fix generation (`internal/verify/executor.go:126`) with no visible warning, and the CI severity gate does not key on confidence (`gate_matrix_test.go:126-131`), so demotion carries harm with no offsetting benefit.
- Threshold for "high authority": a model whose PageRank score is strictly greater than the uniform baseline `1/N` (N = node count). Deterministic, zero configuration.
- Agreement edges are count-weighted (edge weight = number of issues two models agreed on). Severity/confidence weighting deferred to v2 to avoid circularity (Confidence is the very signal PageRank replaces) and an unresolved severity-calibration question.

**Scope Boundaries:**
- IN scope: run-global agreement graph, deterministic PageRank (damping 0.85, fixed iteration cap), authority-aware promotion of isolated findings — all inside the reconcile package.
- OUT of scope: global/historical PageRank across runs (no persistent DB); any new wire/serialization field; severity-weighted edges.

**Technical Approach:**
- Build the agreement graph ONCE per run from all merge groups across all location clusters (authority is a run-global property of a model). Refactor `Reconcile` to a two-pass shape: collect merge groups → build graph → run PageRank → assign confidence (today confidence is assigned inline in `Merge`).
- For every merge group with ≥2 distinct reviewers, add an undirected count-weighted edge between each pair of reviewers in that group. Authority is keyed by Reviewer name; unattributed (empty-Reviewer) findings carry no authority.
- BACKWARD-COMPAT INVARIANT: when no model has differential authority (no agreement edges, or uniform PageRank), confidence assignment is byte-identical to today — existing golden fixtures and "two reviewers → HIGH" / "single reviewer → MEDIUM" tests pass unchanged.
- PageRank adjusts the reconcile-stage (v1) confidence "prior"; verify/debate still override via verdicts (`ConfidenceForVerdict`).
- Pure offline, stdlib-only (hand-rolled power iteration; no gonum / linear-algebra dependency).
