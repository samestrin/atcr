# Feature Request: Bipartite Matching & DBSCAN

- **Estimated time**: 1 week
- **Tasks**: 3
- **Components**: 1
- **Execution**: execute-epic

## Context

Deduplication currently groups findings using a greedy clustering algorithm based on an arbitrary similarity threshold.

## Problem Statement

-   **Sub-optimal grouping**: Greedy clustering algorithms often group the wrong things or leave stragglers when matching multiple reviewers' outputs.
-   **Hallucination isolation**: Currently, identifying isolated "noise" from a single model relies on arbitrary proximity thresholds.

## Proposed Solution

1.  **Resolution Phase**: Implement the Kuhn-Munkres (Hungarian) algorithm to match Model A's findings to Model B's findings optimally. The edge weights for the bipartite graph are the similarity distances (derived from 13.0 NCD and 13.1 AST Isomorphism).

> **Note (2026-06-27, from /execute-epic 13.0):** Epic 13.0 NCD was **not adopted** — its labeled-corpus benchmark falsified AC3 (NCD distributions for lexically-diverse duplicates and unrelated findings fully overlap; NCD tracks the shared code-review register, not semantics — best oracle accuracy 82%, with a false-merge error profile worse than Jaccard). See `.planning/epics/completed/13.0_semantic_ncd_deduplication.md`. **Re-source the second edge-weight signal** (e.g. token-set/Jaccard distance, or embeddings — embeddings are out of the original 13.0 scope); do not depend on NCD.
2.  **Noise Isolation**: Implement DBSCAN (Density-Based Clustering) over the resulting similarity matrix. Findings that fall outside the dense clusters are automatically and deterministically isolated into the `ambiguous.json` sidecar.

## Clarifications

### Recorded 2026-06-28 (rubber duck — /execute-epic)

**Key Decisions:**
- This is a **literal production rewrite** (not an additive spike). Bipartite matching replaces `dedupeCluster` (reconcile/dedupe.go:50-111) as the core partitioning function. AC2/AC3 benchmark tests are the TDD correctness gate proving the rewrite before merge — not an evidence gate deciding *whether* to rewrite. (Contrast 13.0 NCD, which was additive and rejected when its corpus benchmark falsified its AC.)
- **Edge-weight signal = composite, per-pair:** `distance(a,b) = 0` when findings `a` and `b` share a non-empty AST GroupKey (13.1 isomorphism); otherwise `distance = 1 − token-set Jaccard` reusing the existing `classify()` logic (reconcile/dedupe.go:128). Deterministic, stdlib-only, no new dependency. This satisfies the dependency note that the 13.1 AST metric "must be available to use as an edge weight."
- **N-model handling = incremental pairwise** Kuhn-Munkres (s₁↔s₂ → virtual merged source, merged↔s₃, …). Targets the success criterion "optimally aligns findings between N models." Ordering sensitivity is bounded for typical N ≤ 5 reviewers.
- **DBSCAN noise** is represented as **single-entry `AmbiguousCluster{Findings:[one]}`** (length 1, no Similarity). The `internal/debate` consumer already guards `if len(c.Findings) < 2 { skip }` (internal/debate/cluster.go:136), so this needs **no breaking schema change and no debate edits**.

**Scope Boundaries:**
- IN scope: rewrite `reconcile/dedupe.go` partitioning to bipartite matching; composite AST+Jaccard edge weight; incremental pairwise N-way reduction; DBSCAN noise isolation into ambiguous.json; AC2/AC3 corpus benchmark tests; **regenerating golden-corpus fixtures and rebaselining `AmbiguousHash`** (any rewrite changes output — definitively in scope); preserving source provenance through the cluster boundary (AllFindings currently flattens sources at reconcile.go:75).
- OUT of scope: prompt-based LLM arbitration; breaking the `ambiguous.json` wire shape; touching `internal/debate`; new third-party Go dependencies (reconcile/ stays zero-dep, pure stdlib).

**Technical Approach:**
- Hungarian/Kuhn-Munkres and DBSCAN implemented **in-tree, pure stdlib** (reconcile/ go.mod require block stays empty).
- Bipartite matching operates **within an existing location/AST cluster**, keeping V (findings per file per model) small per the O(V³) constraint.
- Bipartite **1:1 constraint** bounds the over-merge risk of "AST-match → distance 0": each finding matches at most one finding per other source, rather than single-linkage union-find collapsing transitively. Rectangular |A|≠|B| problems pad to square with a no-match sentinel cost above the merge threshold; assignments costing above the merge threshold are left unmatched (kept separate).
- DBSCAN `eps` derived from the existing merge threshold (`eps = 1 − MergeThreshold = 0.3`), `minPts = 2` (a finding corroborated by ≥1 other is a core point) — principled, not an arbitrary k.
- Labeled corpus from 13.0 (`.planning/epics/completed/13.0_semantic_ncd_deduplication_evidence/ncd_corpus.json`) is the AC2/AC3 benchmark fixture.
