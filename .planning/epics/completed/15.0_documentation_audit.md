# Sprint Plan 15.0: Documentation & Code Review Alignment

- **Estimated time**: 3 days
- **Tasks**: 4
- **Components**: 2

## Context

Before launching the `atcr.dev` website, we need absolute certainty that our internal documentation matches the current state of the engine. The website will use our markdown documentation as a single source of truth to populate the `docs.html` and `faq.html` pages.

## Problem Statement

As we've added features (like Reconciler v2 and Semantic NCD Deduplication), the CLI instructions, config schemas, and technical architecture documents have drifted. If we build the website based on outdated documentation, new users will be confused and unable to configure their `atcr.yaml` files correctly.

## Proposed Solution

1. Audit all existing markdown documentation in the repository.
2. Update CLI command references to match the current binary's `--help` output.
3. Update `atcr.yaml` configuration examples to include the newest persona and deduplication settings.
4. Establish the `docs/` folder as the definitive single source of truth for the website build process.

## User Personas

## Success Criteria
