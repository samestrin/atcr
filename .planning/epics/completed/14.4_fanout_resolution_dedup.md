# Tech Debt: Remove dead buildAgent and dedup fan-out mode/payload resolution

- **Estimated time**: ~1 hour
- **Tasks/Components**: 2/1

## Current State

Epic 14.3's T3 chunking refactor (`internal/fanout/review.go`) extracted `renderAgent`
and inlined the agent mode/payload-resolution logic (forceMode / `EffectivePayloadMode`,
payload-map lookup, not-found / no-payload errors) directly into the `add` closure inside
`buildSlots`. `buildAgent` (`internal/fanout/review.go:941`) still contains the identical
resolution logic, but it no longer has any production caller — only tests reference it.

`buildAgent` was NOT dead before Epic 14.3 (confirmed via
`git show d19999a2 -- internal/fanout/review.go`); the T3 change is what orphaned it in
production by inlining the resolution into the `add` closure.

## Pain Points

### Divergent duplicate logic (drift risk)
The `add` closure and `buildAgent` are now two independent copies of the same
mode/payload-resolution rules. A future change to one (a new payload mode, a changed
error message, an added resolution step) will silently NOT reach the other, so the tests
exercising `buildAgent` validate a code path production no longer runs.

### Dead production code
`buildAgent` is reachable only from test files. It reads as production API but exercises
nothing production calls, misleading readers about the real resolution path
(`buildSlots` → `renderAgent`).

## Evidence

Dead in production — the only references are test files (9 call sites), grep-confirmed:
- `internal/fanout/retry_wiring_test.go:42,47`
- `internal/fanout/review_test.go:562`
- `internal/fanout/review_scope_inject_test.go:22,29,48,62`
- `internal/fanout/review_tools_test.go:41,104`

Production references to `buildAgent` are comment-only (e.g. `review.go` doc comments).
No non-test caller exists.

## Proposed Solution

Delete `buildAgent` and retarget its tests at the real production seam so there is
exactly one resolution site.

### Option A (preferred): delete + retarget tests
1. Delete `buildAgent` (`internal/fanout/review.go`).
2. Rewrite the 9 test call sites to drive `buildSlots` (which owns the resolution in the
   `add` closure) and/or `renderAgent` directly, asserting the same persona/payload/retry
   properties they assert today.

### Option B (rejected): route the `add` closure through `buildAgent`
Rejected because `buildAgent` resolves the whole-payload (bulk) path only; the chunked
strategy renders one `renderAgent` per bin-packed chunk, so routing `add` through
`buildAgent` cannot cover the chunked per-chunk path. It would not remove the divergence.

## Scope

### In Scope
- Remove `buildAgent` from `internal/fanout/review.go`.
- Rewrite the 4 test files that call it to drive `buildSlots`/`renderAgent`.
- Confirm no unused imports/helpers remain after removal.

### Out of Scope
- Any change to production resolution behavior in `buildSlots`/`renderAgent` — this is a
  pure dedup/dead-code removal; behavior must not change.
- The chunked strategy logic itself.
