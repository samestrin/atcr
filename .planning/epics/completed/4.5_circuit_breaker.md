# Epic 4.5: Circuit Breaker and Provider Health Tracking

**Status:** In Progress (active branch: `feature/epic-4.5-circuit_breaker`)  
**Created:** 2026-06-14  
**Priority:** Medium (resilience, cost savings)

## Problem

When an LLM provider (OpenAI, Anthropic) is experiencing an outage or degraded performance, ATCR continues to send requests to that provider for every agent. A review with 10 agents using the same provider will make 10 independent attempts, each timing out or failing, wasting time and API quota.

### Concrete Failure

OpenAI is down for 30 minutes. A user runs `atcr review` with 5 OpenAI agents. Each agent tries 3 times (retries from Epic 2.2), waiting 120 seconds per timeout. Total time wasted: 5 agents × 3 retries × 120s = 30 minutes. All 15 API calls fail. The review produces no results.

## Root Cause

Each agent invocation is independent. There is no shared state tracking provider health across agents. The retry logic in `internal/llmclient/client.go` retries individual requests, but there is no global "this provider is failing, stop trying" signal.

## Solution

Implement a circuit breaker pattern per provider. Track recent API call success/failure rates. When a provider's error rate exceeds a threshold, "open the circuit" and skip that provider for a cooldown period. Agents using that provider fail fast (or fall back to their fallback chain).

### Core Design

```go
// internal/circuitbreaker/breaker.go
package circuitbreaker

import (
    "sync"
    "time"
)

type State int

const (
    StateClosed   State = iota // normal operation
    StateOpen                  // circuit tripped, fail fast
    StateHalfOpen              // testing if provider recovered
)

type Breaker struct {
    mu              sync.Mutex
    provider        string
    state           State
    failureCount    int
    successCount    int
    lastFailure     time.Time
    threshold       int           // failures before opening
    cooldown        time.Duration // how long to wait before half-open
}

func New(provider string, threshold int, cooldown time.Duration) *Breaker

// Allow returns true if the request is allowed (circuit closed or half-open).
// Returns false if circuit is open (fail fast).
func (b *Breaker) Allow() bool

// RecordSuccess records a successful API call.
func (b *Breaker) RecordSuccess()

// RecordFailure records a failed API call.
func (b *Breaker) RecordFailure()

// State returns the current circuit state.
func (b *Breaker) State() State

// Registry holds breakers per provider.
type Registry struct {
    mu       sync.RWMutex
    breakers map[string]*Breaker
}

func NewRegistry() *Registry
func (r *Registry) Get(provider string) *Breaker

// Global registry.
var DefaultRegistry = NewRegistry()
```

### Integration with llmclient

```go
// internal/llmclient/client.go
func (c *Client) Complete(ctx context.Context, inv Invocation) (string, error) {
    breaker := circuitbreaker.DefaultRegistry.Get(inv.Provider)

    // Check circuit before attempting
    if !breaker.Allow() {
        return "", &CircuitOpenError{Provider: inv.Provider}
    }

    // Make the API call
    result, err := c.send(ctx, resolveEndpoint(inv.BaseURL), key, body)

    // Record outcome. 5xx, timeouts, and connection-level transport errors trip
    // the breaker (AC10). 4xx errors — including 429 rate-limits (owned by Epic
    // 4.6's backoff) and auth errors — are NOT counted as breaker failures, and
    // caller cancellation records nothing.
    if err != nil {
        if isBreakerFailure(err) { // 5xx / timeout only
            breaker.RecordFailure()
        }
        return "", err
    }
    breaker.RecordSuccess()
    return result, nil
}
```

### Integration with fanout engine

When an agent's API call fails with `CircuitOpenError`, the engine treats it as a permanent failure (not retryable) and moves to the fallback chain immediately.

### Tuning Parameters

- **Threshold**: 3 consecutive failures → open circuit.
- **Cooldown**: 60 seconds → transition to half-open.
- **Half-open**: 1 successful call → close circuit; 1 failure → reopen.

These are hardcoded initially; can be made configurable later.

- Manual circuit breaker control (`atcr circuit-breaker reset` command).

## Risks

| Risk | Likelihood | Impact | Mitigation |
|------|------------|--------|------------|
| Circuit opens too aggressively (transient blip) | Medium | Medium | Threshold of 3 consecutive failures is conservative; can increase if needed |
| Circuit stays open too long (provider recovered) | Low | Low | 60-second cooldown is short; half-open state tests recovery |
| Circuit breaker adds latency | Low | Low | Circuit check is in-memory (<1μs) |
| Race condition in concurrent breaker access | Low | High | Mutex protects state; test with concurrent access |

## Clarifications

### Recorded 2026-06-19 (rubber duck — /execute-epic)

**Key Decisions:**
- **Provider identity = logical provider name** (`AgentConfig.Provider`, the registry `Providers` map key), NOT the normalized BaseURL and NOT a new field on `llmclient.Invocation`. BaseURL is a lossy proxy: two registry providers can share one `base_url` but differ by `api_key_env` (keying on BaseURL would let provider A's failure fail-fast provider B), and trailing-slash `/v1` variants splinter one provider into several series. `doctor/resolve.go` already de-dups on `Provider\x00Model\x00BaseURL`, treating provider-name as a distinct identity. `registry.Provider` has no Name field — the name is the map key, surfaced as the `AgentConfig.Provider` string (`ac.Provider` in fanout, `skeptic.Config.Provider` in verify).
- **Provider threaded via a new `Agent.Provider` field + request context**, not via a `Completer`-interface parameter. The engine sets `ctx = circuitbreaker.NewContext(ctx, a.Provider)` in `invokeAgent` (one chokepoint covering single-shot AND tool-loop AND verify, which runs through the engine); `llmclient.send` reads the provider from context. This keeps the change to the three Agent-construction sites (`review.go` buildAgent + buildFallbackAgent, `verify` buildSkepticAgent) with zero `Completer`/test-fake churn. An empty provider (e.g. `atcr doctor`'s direct `llmclient.Complete` calls) makes the breaker a no-op — correct, since a diagnostic must probe every endpoint regardless of circuit state.
- **Metric shape = Option A**: add a settable `gauge` primitive to `internal/metrics` (gauge type + registry map + `# TYPE … gauge` exposition branch), emit `atcr_circuit_breaker_state{provider="<name>"}` valued 0=closed / 1=open / 2=half-open. Reuses the existing single-label `Key(name,label,value)` helper and the `api_errors_total{status="429"}` enum-as-label-value precedent. Add `NameCircuitBreakerState` + `LabelProvider` constants to `internal/metrics/names.go`. The scope line's "labels: provider, state" is a non-binding sketch; AC7 ("reflects current state per provider") is satisfied by the numeric-value form.
- **Transport-error breadth = extend AC10**: a breaker failure is 5xx OR a timeout OR a connection-level transport error (connection refused/reset, EOF, DNS). ALL 4xx — including 429 and 401 — stay closed (a 4xx means the server replied; categorically distinct from a transport-layer failure where no response arrived). Caller-initiated cancellation (`context.Canceled`) records nothing (the call was aborted; it proves nothing about provider health). This honors AC10's intent (4xx never trips) while serving the epic's success criterion "outages detected within 3 API calls" — a hard-down provider (DNS NXDOMAIN, connection refused) returns fast transport errors, never a 5xx and rarely a slow timeout. The 3-attempt per-call retry budget + conservative 3-consecutive-failure threshold absorb transient blips.

**Scope Boundaries:**
- IN scope: `internal/circuitbreaker` (Breaker + Registry + provider-context helpers), `internal/metrics` gauge primitive, `llmclient` Allow/Record wrap + `CircuitOpenError` + `isBreakerFailure`, `internal/fanout` `Agent.Provider` + context bridge + fallback-on-CircuitOpen regression test, `internal/verify` provider threading, unit + integration tests.
- OUT of scope (unchanged): per-agent breakers, configurable thresholds, persistence, manual reset command.

**Technical Approach:**
- Breaker state machine with injectable clock (test cooldown transitions without real sleeps). Half-open allows a SINGLE in-flight probe; concurrent agents fail fast until it resolves (addresses the race-condition risk in the epic's risk table). 1 success → close; 1 failure → reopen + reset cooldown. Thresholds hardcoded: 3 consecutive failures, 60s cooldown.
- Breaker outcome is 3-way: genuine 200 → RecordSuccess; 5xx/timeout/transport → RecordFailure; non-tripping (4xx incl. 429/401) or `context.Canceled` → record nothing.
- The Breaker updates the gauge on every state transition (circuitbreaker imports internal/metrics; no cycle). `CircuitOpenError` lives in `llmclient` (per the epic design block); the fanout engine already descends the fallback chain on any non-OK status, so AC6 is largely satisfied by existing logic — the fanout task adds the explicit regression test + a contract comment.

**Follow-up for the epic owner (do not silently override a locked AC):**
- AC10 (line 152) and the integration comment (lines 97–100) should be reworded from "5xx and timeouts only" to "5xx, timeouts, and connection-level transport errors; all 4xx including 429/401 stay closed." A transport-failure-trips-the-breaker test is added (AC9-style) to lock the extended behavior.
