# Epic Plan 19.1: Audit Trail

- **Estimated time**: 2-3 days
- **Tasks/Components**: 3/2
- **Execution**: execute-epic --strong

## Objective

Emit an append-only audit log per run and generate a one-page compliance report for a given PR.

## Context

Compliance-minded teams need a tamper-evident record of what atcr reviewed and when. The review pipeline has the base/head SHAs and PR number in hand but doesn't persist them as an audit record.

## Task Breakdown

1. Write an append-only `audit.log.jsonl` in the review output directory with run timestamp, base/head SHA, PR number, and findings summary (counts by severity).
2. Add `internal/audit` record type + reader.
3. Add `atcr audit-report --pr <n>` cobra command in `cmd/atcr/audit_report.go` that renders a one-page markdown compliance report for that PR's runs.

## Acceptance Criteria

- [ ] Each `review` run appends exactly one audit record.
- [ ] `atcr audit-report --pr 1234` renders a report with SHAs, timestamp, and findings summary.
- [ ] An unknown `--pr` value exits non-zero with a clear message.
- [ ] `go test ./...` passes; covered by `cmd/atcr/audit_report_test.go` + `internal/audit/*_test.go`.

## Components Touched

- `cmd/atcr` (new `audit_report.go`, hook in review output)
- `internal/audit` (new package)

## Out of Scope

Cryptographic signing/notarization; external SIEM export; retention policy.

## Dependencies

- Review output directory + PR-number plumbing (exists).
