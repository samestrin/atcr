# Feature Request: Public Launch — Go-Public, Module Publication & External Install

- **Estimated time**: TBD
- **Tasks/Components**: TBD
- **Execution**: init-plan

## Objective

Flip the `atcr` repository from private to public and make `go install github.com/samestrin/atcr/cmd/atcr@latest` genuinely work for external developers — the launch milestone that unblocks the standalone-skill distribution work punted from Epic 20.0 (Story 3 / AC1). This is the go-public step in the launch cluster (`33.0` docs sweep → `33.1` launch content → **`33.2` go public + atcr.dev + launch**).

## Context

Epic 20.0 (Standalone Skill Release) delivered its dispatcher rewrite (Story 1) and the backend-contract regression test (Story 2), but **paused Story 3** (install.sh + external `go install`) on a blocker that turned out to be deeper than first framed. See `20.0` TD-002.

The binding constraint is **repository privacy**, not a single `go.mod` line:

- `github.com/samestrin/atcr` is a **private** repo. The public Go module proxy (`proxy.golang.org`) returns 404 for it and for the nested `reconcile` module, so `go install …@latest` cannot work for anyone without repo access — regardless of any code change.
- The root `go.mod` carries `replace github.com/samestrin/atcr/reconcile => ./reconcile` (the in-tree Epic 8.0 module). Go refuses to `go install <module>@<version>` any module whose published `go.mod` contains a replace directive, so the replace must go — but only once `reconcile` is publishable (i.e., the repo is public).

Making the repo public is a deliberate release/lifecycle decision, correctly sequenced after the `33.0` documentation sweep and `33.1` content are ready to become the atcr.dev source of truth.

## Problem Statement

For the public launch, external developers must be able to install and run `atcr` from a clean machine with no repo access, and the atcr.dev website must be updated from a finalized, accurate source of truth. Today none of that is possible: the repo is private, the module is not publishable (replace directive), and CI/module-fetch behavior has never been exercised against a public module.

## Proposed Solution

The reconcile-publication and `go.mod` mechanics below were **prototyped end-to-end during Epic 20.0** and are proven to work the moment the repo is public — this epic inherits a de-risked plan, not a fresh investigation.

1. **Make the repository public** (coordinated with `33.0`/`33.1` completion so the public docs/content are ready at flip time).
2. **Publish `reconcile` as a versioned module + drop the replace directive** (prototyped):
   - Tag `reconcile/vX.Y.Z`; change root `go.mod` to `require github.com/samestrin/atcr/reconcile vX.Y.Z` and remove the `replace`; commit `go.sum` hashes so external installs verify without a network sumdb lookup.
   - Preserve in-tree co-development via a **local, uncommitted** `go.work` (`go install` ignores it). Do **not** commit a root `go.work` and do **not** add the isolated wasip1 parser modules (`internal/astgroup/parsers/src/*`) to it — a committed root workspace breaks their nested-module builds. Gitignore `go.work`/`go.work.sum`.
   - Verify the self-hosted `gauntlet` CI runners can fetch the now-public module from the proxy (private-repo direct-git fetch previously failed on missing github.com auth).
3. **Deliver `install.sh` + verify real `go install …@latest`** (Epic 20.0 Story 3, resumed): thin `~40`-line `go install` wrapper (Go-toolchain preflight, non-fatal PATH warning), a `//go:build integration` test that shells out to `install.sh` and runs a **real** `go install` (per the 20.0 Phase 2 clarification), and a README Quickstart entry alongside the existing manual `go install` line.
4. **Finish the public-dependent documentation** (Epic 20.0 Stories 4/5 remainder): make the `go install …@latest` lines in `README.md` / `docs/skill-usage.md` true, and complete any accuracy/migration notes that only become correct post-public.
5. **Update atcr.dev + launch**: import the finalized `docs/` as the website source of truth and execute the launch.

## Acceptance Criteria

- [ ] AC1: The `atcr` repository is public, and `go install github.com/samestrin/atcr/cmd/atcr@latest` succeeds on a clean external machine with no repo access.
- [ ] AC2: `reconcile` is published (`reconcile/vX.Y.Z`); the root `go.mod` has no replace directive; `go.sum` records the version; CI is green on the self-hosted runners.

## Notes — De-risked Prototype (Epic 20.0)

Epic 20.0 prototyped steps 2–3 against the live (private) repo and captured the gotchas so this epic avoids them:

- **Private repo → no proxy.** `proxy.golang.org` 404s a private module; external `go install` is impossible until public. This is the true blocker (not the replace directive alone).
- **wasip1 parser modules.** A committed root `go.work` is auto-discovered by `go` commands in `internal/astgroup/parsers/src/{braceparser,goparser,pyparser}` and breaks their isolated nested-module CI. Keep `go.work` local/uncommitted; gitignore it.
- **Runner git-auth.** The self-hosted `gauntlet` runners resolve `github.com/samestrin/*` via direct `git ls-remote` with no github.com auth for module fetches; this was invisible while `reconcile` was a replace (never fetched). Confirm proxy fetch works post-public, or configure runner module-fetch auth.
