# Feature Request: Website Updates for CLI Workflows (v16.0–18.1)

- **Estimated time**: 5 days
- **Tasks/Components**: 4/1
- **Execution**: execute-epic

## Context

The `atcr` platform has introduced several critical CLI features and developer workflows across recent sprints:
1.  **16.0**: Quick Start CLI onboarding wizard (`atcr init` / `atcr quickstart`).
2.  **17.0**: Auto-merged fixes execution (`--auto-fix` patch merging and PR pipeline).
3.  **18.0**: Technical debt CLI management commands (`atcr debt list`, `atcr debt add`, `atcr debt dashboard`).
4.  **18.1**: Stable Symbol Anchoring for resilient technical debt resolution.

The public website `atcr.dev` must be updated to ensure these new capabilities, commands, and workflows are clearly documented and showcased.

## Problem Statement

The public documentation and feature pages on `atcr.dev` currently reflect older versions of the codebase. Developers visiting the landing page or documentation site lack instructions for the interactive onboarding, auto-fixing, and technical debt namespaces, leading to poor discovery and usage of these features.

## Proposed Solution

1.  **Code Review & Feature Compilation**: Review the final CLI implementation details and command-line interfaces for Epics 16.0, 17.0, 18.0, and 18.1.
2.  **Documentation Site Update**: Update the markdown documentation files and HTML pages (including `docs.html` and `faq.html`) in the `atcr.dev` repository (`/Users/samestrin/Documents/GitHub/atcr.dev`) to document:
    - How to get started instantly using `atcr init`.
    - How to leverage the automated patch execution pipeline with `--auto-fix`.
    - How to use the `atcr debt` CLI subcommands to query and manage technical debt.
    - How stable symbol anchoring eliminates line drift challenges.
3.  **Landing Page / Features Section Updates**: Update the homepage features list and visual elements to highlight local auto-fixing and the technical debt manager.

## Acceptance Criteria

- [ ] AC1: Homepage features lists on `atcr.dev` are updated with sections for onboarding wizard, auto-fixes, and technical debt management.
- [ ] AC2: `docs.html` includes comprehensive CLI usage guides for `atcr init`, `--auto-fix`, and the `atcr debt` command family.
- [ ] AC3: FAQ page (`faq.html`) includes answers regarding how `--auto-fix` validates code before applying patches, and how grouping works under the hood.

## Technical Constraints

- Website modifications must be applied directly to the static hosting templates/source within the `atcr.dev` repository (`/Users/samestrin/Documents/GitHub/atcr.dev`).
- Must not introduce external client-side JavaScript dependencies to the static pages.

## Non-Functional Requirements

- Page performance (load time, static accessibility) must remain high.
- Documentation must accurately reflect the production version of the `atcr` CLI.

## Out of Scope

- Implementing interactive dashboards inside the website (static doc updates only; actual dynamic dashboards belong to the Team Edition).
