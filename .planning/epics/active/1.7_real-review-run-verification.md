# Epic Plan 1.7: Real Review Run Verification (Story-05 manual gates)

**Estimated Durations**: 1 day (plus manual provider run)

## Objective

Close the Story-05 manual verification gates that require a real `atcr` review run with a live, user-configured provider: end-to-end orchestration loop verification (AC 05-03), adversarial-tone inspection of the host review (AC 05-04 Scenario 1), and opportunistic ambiguity-adjudication sensibility verification (AC 05-04) — in one authorized combined run.

## Context

Sprint 1.0 shipped the full review pipeline with mock-based automated verification: the sprint's recorded end-to-end gate is "pipeline works with httptest mock provider" (sprint-plan.md:83), and provider wiring is explicitly environment-specific — it lives only in the user's personal `~/.config/atcr/registry.yaml` and does not ship as defaults (plan/plan.md:171). Four Story-05 DoD items were therefore left as unchecked Manual Review gates and routed to TD as MEDIUM items. The clarification adjudication (2026-06-11) settled their disposition: the AC 05-02 format gate is satisfiable now with a recorded httptest mock-provider sandbox run (re-attempt, handled outside this epic); the remaining three gates genuinely require a live-provider run, which is not authorized anywhere in the sprint record and spends real tokens, so they are deferred here. One real run closes all of them at once. Epic 1.2 (`atcr doctor` endpoint self-test) should land first so a misconfigured provider is caught before the run burns time and tokens.

## Problem Statement

1. **Orchestration loop unverified end-to-end (AC 05-03)**: the range → review → poll → host review → reconcile → report sequence is verified only by static SKILL.md section tests; no real review run artifacts exist in the repo (skill/SKILL.md:33).
2. **Adversarial tone unverified in practice (AC 05-04)**: the adversarial personality clause (skill/SKILL.md:61-63) is statically tested, but no real `sources/host/review.md` output exists to confirm the tone lands (skill/SKILL.md:61).
3. **Adjudication sensibility unverified on real data (AC 05-04)**: adjudication mechanics (stable cluster ids, adjudication.json, idempotent re-runs) are unit-tested, but no real run with gray-zone `ambiguous.json` clusters has exercised host adjudication quality (skill/SKILL.md:96).

## Proposed Solution

One authorized, combined live-provider review run of `feature/1.0_atcr_core` against local `main` (the zero-argument shape from plan/original-requirements.md:200 — "runs end-to-end on a feature branch with zero arguments"):

1. **Pre-flight**: configure a provider roster in the personal registry; run Epic 1.2's `atcr doctor` self-test (once landed) to validate endpoints before spending tokens.
2. **Execute the full Skill orchestration loop** per skill/SKILL.md: `atcr range` → `atcr review` (background) → poll `atcr status` → host review (write `sources/host/findings.txt` + `review.md`) → `atcr reconcile` → `atcr report --format md`. Record the review directory path and outcome (AC 05-03).
3. **Tone inspection**: the reviewing agent (not the host model — no self-certification) applies AC 05-04 Scenario 1's checks to `sources/host/review.md`: no praise/compliments/positive observations; every section ties to a finding or states "no issues found in <area>". Tick the Manual Review checkbox on Sam's behalf as part of the recorded resolution (AC 05-04).
4. **Adjudication, opportunistic**: if the run yields gray-zone `ambiguous.json` clusters, perform host adjudication per skill/SKILL.md:96-113 and review merge/distinct decisions for sensibility. If the clusters array is empty, fall back to a deterministic synthetic seed: hand-authored `sources/*/findings.txt` files with same-location problem texts crafted into the Jaccard 0.4–0.7 band (thresholds at internal/reconcile/dedupe.go:16-17,147-149), run through real `atcr reconcile` (AC 05-04). Shipping unverified carries no correctness risk meanwhile — unadjudicated gray-zone clusters remain unmerged by conservative default and adjudication is explicitly optional.
