# Epic 4.7: Idempotency and Safe Retry

**Status:** Draft  
**Created:** 2026-06-14  
**Priority:** Medium (reliability, user experience)

## Problem

ATCR operations are not idempotent. Retrying a failed review creates duplicate artifacts or fails with confusing errors:

1. **Review retry.** `atcr review` fails partway through. The user retries the same command. A new review directory is created (`2026-06-14_feature-branch_150405`), but the previous incomplete directory is left on disk. The user now has two review directories, one incomplete.

2. **Reconcile retry.** `atcr reconcile` fails during reconciliation. The user retries. The reconciler re-processes all agents, overwriting `reconciled.json`, but the previous reconciled output is lost (no backup).

3. **Verify retry.** `atcr verify` fails during verification. The user retries with `--fresh` to re-verify. The previous verdicts are overwritten, but the user wanted to keep them for comparison.

4. **Output directory collision.** `atcr review --output-dir /tmp/review` is run twice. The second run overwrites the first run's artifacts without warning.

## Root Cause

ATCR operations do not check for existing state before writing. Review directories are created with timestamps, but there is no mechanism to detect and resume or overwrite existing reviews. Reconciliation and verification overwrite output files without backup.

## Solution

Make operations idempotent or safe-to-retry:

1. **Review idempotency.** If a review with the same ID already exists, prompt the user: "Review <id> already exists. Overwrite? Resume? Cancel?" (CLI mode) or return an error (MCP mode).

2. **Atomic writes.** Write output files to a temporary file, then rename. Prevents partial writes from corrupting existing files.

3. **Backup before overwrite.** Before overwriting `reconciled.json` or `verified.json`, create a backup (`reconciled.json.bak`).

4. **Output directory collision detection.** If `--output-dir` points to an existing directory with review artifacts, fail with an error: "Output directory already exists. Use --force to overwrite."

```go
// cmd/atcr/review.go
func runReview(cmd *cobra.Command, _ []string) error {
    // Check if review ID already exists
    reviewDir := resolveReviewDir(idOverride)
    if _, err := os.Stat(reviewDir); err == nil {
        // Review directory exists
        if !boolFlag(cmd, "force") {
            return usageError(fmt.Errorf("review directory %s already exists; use --force to overwrite", reviewDir))
        }
        // Backup existing directory
        backupDir := reviewDir + ".bak"
        os.Rename(reviewDir, backupDir)
        fmt.Fprintf(cmd.ErrOrStderr(), "Backed up existing review to %s\n", backupDir)
    }

    // ... rest of review logic
}
```

```go
// internal/reconcile/reconcile.go
func RunReconcile(ctx context.Context, reviewDir string, ...) (*Result, error) {
    // Backup existing reconciled.json
    reconciledPath := filepath.Join(reviewDir, "reconciled.json")
    if _, err := os.Stat(reconciledPath); err == nil {
        backupPath := reconciledPath + ".bak"
        os.Rename(reconciledPath, backupPath)
    }

    // ... reconciliation logic

    // Write output atomically
    return result, atomicfs.WriteJSON(reconciledPath, result)
}
```

## Scope

### In Scope

- Atomic file writes (`internal/atomicfs` package, extend existing).
- Review directory collision detection in `cmd/atcr/review.go`.
- `--force` flag to overwrite existing review directory.
- Backup before overwrite (`*.bak` files).
- Error message when output directory already exists.
- Unit tests for atomic writes.
- Integration test: run review twice, verify backup created.

### Out of Scope

- Resume support (`--resume <id>`) — future epic (builds on Epic 4.1 partial results).
- Interactive prompt to choose overwrite/resume/cancel (CLI complexity; `--force` is simpler).
- Garbage collection of `.bak` files (user can delete manually).
- Idempotency for `atcr init` (separate concern).

## Risks

| Risk | Likelihood | Impact | Mitigation |
|------|------------|--------|------------|
| Backup files accumulate on disk | Medium | Low | Document that users can delete `.bak` files; future epic could add garbage collection |
| Atomic write fails (disk full, permissions) | Low | High | Atomic write uses temp file + rename; if rename fails, original file is untouched |
| `--force` flag is dangerous (accidental overwrite) | Low | Medium | Require explicit `--force`; no interactive prompt (CLI simplicity) |

## Open Questions

1. **Should we support resume (`--resume <id>`)?**
   - Option A: no — too complex for now; `--force` is simpler.
   - Option B: yes — allow resuming interrupted reviews.
   - Recommendation: Option A; resume is a future epic.

2. **Should backup files be compressed?**
   - Option A: no — plain `.bak` files are simpler.
   - Option B: yes — `.bak.tar.gz` to save disk space.
   - Recommendation: Option A; review directories are small.
