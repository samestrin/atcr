# Epic Plan 39.0: Public Core Dependency Injection (Enterprise Hooks)

- **Estimated time**: TBD
- **Tasks/Components**: 4 tasks / 4 components
- **Execution**: init-plan

## Objective

Refactor the public `atcr` repository to support a plugin architecture via Go interfaces (Dependency Injection), allowing the private `atcr-enterprise` repository to seamlessly inject proprietary features without maintaining a hard fork of the core codebase.

## Context

In a Two-Repo Open Core model, the public repository must not contain the enterprise features, but it must provide "hooks" for them to attach to. Without clean interfaces, the private repository would have to fork and modify the core logic (e.g., the reconciler or the command runners), which creates a maintenance nightmare when trying to sync upstream changes.

## Proposed Solution

1. **Enterprise Hooks Interface:**
   - Create `internal/enterprise/hooks.go` in the public repository.
   - Define a comprehensive interface `EnterpriseHooks` that covers the integration points needed by Epics 34-37. Examples:
     - `OnReviewComplete(report ReviewReport)` (For Ticketing/Audit Logging)
     - `CheckMergeGate(diff GitDiff) error` (For Protected Paths/Accountability)
2. **No-Op Implementation:**
   - Create a `NoOpHooks` struct in the public repo that implements `EnterpriseHooks` by doing nothing.
   - The public `cmd/atcr` binary will instantiate the core engine using `NoOpHooks`.
3. **Engine Refactoring:**
   - Update `internal/reconcile` and `internal/git` to accept the `EnterpriseHooks` interface and call the hook methods at the appropriate lifecycle stages.
4. **Exported CLI Bootstrapping:**
   - Refactor `cmd/atcr/main.go` so the Cobra command tree can be easily imported and extended by the private repository's `cmd/atcr-enterprise/main.go`.

## Acceptance Criteria

- [ ] An `EnterpriseHooks` interface is defined in the public repo, capturing the lifecycle events needed for Audit, Integrations, and Compliance.
- [ ] The core engine (reconciler, executor) safely calls these hooks during execution.
- [ ] The public binary defaults to a `NoOp` implementation and exhibits no change in behavior.
- [ ] The private repo can inject a custom `EnterpriseHooks` implementation and successfully trigger side-effects (e.g., printing a log) during a review run.
