# Epic Plan 6.1: Skill Backend Swap

**Estimated Durations**: 2-3 weeks

## Objective

Modify the existing `execute-code-review` skill's multi-agent mode to invoke `atcr` instead of `llm-support`, and update `/reconcile-code-review` to accept atcr's output structure. This is the sole sub-plan of Epic 6.0 — the work is tightly coupled and cannot be split without leaving a broken pipeline.

## Context

Epic 6.0 describes the integration strategy: the skill's multi-agent mode replaces `llm-support review_direct` / `multi_review` with `atcr review --output-dir` + `atcr reconcile`, and the reconcile skill gains atcr's output as a source. This plan is the execution.

The skill lives in `~/Documents/GitHub/claude-prompts/.claude/skills/execute-code-review/multi-agent.md`. The reconcile skill lives in `~/Documents/GitHub/claude-prompts/.claude/skills/reconcile-code-review/`.

## Problem Statement

1. `multi-agent.md` Phase 2 Step 6 invokes `llm-support review_direct` or `llm-support multi_review`. These commands must be replaced with `atcr review --output-dir ${CR_OUT_DIR}` followed by `atcr reconcile`.
2. Step 8 verifies output files `td-stream.txt` and `multi-review-summary.json`. These must be replaced with atcr's actual output files.
3. Step 9 reads `multi-review-summary.json` for `reviewers`, `totalFindings`, `partial` fields. This must read atcr's `reconciled/summary.json` instead.
4. `/reconcile-code-review` auto-discovers sources by `td-stream.txt` filename. It must also discover atcr's `reconciled/findings.txt`.

## Proposed Solution

### Changes to `multi-agent.md`

**Phase 2, Step 6** — replace the `llm-support` invocation:

Before:
```
llm-support review_direct \
  --config "${CONFIG_PATH}" \
  --repo "${REPO_ROOT}" \
  ${RANGE_FLAGS} \
  ${SCOPE_FLAG} \
  --output-dir "${CR_OUT_DIR}"
```

After:
```
atcr review \
  --output-dir "${CR_OUT_DIR}" \
  ${RANGE_FLAGS}
```

Followed by:
```
atcr reconcile "${CR_OUT_DIR}"
```

Notes:
- `--config` and `--repo` are not needed — atcr reads config from `.atcr/config.yaml` and `~/.config/atcr/registry.yaml`, and the repo is the CWD.
- `${SCOPE_FLAG}` (`--sprint-plan`) is dropped — atcr has no sprint-plan concept (out of scope per 1.0). Reviewers fall back to diff-touched-line scope.
- `${RANGE_FLAGS}` maps directly: `--base`, `--head`, `--merge-commit` are the same in atcr.
- The `atcr review` command creates the review directory at `--output-dir`. `atcr reconcile` operates on that directory.

**Phase 3, Step 8** — update output verification:

Before:
```
paths: ["${CR_OUT_DIR}td-stream.txt", "${CR_OUT_DIR}multi-review-summary.json"]
```

After:
```
paths: [
  "${CR_OUT_DIR}sources/pool/findings.txt",
  "${CR_OUT_DIR}sources/pool/summary.json",
  "${CR_OUT_DIR}reconciled/findings.txt",
  "${CR_OUT_DIR}reconciled/summary.json"
]
```

**Phase 3, Step 9** — update summary reporting:

Before: read `multi-review-summary.json` for `reviewers`, `totalFindings`, `partial`.

After: read `reconciled/summary.json` for `sources` (count), `total_findings`, `partial`. The display format is unchanged — only the data source changes.

### Changes to `/reconcile-code-review`

The reconcile skill auto-discovers sources by scanning for `td-stream.txt` files under `code-review/`. It must also discover atcr's output:

1. Add `reconciled/findings.txt` as an additional source pattern (in addition to `td-stream.txt`).
2. When a source is `reconciled/findings.txt`, read it as 9-column format (SEVERITY|FILE:LINE|PROBLEM|FIX|CATEGORY|EST_MINUTES|EVIDENCE|REVIEWERS|CONFIDENCE) instead of 8-column.
3. Sources under `.atcr/reviews/` are NOT auto-discovered — only sources under the sprint's `code-review/` directory (which is where `--output-dir` points).

### Pre-flight check

Add a pre-flight check in Phase 1 Step 3 (backend selection) to verify the `atcr` binary is installed:

```
atcr --version
```

If the binary is not found, display an actionable error and HARD STOP.

## Success Criteria

- [ ] Output verification checks atcr's actual output files.

- [ ] `/reconcile-code-review` still reads legacy `td-stream.txt` (8-column format) for backwards compatibility.
- [ ] A sprint review run through the modified skill produces output that `/reconcile-code-review` can consume end-to-end.
- [ ] The `llm-support` invocation paths are removed from `multi-agent.md` (no dead code).

## Task Breakdown

1. Modify `multi-agent.md` Phase 2 Step 6: replace `llm-support` invocation with `atcr review` + `atcr reconcile`.
2. Modify `multi-agent.md` Phase 3 Step 8: update output file verification.
3. Modify `multi-agent.md` Phase 3 Step 9: update summary reporting to read atcr's summary.json.
4. Add `atcr --version` pre-flight check in Phase 1 Step 3.
5. Update `/reconcile-code-review` source discovery to accept `reconciled/findings.txt` (9-column format).
6. End-to-end validation: run a sprint review through the modified skill, verify reconcile consumes the output.
7. Remove dead `llm-support` invocation paths from `multi-agent.md`.

## Out of Scope

- Modifying the skill's default (host Claude) review mode.
- Changing atcr's output format.
- Sprint verification logic.
- Updating the `llm-support` CLI itself (it continues to exist for other consumers).

## Dependencies

- Epic 1.8 (`--output-dir` support in atcr) — must land first.
- ATCR binary built and installed.
- Access to `~/Documents/GitHub/claude-prompts/` to modify the skill and reconcile skill.

## Risks

| Risk | Likelihood | Impact | Mitigation |
|------|------------|--------|------------|
| Atcr's summary.json schema doesn't have the fields the skill expects | Medium | Medium | Read atcr's actual schema before implementation; the skill's display needs only source count, finding count, partial flag — easy to map |
| Reconcile skill's 9-column parser is not implemented | Medium | High | The 9-column format is documented in atcr's findings-format spec; implement the parser as part of this plan |
| Parallel-run validation reveals finding count discrepancies | Medium | Low | Expected — atcr's reconciler uses different dedupe/confidence logic. The pipeline must not break, but finding counts may differ. Document this in the skill |
| `--sprint-plan` scope flag is lost | Low | Medium | Atcr has no sprint-plan concept. Reviewers fall back to diff-touched-line scope. Document this behavioral change in the skill |
