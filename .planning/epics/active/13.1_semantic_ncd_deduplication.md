# Sprint Plan 13.1: Semantic NCD Deduplication

**Estimated Duration**: 4 days
**Status**: Draft

## Objective

Implement Normalized Compression Distance (NCD) using Go's fast `zstd` compression to calculate semantic similarity between review findings, bypassing the need for Python ML/NLP packages or LLM embeddings.

## Problem Statement

Standard text-overlap algorithms (Jaccard, TF-IDF) fail when two models identify the same issue using entirely different vocabularies. 

## Proposed Solution

-   Calculate `NCD(x, y) = (C(x+y) - min(C(x), C(y))) / max(C(x), C(y))`.
-   Concatenating and compressing two review findings provides a deterministic, language-agnostic semantic similarity score.

## Acceptance Criteria

- [ ] AC1: NCD algorithm implemented and bench-marked to process 1,000 finding comparisons in <50ms using `zstd`.
- [ ] AC2: Replaces the existing `internal/reconcile` text-similarity threshold logic.
- [ ] AC3: Benchmark suite proves NCD correctly identifies duplicate findings that previous lexical algorithms missed due to vocabulary differences.
