# Epic Plan 13.0: Reconciler v2 Semantic Algorithms (Overview)

**Estimated Durations**: 4 weeks total
**Status**: Draft

## Objective

Coordinate the rollout of the "v2" reconciler pipeline. Epic 13 transitions `github.com/atcr/reconcile` from legacy string-matching to advanced, deterministic semantic and structural heuristics. 

## Structure

Because this is a massive shift in core logic, the work is chunked into focused, parallelizable sprints:

- **[Sprint 13.1: Semantic Deduplication (NCD)](13.1_semantic_ncd_deduplication.md)**
  Replacing basic text-overlap with Normalized Compression Distance (zstd) for zero-LLM semantic similarity.
- **[Sprint 13.2: AST Plugin Architecture](13.2_ast_plugin_architecture.md)**
  Replacing proximity-based line grouping with AST Subgraph Isomorphism, leveraging dynamic Wasm plugins so we only load parsers for the language being reviewed.
- **[Sprint 13.3: Bipartite Matching & DBSCAN](13.3_resolution_bipartite_dbscan.md)**
  Replacing greedy clustering with optimal bipartite matching and density-based isolation of model hallucinations.
- **[Sprint 13.4: Deterministic PageRank](13.4_signal_pagerank.md)**
  Replacing flat vote-counting with inherited, graph-based confidence scoring for the reviewer panel.

## Constraints

-   **Zero LLM in the loop**: The reconciler MUST remain 100% deterministic and offline.
-   **Stable Integration**: Each sprint must cleanly integrate into the extracted `github.com/atcr/reconcile` module without breaking the public `Result` API contract established in Epic 8.0.
