# Epic Plan 1.0: atcr Core — Review Engine, Reconciler, and Skill

**Estimated Durations**: 4-6 weeks

## Objective

Build v1 of **atcr (Agent Team Code Review)**: a standalone, shareable tool that fans a code change out to a panel of heterogeneous LLM reviewer personas, then deterministically reconciles their findings into a single deduplicated, confidence-scored deliverable. Composed of a single Go binary (CLI + MCP server over one engine) and a companion Agent Skill that contributes the host-model review and orchestrates the flow.

This is a **fresh rewrite** informed by the review subsystem in `llm-tools` (Go) and `claude-prompts` (skills) — not an extraction. The two codebases evolve independently.

## Context

The source system runs code reviews as a sprint-pipeline stage: `/execute-code-review` (host Claude review + optional multi-agent fan-out via the `llm-support` CLI) followed by `/reconcile-code-review` (prompt-driven merge into a technical-debt backlog). The differentiated core — heterogeneous reviewer panels, machine-parseable findings streams, and cross-reviewer agreement as a confidence signal — is general-purpose and worth sharing as "a review panel, not a reviewer."

Key architectural fact discovered during design: dedupe and confidence computation do not exist in Go today — `MergeStreams` in `llm-tools` is concatenative, and all clustering/dedupe/confidence logic lives as prompt instructions in the reconcile skill. The deterministic Go reconciler is therefore the largest piece of net-new work and the core value-add of atcr.

## Problem Statement

1. Existing LLM review tools (CodeRabbit, Qodo PR-Agent, Copilot review, Claude Code `/code-review`) are single-model, single-vendor, mostly SaaS. None offer a local-first, BYO-keys panel of diverse models with cross-model agreement scoring.
2. Anyone who manually fans a diff out to N models gets N walls of prose with no way to merge them. The merge/dedupe/confidence step is the hard part and nobody ships it.
3. The source system's implementation is unusable standalone: coupled to sprint paths, `.planning/` structure, DoD checks, pre-seeded TD captures, and a TD-backlog README.

## Proposed Solution

### Architecture

```
┌────────────────────────────────────────────────┐
│              LLM Client / IDE / CI             │
└───────┬───────────────────┬────────────────────┘
        │ MCP (atcr serve)  │ Skill (host review + orchestration)
        v                   v
┌───────────────────────────────────────────────┐
│            atcr binary (Go, CLI-first)        │
│  range resolution · payload build · fan-out   │
│  findings parsing · reconcile · report        │
└───────────────────────────────────────────────┘
```

Design principle (carried over): prompts orchestrate; the Go binary does everything that must be deterministic (ranges, payloads, fan-out, merge, dedupe, confidence, validation); every cross-stage handoff is a machine-parseable file on disk.

### Command surface

| Command | Purpose |
|---------|---------|
| `atcr review [--id <id>] [--base X --head Y \| --merge-commit SHA]` | Resolve range, build payloads, create the review directory, fan out to the persona pool, write per-agent artifacts + merged findings |
| `atcr reconcile [<id-or-path>]` | Discover sources, normalize, cluster, dedupe, compute confidence, write reconciled artifacts. Defaults to `.atcr/latest` |
| `atcr report [<id-or-path>] --format md\|json\|checklist` | Render views over reconciled findings |
| `atcr range [--base X --head Y \| --merge-commit SHA]` | Pre-flight range resolution only; prints resolution JSON |
| `atcr init` | Write project `config.yaml` and editable persona files from embedded defaults |
| `atcr serve` | MCP stdio server wrapping the same engine |

**Anchor ergonomics:** `atcr review` creates the review directory and records its id in `.atcr/latest`; every later command takes the id or path as its single anchor argument, defaulting to latest. `atcr review && atcr reconcile` works with zero arguments on a feature branch.

**Exit-code semantics (CI integration):** `atcr reconcile --fail-on <severity>` (and `atcr review --fail-on` for one-shot runs) returns nonzero when any finding at/above the threshold survives reconciliation. This makes atcr a PR gate with no glue code.

### Filesystem layout

```
.atcr/                          # repo root
  config.yaml                   # project config: roster, payload mode, timeouts, fail-on
  latest                        # pointer file containing the most recent review id
  reviews/
    <id>/                       # default id: <YYYY-MM-DD>_<branch-slug>; --id overrides
      manifest.json             # base/head SHAs, detection mode, payload mode(s), roster, timestamps
      payload/                  # what reviewers saw (diff.txt, blocks/, or files/)
      sources/
        pool/                   # written by `atcr review`
          raw/<agent>/{review.md, findings.txt, status.json}
          findings.txt          # merged across agents
          summary.json
        host/                   # written by the Skill (+1 host-model reviewer)
          review.md
          findings.txt
      reconciled/               # written by `atcr reconcile`
        findings.txt            # 9-col with REVIEWERS + CONFIDENCE
        findings.json
        report.md
        ambiguous.json          # clusters needing semantic adjudication
        summary.json
```

**Source discovery rule:** any child of `sources/` containing a `findings.txt` is a reconcile source. `sources/` is an open extension point — CI or other tools can drop `sources/<name>/findings.txt` and reconcile picks it up with zero config. `reconciled/` is output, never an input source.

### Findings stream format (the public contract)

Versioned, documented, pipe-delimited. Per-source files carry a `# atcr-findings/v1` header.

Per-source (8 columns):
```
SEVERITY|FILE:LINE|PROBLEM|FIX|CATEGORY|EST_MINUTES|EVIDENCE|REVIEWER
```

Reconciled (9 columns):
```
SEVERITY|FILE:LINE|PROBLEM|FIX|CATEGORY|EST_MINUTES|EVIDENCE|REVIEWERS|CONFIDENCE
```

Rules carried over: SEVERITY ∈ {CRITICAL, HIGH, MEDIUM, LOW}; extraction by strict severity-prefix regex (`^(CRITICAL|HIGH|MEDIUM|LOW)\|`) so prose mentions are ignored; literal `|` within fields replaced with `/`; short rows padded to full column count before merge.

### Payload modes (per-agent reviewer input)

A first-class axis — small and low-active-parameter MoE models produce better findings from real code than from unified diffs.

| Mode | Reviewer sees | Notes |
|------|---------------|-------|
| `blocks` | Changed hunks expanded to the enclosing function/block with real line numbers (git `--function-context` baseline) | **Default.** Sweet spot for small models: readable code, still scoped to the change |
| `diff` | Unified diff | Most compact and token-friendly; right choice for frontier models and large ranges. Documentation must state this clearly |
| `files` | Full head-version content of changed files with changed regions marked | Highest token cost; small ranges or audit-style review |

- Project default in `config.yaml`; per-agent override in the registry (`payload: blocks`). One run can mix payloads; `manifest.json` records who saw what.
- Per-payload byte budget with deterministic truncation (drop whole files by size rank); anything dropped is recorded in the agent's `status.json` — never silent.
- Persona prompts carry a scope rule per payload mode (`files` mode surfaces pre-existing issues; reconcile flags findings outside changed ranges).

### Configuration

`~/.config/atcr/registry.yaml` (user-level — providers and agents):
```yaml
providers:
  <name>: { api_key_env: <ENV_VAR>, base_url: <openai-compatible endpoint> }
agents:
  <name>:
    provider: <provider>        # required
    model: <model-id>           # required
    temperature: 0.7            # optional
    timeout_secs: 600           # optional
    rate_limited: false         # optional → serial lane
    fallback: <agent>           # optional; chains validated at load (dangling + cycle = error)
    payload: blocks             # optional per-agent override
```

`.atcr/config.yaml` (project-level): roster (`agents`, `serial_agents`), default payload mode, timeouts, `fail_on`, output root. Precedence: CLI flag > project config > registry > embedded default.

Persona prompt resolution chain (carried over): `--task-message` flag > `<agent>.md` in the registry dir > `_base.md` > embedded default. Prompts are Go text/templates with payload vars (`{{.Payload}}`, `{{.PayloadMode}}`, `{{.FileCount}}`, `{{.BaseRef}}`, `{{.HeadRef}}`, `{{.AgentName}}`, large-payload guidance vars).

Ships with six embedded personas (installed editable via `atcr init`): **bruce** (generalist/correctness), **greta** (algorithmic correctness), **kai** (architecture/design fit), **mira** (production feasibility), **dax** (test coverage/error paths), **otto** (style/readability/idiom).

### Range resolution

Decision tree (carried over verbatim in behavior): explicit `--base`/`--head` → `--merge-commit SHA` (base = SHA^, head = SHA) → auto: merge-base of HEAD against the detected default branch (`origin/HEAD` symbolic-ref → `origin/main` → `origin/master` → local `main` → local `master`). Empty range (0 commits or base == head) is a **hard error** before any provider call — never a silent zero-findings pass. Shallow clones detected with `git fetch --unshallow` guidance.

### Fan-out engine

Carried-over behaviors that must not be lost:
- Global timeout wraps the whole operation via context; parallel agents drain cleanly via WaitGroup even on cancellation; serial agents check context before each invocation.
- Parallel lane (default) + serial lane (rate-limited providers), fallback agents tried on primary failure with `fallback_used`/`fallback_from` recorded.
- Partial-success semantics: error only if **all** agents fail; `partial: true` recorded in summary when some fail.
- OpenAI-compatible client with retry on 429/5xx; tuned retry (≈500 ms initial, 1.5× backoff) so retries don't exhaust the agent timeout.
- API keys resolved from env vars at invoke time with clear errors; agent names sanitized (`filepath.Base`) against path traversal.
- Crash-safe incremental writes; per-agent `status.json` always written (ok | failed | timeout).

### Reconciler (net-new Go work — the core of v1)

Hybrid design: deterministic Go for everything mechanical, optional model adjudication only where it earns its keep.

Go pipeline:
1. **Discover** sources under `sources/` (allowlist via `--sources` optional).
2. **Normalize** rows to internal records; tolerate short rows by padding; skip comments/blanks.
3. **Cluster** by (FILE, LINE ± 3).
4. **Dedupe within clusters** by normalized PROBLEM text similarity (deterministic: token-set similarity with threshold).
5. **Merge duplicates:** REVIEWERS comma-joined; SEVERITY = max with `disagreement: <lo> vs <hi>` annotation preserved; PROBLEM/FIX = longest/most detailed; CATEGORY = modal; EST_MINUTES = max.
6. **Confidence:** HIGH = 2+ distinct reviewers; MEDIUM = single reviewer; LOW reserved for untrusted sources.
7. **Ambiguity sidecar:** same-location clusters whose PROBLEM texts fall in the similarity gray zone are written to `ambiguous.json` (not silently merged or split). The Skill may adjudicate them semantically and re-run the merge step; without adjudication they default to unmerged (conservative).
8. **Emit** `findings.txt`, `findings.json`, `report.md`, `summary.json`.

### Deliverables of a run

1. `reconciled/findings.txt` — the machine contract (integration surface for downstream tools, including the source system's `/reconcile-code-review`).
2. `reconciled/findings.json` — structured records for scripting.
3. `reconciled/report.md` — human report: executive summary (counts by severity × confidence, panel table with per-reviewer status/model/duration/finding count), findings grouped by severity with problem/fix/evidence/reviewers/disagreements.
4. `reconciled/summary.json` — run stats (sources, per-source counts, clusters collapsed, disagreements, partial flag).
5. `atcr report --format checklist` — render-only markdown checkbox table (a view, not a backlog: no global numbering, no persistence, no state).

### The atcr Skill

Lives in `skill/` in this repo, installable into `.claude/skills/atcr/`.
- Runs the **host-model review** (the "+1" reviewer): reviews the payload adversarially and writes `sources/host/{review.md, findings.txt}` in the v1 format — so a user with a single API key still gets 2+ sources and a working confidence signal.
- Orchestrates: `atcr range` pre-flight → `atcr review` (background, polled) → host review → `atcr reconcile` → present `report.md`.
- Optionally adjudicates `ambiguous.json` clusters and re-invokes the merge.
- No sprint knowledge: input is a git range/branch/PR; output is the review directory.

### MCP server (`atcr serve`)

Stdio MCP exposing the engine as tools (`atcr_review`, `atcr_reconcile`, `atcr_report`, `atcr_range`, `atcr_status`) via the official `modelcontextprotocol/go-sdk`. Thin layer over the same internal packages as the CLI — no logic in the handlers.

## Success Criteria / Acceptance Criteria

### Functional
- [ ] `atcr init` produces a working config and six editable persona files; `atcr review && atcr reconcile` runs end-to-end on a feature branch with zero arguments.
- [ ] Range resolution follows the decision tree exactly; empty range and shallow-clone cases produce hard errors with guidance text.
- [ ] All three payload modes produce correct payloads; per-agent payload override works; truncation is recorded, never silent.
- [ ] Fan-out runs parallel + serial lanes with fallbacks; partial failure produces `partial: true` and exit 0 if ≥1 agent succeeded; all-fail produces nonzero exit.
- [ ] Reconciler merges multi-source findings: duplicates across reviewers collapse with REVIEWERS joined and CONFIDENCE=HIGH; severity disagreements preserved inline; ambiguous clusters land in `ambiguous.json`.
- [ ] `--fail-on <severity>` exit-code gate works for CI.
- [ ] `atcr report` renders md, json, and checklist formats from the same reconciled data.
- [ ] `atcr serve` exposes the engine over MCP stdio; tools callable from Claude Code.
- [ ] The Skill performs host review, writes v1-format findings to `sources/host/`, and the full orchestration loop produces a reconciled report.

### Quality
- [ ] Findings-stream format documented as a versioned spec (`docs/findings-format.md`).
- [ ] README rewritten around the actual architecture (panel + reconcile), including payload-mode guidance that explicitly states `diff` is the most compact/token-friendly mode and `blocks` is the default for small-model quality.
- [ ] Unit tests cover: range decision tree, payload builders, stream parse/escape/pad, fan-out concurrency/fallback/timeout (httptest provider mocks), reconciler clustering/dedupe/confidence/disagreements, exit codes.
- [ ] `go vet` and `golangci-lint` clean; CI green.

## Task Breakdown (dependency order)

1. **Scaffold:** Go module, cobra CLI skeleton, `internal/` package boundaries (`gitrange`, `payload`, `registry`, `llmclient`, `fanout`, `stream`, `reconcile`, `report`, `mcp`), CI wiring to existing `.github/workflows/ci.yml`.
2. **Config + init:** registry.yaml and `.atcr/config.yaml` loading, precedence rules, load-time fallback-chain validation (dangling + cycle), `atcr init`, embedded provider defaults.
3. **gitrange:** resolution decision tree, default-branch detection, empty-range hard error, shallow detection, diff computation; `atcr range` command.
4. **Payload engine:** `diff`, `blocks` (function-context expansion), `files` builders; byte budgets + deterministic truncation with recording; payload template vars.
5. **Personas + prompt resolution:** six embedded persona templates with per-payload-mode scope rules; resolution chain (flag > agent.md > _base.md > embedded).
6. **LLM client:** OpenAI-compatible chat client, retry policy (429/5xx, tuned backoff), per-agent temperature/timeout.
7. **Fan-out engine:** parallel/serial lanes, fallback invocation, global-timeout context handling, partial-success semantics, per-agent artifacts, merged pool findings, `summary.json`; `atcr review` command + review-directory/manifest/`latest` management.
8. **Findings stream package:** v1 spec, extraction regex, escaping, padding, parser/writer shared by fan-out and reconcile.
9. **Reconciler:** discovery, normalization, clustering (FILE, LINE±3), deterministic text-similarity dedupe, merge rules, confidence, disagreement annotation, `ambiguous.json` sidecar, all four reconciled artifacts; `atcr reconcile` + `--fail-on`.
10. **Report:** md/json/checklist renderers over `findings.json`; `atcr report`.
11. **MCP server:** `atcr serve` with tool handlers over the engine.
12. **Skill:** `skill/SKILL.md` + instructions (host review writing v1 findings, orchestration loop, ambiguity adjudication path).
13. **Docs:** README rewrite, `docs/findings-format.md`, `docs/registry.md`, `docs/payload-modes.md` (with the diff-vs-blocks token guidance), CI gate examples.

## Technical Constraints

- Go 1.24+; key dependencies: `spf13/cobra`, `gopkg.in/yaml.v3`, `modelcontextprotocol/go-sdk`. Keep the dependency tree small — this is a shareable tool.
- All provider interaction via OpenAI-compatible `/chat/completions`; no provider-specific SDKs in v1.
- No filesystem writes outside the review directory and config locations.
- Single binary; MCP and CLI share one engine, no logic forks.

## Out of Scope (v1)

- Openclaw / remote-container backend (dropped permanently — direct API only).
- Sprint-pipeline coupling: SPRINT_PATH, DoD/work-item verification, test/coverage/lint gates, pre-seeded TD captures, merge-commit file scraping (the `--merge-commit` flag stays).
- Backlog management (TD README, global numbering, persistence) — downstream consumers own this.
- SARIF output and GitHub PR-comment posting (candidate v2).
- Web UI, hosted service, telemetry.

## Risks

| Risk | Likelihood | Impact | Mitigation |
|------|------------|--------|------------|
| Deterministic text-similarity dedupe under/over-merges vs. the prompt-based judgment it replaces | Medium | High | Conservative threshold + `ambiguous.json` sidecar; Skill adjudication path; tune with fixture corpora from real multi-agent runs |
| `blocks` payload builder edge cases (languages without braces, generated files, renames, binary files) | Medium | Medium | Fall back to plain `-U<n>` context diff per file when function-context fails; explicit tests per case |
| `files` mode produces out-of-change findings that pollute reconciliation | Medium | Medium | Per-payload scope rules in personas; reconcile annotates findings outside changed ranges |
| Token cost surprises for `blocks`/`files` on large ranges | Medium | Medium | Byte budgets with recorded truncation; docs steer large ranges to `diff` |
| Format spec churn after publication | Low | High | Version header (`atcr-findings/v1`) from day one; additive-only evolution policy in the spec |
