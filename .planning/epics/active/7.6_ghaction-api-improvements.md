# Tech Debt: GitHub Action API Improvements (7.6)

- **Estimated time**: 2–3 days
- **Tasks/Components**: 2 items, 2 files (internal/ghaction/client.go, internal/ghaction/render.go)

## Current State

The `internal/ghaction` package has two related inefficiencies deferred from group 6's `cmd/atcr/github.go` scope:

1. **Sequential inline comment posting** (`postInlineComments` in `cmd/atcr/github.go:180-191`) posts one review comment per finding in a loop with a 100ms delay between calls. For PRs with many findings this is slow and API-call-heavy.
2. **Double `Conclusion` computation** (`runGithub` in `cmd/atcr/github.go:115-116`) calls `ghaction.Conclusion(findings, failOn)` directly, then immediately calls `ghaction.BuildCheckOutput(findings, failOn)` which internally calls `Conclusion` again at `internal/ghaction/render.go:102`. The result is two identical O(N) traversals of the findings slice.

Both fixes require changes to `internal/ghaction/` — outside group 6's write scope — and belong together since they touch the same package boundary.

## Pain Points

### Sequential Inline Comment Posting
- Each finding posted as a separate `POST /repos/{owner}/{repo}/pulls/{pull_number}/comments` call
- GitHub's batch review endpoint (`POST /repos/{owner}/{repo}/pulls/{pull_number}/reviews`) accepts an array of comments in one request
- 100ms inter-call delay was added for rate-limit mitigation but doesn't address the fundamental O(N) API call count

### Double Conclusion Computation
- `ghaction.Conclusion` is called twice with identical inputs (`findings`, `failOn`)
- Fix description in the original TD item was a verbatim copy of Q1's fix text (copy-paste error) — actual fix is `BuildCheckOutput`'s return signature
- Impact is low (O(N) over findings, both results used), but the API surface is misleading

## Proposed Solution

### Item 1: Batch Review POST
1. Add `CreateReview(ctx, owner, repo, pr, sha string, comments []CommentRequest) error` to `ghaction.Client` in `internal/ghaction/client.go`
2. Implement using `POST /repos/{owner}/{repo}/pulls/{pull_number}/reviews` with `event: "COMMENT"` and the `comments` array
3. Update `postInlineComments` in `cmd/atcr/github.go` to call `CreateReview` instead of looping `CreateReviewComment`
4. Retain per-comment fallback if batch fails (some GitHub Enterprise versions may not support the endpoint)

### Item 2: Single Conclusion Computation
1. Change `BuildCheckOutput` in `internal/ghaction/render.go` to return `(CheckOutput, string, int)` (the output, conclusion, and failCount)
2. Update `runGithub` in `cmd/atcr/github.go` to use the returned values instead of calling `Conclusion` separately
3. Update `BuildCheckOutput`'s callers and tests accordingly
