# Feature Request: Final Review and Documentation Sweep

- **Estimated time**: 2-3 days
- **Tasks/Components**: TBD
- **Execution**: init-plan

## Context

Following the completion of the core engine documentation updates (Epic 22.0) and the built-in reviewer persona renaming/generalization (Epic 23.0), we need a final documentation sweep in the `atcr` repository. This ensures that all CLI commands, configurations, and the renamed personas (Sasha, Penny, Ingrid/Ian) are perfectly documented before updating the public website (`atcr.dev` via Epic 2.0).

## Problem Statement

When the `atcr.dev` website is updated, it must pull directly from a completely finalized and accurate source of truth inside the `atcr` repository. Any discrepancies or stale references (such as lingering mentions of `sentinel`, `tracer`, or `idiomatic` in user manuals or help commands) will lead to user confusion or incorrect website documentation.

## Proposed Solution

1.  **Code-to-Docs Audit**: Conduct a final audit of all user-facing documentation (`README.md`, `docs/`, inline command help text, and schemas) against the final production codebase.
2.  **Verify Persona References**: Ensure that all documentation references the new human persona names (`sasha`, `penny`, `ingrid`/`ian`) and their generalized multi-language capabilities.
3.  **Ensure Website Compatibility**: Verify that the markdown files under `docs/` are formatted cleanly and are ready to be used as the source of truth for the `atcr.dev` website repository.

## Acceptance Criteria

- [ ] AC1: No legacy persona names (`sentinel`, `tracer`, `idiomatic`) remain in any documentation or command help screens.
- [ ] AC2: All new features up to Epic 23.0 are fully and accurately documented in the core `atcr` repository.
- [ ] AC3: Documentation files are validated and ready to be imported into the `atcr.dev` repository.
