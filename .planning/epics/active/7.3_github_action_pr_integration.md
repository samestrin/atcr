# Epic 7.3: GitHub Action / PR Integration

**Status:** Draft (stub — created 2026-06-19 to track a gap surfaced in epic reconciliation)
**Created:** 2026-06-19
**Priority:** High (the primary delivery surface for findings; differentiator)

## Why this exists

Epic 7.0 (Executor Model for Fix Generation) Phase 4 was written as "**update** the GitHub
Action to post inline comments with fixes" — but **ATCR has no GitHub Action today**. The
repo has only `.github/workflows/ci.yml` (its own CI); there is no `action.yml`, no
published Action, and no PR-comment surface. 7.0 assumed an artifact that does not exist, so
the PR-delivery work was unowned. This epic owns it.

## Problem

ATCR produces excellent findings artifacts (`reconciled/findings.txt`, 9-column format) but
there is no first-class way to surface them on a GitHub pull request. Teams that want
"review my PR" must wire up their own glue. Without a maintained Action, the findings — and
the `FIX` column that Epic 7.0 populates — never reach the developer where they work.

## Goal

Ship a maintained, composable GitHub Action that runs `atcr review` + `atcr reconcile` on a
pull request and surfaces the result as a PR check and (optionally) inline comments. When
Epic 7.0's executor has populated the `FIX` column, the Action renders the fix inline —
the "like lgtmaybe" experience, on top of ATCR's multi-model panel.

## Scope (draft)

### In Scope

- An `action.yml` (composite or Docker action) that installs/locates the `atcr` binary, runs
  `atcr review --output-dir … <range>` + `atcr reconcile`, and parses `reconciled/findings.txt`.
- A summary PR check (pass/fail honoring `--fail-on`) with a findings table.
- Optional **inline comments** keyed off `FILE:LINE`, rendering `PROBLEM` and, when present,
  the `FIX` column (Epic 7.0) and executor attribution.
- A toggle to post artifacts only (no inline comments).
- An example workflow in `docs/` and a real-PR integration test.

### Out of Scope

- Fix **generation** (Epic 7.0 owns the `FIX` column; this epic only renders it).
- Hosting/SaaS; non-GitHub forges (GitLab/Bitbucket) — separate epics if demanded.
- Auto-applying fixes / opening fix PRs.

## Acceptance Criteria (draft — refine before scheduling)

- [ ] AC1: `action.yml` exists; a workflow can call it on `pull_request` and it runs `atcr review` + `atcr reconcile`.
- [ ] AC2: The Action posts a PR check summarizing findings and honoring `--fail-on` for the merge gate.
- [ ] AC3: Inline comments render `PROBLEM` at `FILE:LINE`; when the `FIX` column is populated (Epic 7.0), the fix and executor attribution are shown — format "ATCR found: [problem]. Fix: [fix]. Suggested by: [executor]".
- [ ] AC4: A toggle disables inline comments (artifacts/check only).
- [ ] AC5: Real-PR integration test demonstrates the end-to-end flow.

## Dependencies

- **Epic 7.0 (Executor Model)** — supplies the `FIX` column this Action renders (soft: the Action also works on findings without fixes, posting `PROBLEM` only).
- A published/locatable `atcr` binary (release artifact or build step in the Action).
