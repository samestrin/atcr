# Epic Plan 22.0: Process-Group Reaping on Validation Timeout

- **Estimated time**: 0.5 day
- **Tasks/Components**: 2/1

## Objective

When a local validation command times out, kill its entire process group (not just the direct child) so grandchild processes spawned by shells like `sh -c make ...` are reaped instead of left running.

## Context

`exec.CommandContext` in `internal/verify/localvalidate.go` SIGKILLs only the direct child on timeout. `cmd.WaitDelay` force-closes the parent's pipe fds but does not reap the process group, so a validation command that spawns subprocesses leaves grandchildren orphaned and still running after the deadline. This was originally deferred (TD-006, sprint-plan §2.5 of the `17.0_auto_merged_fixes` sprint) as "document, do not implement process-group kill now" — deferred to a real plan per the current TD-citation rule rather than left as a bare TD/sprint-plan reference.

## Task Breakdown

1. Set `cmd.SysProcAttr.Setpgid = true` (unix build-tagged) when constructing the validation command in `internal/verify/localvalidate.go`.
2. On the cancel/timeout path, kill the whole process group via `syscall.Kill(-pid, syscall.SIGKILL)` instead of relying on `cmd.WaitDelay` alone.

## Acceptance Criteria

- [ ] A validation command that spawns a subprocess (e.g. `sh -c 'sleep 100'`) has both the shell and the sleep reaped when the validation timeout fires.
- [ ] Non-timeout validation runs (pass/fail without a spawned subprocess) are unaffected.
- [ ] `go test ./...` passes; a new regression test in `internal/verify` covers the grandchild-reaping behavior on unix.

## Components Touched

## Out of Scope
