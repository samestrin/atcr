# Epic Plan 26.0: .atcrignore Token Waste Protection

- **Estimated time**: TBD
- **Tasks/Components**: TBD
- **Execution**: init-plan

## Objective

Prevent ATCR from wasting LLM tokens (and money) on auto-generated code, vendored dependencies, or massive lockfiles by respecting `.gitignore` and introducing a native `.atcrignore` filtering layer.

## Context

If a developer runs a codegen tool or bumps a massive `vendor/` dependency, the resulting git diff can be hundreds of thousands of lines. While Epic 19.5 handles truncation and dropping files to fit the token budget, it does so based on size/priority. It is far safer to explicitly never send these files to the payload builder in the first place.

## Proposed Solution

1. **File Filtering Layer**: Update the diff ingestion logic (`internal/payload/builder.go` or `internal/gitrange`) to completely exclude files that match standard ignore patterns.
2. **.gitignore Support**: By default, parse and respect the repository's `.gitignore`.
3. **.atcrignore Support**: Support a dedicated `.atcrignore` for files that should be committed to git but ignored by the AI (e.g., `go.sum`, `package-lock.json`, `docs/`).

## Acceptance Criteria

- [ ] Files matching `.gitignore` are excluded from the review payload.
- [ ] Files matching `.atcrignore` are excluded from the review payload.
- [ ] `atcr review` logs a trace/debug message noting which files were skipped due to ignore rules.

## Components Touched

- `internal/gitrange` or `internal/payload`
- `cmd/atcr`

## Out of Scope

- Complex globbing beyond standard `.gitignore` syntax.

## Dependencies

- None.
