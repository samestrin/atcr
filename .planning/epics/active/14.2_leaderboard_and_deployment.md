# Sprint Plan 14.2: Leaderboard & Production Deployment

- **Estimated time**: 5 days
- **Tasks**: 6
- **Components**: 3

## Context

The final piece of the `atcr.dev` launch is establishing credibility (via the public model-evaluation leaderboard) and deploying the code so pilot signups can begin. This sprint integrates the evaluation data, sets up lead generation, and configures the automated deployment pipeline to static hosting.

## Problem Statement

We need to:
1. Port and format the evaluation table on the leaderboard page (`leaderboard.html`), displaying real findings and scores generated in Epic 10.0.
2. Establish the signup form on the Team Edition page (`team-edition.html`) to capture volunteer teams for the Epic 15.0 pilot program.
3. Configure static deployment pipelines (e.g., GitHub Pages or Cloudflare Pages) linked to the `atcr.dev` custom domain.
4. Finalize SEO metadata, favicons, and error pages (`404.html`).

## Proposed Solution

1. Port mockup structures for [leaderboard.html](file:///Users/samestrin/Documents/GitHub/atcr.dev/the-artifact/mockups/leaderboard.html) and [team-edition.html](file:///Users/samestrin/Documents/GitHub/atcr.dev/the-artifact/mockups/team-edition.html).
2. Populate the leaderboard ledger table using evaluation numbers derived from Epic 10.0.
3. Hook up the pilot signup form fields to a secure static form provider (such as Formspree, Netlify Forms, or Cloudflare Workers endpoint).
4. Configure DNS records for the custom `atcr.dev` domain and create a GitHub Actions deployment workflow.

## User Personas

### Primary: Software Engineer (Devin)
- Wants to compare model performance on the conformance suite to verify reviewer accuracy.
- Wishes to sign up their team for the Team Edition Pilot.

## User Journeys

### Leaderboard Review & Pilot Signup
1. Accesses `atcr.dev/leaderboard.html`.
2. Inspects model rankings, consensus scores, and benchmark counts.
3. Clicks the "Team Edition" navigation link to read the validation program criteria.
4. Fills out the pilot application form (team size, stack, pain points, email) and hits submit.
5. Receives a friendly confirmation screen.

## Acceptance Criteria

- [ ] AC1: The leaderboard page correctly displays ranked models with agreement metrics, structured columns, and no styling layout bugs.
- [ ] AC2: The Team Edition signup form handles field validations (e.g., email syntax, mandatory fields) and submits payload securely via HTTPS.
- [ ] AC3: Submitting the pilot form successfully records lead information and presents a clean thank-you confirmation state.
- [ ] AC4: The custom domain `atcr.dev` resolves properly with active HTTPS certificates.
- [ ] AC5: Merge events to the `main` branch of the `atcr.dev` repo trigger a GitHub Action that deploys the static files within 2 minutes.

## Success Criteria

### Functional
- [ ] Form submit round-trip takes less than 2 seconds.
- [ ] Static pages load correctly over IPv4 and IPv6 networks.

### Business
- [ ] Form inputs are structured cleanly so the pilot team coordinates can be exported easily in CSV/JSON format for validation tracking.

## Technical Constraints

- The deployment pipeline must run as an automated GitHub Action.
- All secrets (such as Formspree/API target IDs) must be injected securely or kept client-safe via obfuscation or proxy configurations.

## Out of Scope

- Building a private dashboard for signup management (spreadsheet/email lists are sufficient for validation).
- Live performance updating (updates will be batched manually or scheduled weekly).

## Dependencies

- Conformance benchmarking data generated in Epic 10.0.
- Custom domain registration (`atcr.dev`) ownership and DNS provider access.

## Risks

| Risk | Likelihood | Impact | Mitigation |
|------|------------|--------|------------|
| Lead capture form gets targeted by spam bots | Medium | Medium | Integrate simple client-side honeypot fields or basic form provider bot protection filters |
| DNS propagation delay | Low | Low | Set short TTL values on the DNS records prior to switching the nameservers or custom domains |
