# Epic Plan 27.0: Local Model / Ollama Persona Integration

- **Estimated time**: TBD
- **Tasks/Components**: TBD
- **Execution**: init-plan

## Objective

Deliver a "Zero Data Egress" out-of-the-box local persona (using Ollama or llama.cpp) in the community registry to completely resolve enterprise privacy objections during onboarding.

## Context

While LiteLLM and proxy configurations are supported, privacy-conscious enterprise teams often refuse to send proprietary code to external APIs like OpenAI or Anthropic. Having a first-class, officially supported local persona (e.g., pointing to `localhost:11434` with a highly optimized prompt for `llama3` or `qwen2.5-coder`) makes the "privacy first" adoption path friction-free.

## Proposed Solution

1. **Local Persona Definition**: Author a persona (e.g., `local-reviewer`) in `registry.yaml` configured to hit a standard local Ollama/llama.cpp endpoint.
2. **Prompt Tuning**: Tune the system instructions specifically for the capabilities and weaknesses of the best open-weight 8B-32B coding models (which require different prompting than Claude 3.5).
3. **Documentation**: Add a "Local / Privacy-First Quickstart" to the docs explaining how to `brew install ollama`, pull the model, and run `atcr` entirely offline.

## Acceptance Criteria

- [ ] `registry.yaml` includes a ready-to-use local persona.
- [ ] A local model can successfully execute a review pipeline using this persona without external network calls.
- [ ] Documentation clearly outlines the zero-egress setup.
