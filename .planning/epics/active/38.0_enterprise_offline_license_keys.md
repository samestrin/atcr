# Epic Plan 38.0: Enterprise Offline License Key Verification

- **Estimated time**: TBD
- **Tasks/Components**: TBD
- **Execution**: init-plan

## Objective

Implement a cryptographically secure, offline-verifiable license key system to unlock enterprise features (Audit Logging, Protected Paths, Accountability Gates, Integrations) within a single compiled `atcr` binary. 

## Context

To minimize distribution overhead (e.g., maintaining private repositories, separate build pipelines, and duplicate documentation), ATCR will use a single-binary distribution model. Both free and enterprise users will download the exact same `atcr` executable. 

However, because ATCR's core value proposition is data privacy ("keep your code on your own machines"), the enterprise license check cannot "phone home" to an ATCR server. The verification must happen 100% locally and offline.

## Proposed Solution

1. **Cryptographic JWTs (JSON Web Tokens):**
   - ATCR HQ will generate licenses as JWTs signed with a private cryptographic key (e.g., Ed25519). 
   - The JWT payload will contain the customer name, tier, and an expiration date.
2. **Embedded Public Key:**
   - The `atcr` Go binary will have the corresponding public key compiled directly into it.
3. **Local Verification:**
   - On execution, `atcr` will look for the license via an environment variable (e.g., `ATCR_LICENSE_KEY`) or a local file (e.g., `.atcr/license`).
   - The `internal/enterprise` package will parse the JWT, verify its signature against the embedded public key, and check that it is not expired.
   - This requires **zero network requests**.
4. **Feature Gating:**
   - Enterprise features (Epics 34, 35, 36, 37) will be wrapped in conditional checks: `if license.IsValid() { ... }`.
5. **Key Generator Utility:**
   - Build a small, separate internal tool (or hidden CLI command) for the founders to generate these JWTs when a customer purchases a subscription.

## Acceptance Criteria

- [ ] `atcr` can securely verify a signed JWT license key locally with no network requests.
- [ ] An expired, invalid, or missing license key gracefully degrades `atcr` to the free tier (no enterprise features).
- [ ] A valid license key successfully unlocks enterprise modules.
- [ ] An internal utility exists to mint new licenses with specified expiration dates.

## Components Touched

- `internal/enterprise/` (New package for license parsing/verification)
- `cmd/atcr/` (Loading the license on startup and embedded public key)
- `cmd/atcr-keygen/` (Internal utility for minting licenses)
