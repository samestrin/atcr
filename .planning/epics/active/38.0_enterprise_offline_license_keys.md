# Epic Plan 38.0: Enterprise Two-Repo Infrastructure & Module Setup

## Objective

Pivot ATCR to a True Open Core (Two-Repo) architecture. Create the private `samestrin/atcr-enterprise` repository, set up its Go module imports to consume the public `atcr` core, and establish the CI/CD pipeline for building the enterprise binary.

## Context

To protect ATCR's enterprise IP (audit logging, ticketing integrations, protected paths) from being trivially bypassed in a public repository, the "license key" model has been abandoned in favor of a dual-repo structure. 

The public repository (`samestrin/atcr`) remains 100% open-source (MIT). The proprietary features will live exclusively in a private repository (`samestrin/atcr-enterprise`). This guarantees that non-paying users physically do not possess the compiled code for enterprise features, making piracy impossible and eliminating the need for offline cryptography checks.

## Proposed Solution

1. **Private Repository Initialization:**
   - Create the private `samestrin/atcr-enterprise` repository.
   - Initialize `go.mod` with `module github.com/samestrin/atcr-enterprise`.
2. **Module Importing:**
   - Import the public core engine: `require github.com/samestrin/atcr vX.Y.Z`.
3. **Main Wrapper:**
   - Create `cmd/atcr-enterprise/main.go`. This file will serve as the exact entrypoint for the enterprise binary. It will import the public CLI bootstrapping logic, but inject the proprietary implementations for the `EnterpriseHooks` (defined in Epic 39.0).
4. **CI/CD Pipeline Syncing:**
   - Set up a GitHub Action to automatically test `atcr-enterprise` against the `main` branch of the public `atcr` repo to catch API breakages before they are released.

## Acceptance Criteria

- [ ] The `atcr-enterprise` repo successfully builds a binary that runs the full ATCR review engine.
- [ ] The enterprise binary's `go.mod` correctly imports the public `samestrin/atcr` module.
- [ ] A GitHub Action validates that the private repo continues to compile against public updates.
