# Feature Request: Public TD Resolve Skill

- **Estimated time**: TBD
- **Tasks/Components**: TBD
- **Execution**: init-plan

## Context

The private `.planning/technical-debt/` pipeline gives internal sprint/epic workflows a mature loop: findings accumulate into `README.md`, get sharded (Epic 12.1), and get autonomously resolved by `claude-prompts:/resolve-td`. The standalone public skill (`skill/SKILL.md`, Epic 20.0) has no equivalent — every `atcr review` is a one-off. Findings land in `.atcr/reviews/<id>/reconciled/` and nothing accumulates into a durable backlog or gets acted on afterward. A standalone user (no `.planning/`, no sprint workflow) has no path from "atcr found 12 issues" to "atcr fixed them" without doing it by hand.

Epic 18.3 (TD Metadata Pipeline Enhancement) is teaching the shared `internal/reconcile` engine to attach narrative context (`justification`) and a source back-reference to each finding in `reconciled/findings.json`. This epic is the first consumer of that enrichment on the public side.

## Problem Statement

Standalone/public `atcr` users have no way to persist findings across review runs into a durable technical-debt backlog, and no autonomous way to resolve them — both capabilities the private workflow already has via `.planning/technical-debt/` + `/resolve-td`. Without this, "public atcr" is a review-only tool while "private atcr" is a review-and-fix tool, which undercuts the goal of releasing atcr as a complete, standalone product.

## Proposed Solution

1. **Design a local, `.atcr/`-scoped technical-debt store.** Format TBD during discovery — informed by the private pipeline's format history (Epic 12.1's flat-table-to-sharded-YAML migration, and the lessons recorded there about merge-conflict-prone single files vs. per-run shards), **and by `internal/scorecard`'s existing append-only, cross-run, month-rotated JSONL store (`internal/scorecard/store.go`, `internal/scorecard/paths.go`)** — the closest in-repo analog, differing only in that it is scoped globally (`os.UserConfigDir()/atcr/scorecard`) rather than per-repo (`.atcr/`). Must work with zero `.planning/` dependency.
2. **New public skill package** (per the packaging convention recorded in Epic 20.0's 2026-07-03 addendum: a separate single-capability skill, e.g. `skill/atcr-resolve/SKILL.md`, not a sub-command dispatcher bolted onto `skill/SKILL.md`) that reads reconciled findings — including the `justification`/back-reference fields from Epic 18.3 when present — and resolves them, adapting `/resolve-td`'s RED→GREEN→ADVERSARIAL→REFACTOR cycle to a repo-agnostic, `.planning/`-free context.
3. **Persistence across runs:** a way for `atcr reconcile` (or a new command) to append reconciled findings into the local TD store rather than only ever writing to a single review's own directory.
4. **Document install/usage** alongside the existing `skill/SKILL.md` guide (`docs/skill-usage.md`), following the same file-copy install convention.
5. **Extract shared skill boilerplate into `skill/CONVENTIONS.md`,** per Epic 20.0's 2026-07-03 addendum, which assigns this extraction to this epic's execution: move the binary-on-`PATH` check, `.atcr/` path-safety rules, and prerequisites currently embedded in `skill/SKILL.md`'s Prerequisites section into a shared reference doc, and point both `skill/SKILL.md` and the new `skill/atcr-resolve/SKILL.md` at it.

## Acceptance Criteria

- [ ] AC1: A local TD store format is defined and documented for `.atcr/`-scoped (no `.planning/`) usage.
- [ ] AC2: Reconciled findings can be persisted into the local TD store across multiple review runs, not just held in one review's own directory.
- [ ] AC3: A new public skill autonomously resolves TD items from the local store, consuming the `justification`/back-reference fields from Epic 18.3 when present and degrading gracefully when absent.
- [ ] AC4: The new skill is documented and installable via the same file-copy convention as `skill/SKILL.md` (`docs/skill-usage.md`).
- [ ] AC5: Shared skill boilerplate (binary-on-`PATH` check, `.atcr/` path-safety rules, prerequisites) is extracted from `skill/SKILL.md` into `skill/CONVENTIONS.md`, and both `skill/SKILL.md` and `skill/atcr-resolve/SKILL.md` reference it instead of duplicating it.

## Out of Scope

- Any change to the private `.planning/technical-debt/` pipeline or `claude-prompts` skills.
- Binary packaging/release automation (Epic 21.0).
- Multi-repo/team-wide TD aggregation (Team Edition concerns — candidate for Epic 19.0 territory if it comes up there).
- A dispatcher-style single public skill routing multiple sub-commands — explicitly rejected in favor of separate single-capability skills (Epic 20.0's 2026-07-03 addendum).

## Dependencies / Related
