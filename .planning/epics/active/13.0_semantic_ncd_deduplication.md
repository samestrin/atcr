# Feature Request: Semantic NCD Deduplication

- **Estimated time**: 4 days
- **Tasks**: 3
- **Components**: 1
- **Execution**: execute-epic

## Context

The reconciler deduplication pipeline currently relies on standard string-overlap algorithms to determine if two models have reported the same finding. 

## Problem Statement

Standard text-overlap algorithms (Jaccard, TF-IDF) fail when two models identify the same issue using entirely different vocabularies. This leaves duplicate findings in the final review, annoying the end-user.

## Proposed Solution

- Calculate Normalized Compression Distance: `NCD(x, y) = (C(x+y) - min(C(x), C(y))) / max(C(x), C(y))`.
- Use Go's fast `zstd` compression.
- Concatenating and compressing two review findings provides a deterministic, language-agnostic semantic similarity score.

## Acceptance Criteria

- [ ] AC1: NCD algorithm implemented using `zstd`.
- [ ] AC2: Replaces the existing `internal/reconcile` text-similarity threshold logic.
- [ ] AC3: Benchmark suite proves NCD correctly identifies duplicate findings that previous lexical algorithms missed due to vocabulary differences.

## Clarifications

### Recorded 2026-06-27 (rubber duck — /execute-epic)

**Key Decisions:**
- NCD replaces Jaccard as the dedup algorithm AND drives the merge gate (Option Y), but the decision boundary stays deterministic: the NCD distance is integer-quantized before the threshold comparison (same two-tier "integer-authoritative, float-advisory" pattern the current cross-multiply already uses). The raw NCD float is advisory/display only.
- NCD is a distance (0 = identical, 1 = unrelated), so the merge/gray thresholds are inverted from the current Jaccard `inter*10 >= union*7` / `>= union*4` and must be calibrated against a labeled corpus.
- The real algorithm lives in the extracted in-tree module `github.com/samestrin/atcr/reconcile` (`reconcile/dedupe.go`), wired in via the `replace ... => ./reconcile` directive in the root `go.mod`. `internal/reconcile` only re-exports it.
- "Native Go zstd" = pure-Go (no CGO) for cross-compilation → `github.com/klauspost/compress/zstd`. stdlib has no zstd; gzip/flate are not substitutes (different ratios change every NCD score).
- NCD is computed over concatenated Problem + Fix + Evidence so the compressed input is long enough (50-150 words) to give reliable compression-distance signal.

**Scope Boundaries:**
- IN scope: NCD primitive + zstd dep; replace classify()/tokenize() with NCD-driven integer-quantized inverted thresholds; threshold calibration + labeled corpus (25-40 pairs); dedupe/grayPair test update; golden-corpus regeneration (`UPDATE_GOLDEN=1`); benchmark suite (AC3); doc-comment updates. 6 tasks (header's "Tasks: 3" superseded), 1 component.
- OUT of scope: changing AmbiguousHash / union-find adjudication / sidecar schema / internal/debate / downstream AmbiguousCluster consumers (they depend on cluster structure, not the similarity algorithm); LLM embeddings or vector DBs.

**Technical Approach:**
- Reuse a single zstd encoder + preallocated byte buffers across pair comparisons to hit `<1ms`/pair and `<50ms`/1000-comparison targets and avoid GC pressure.
- Hit the 1,000-comparison throughput figure by looping over the labeled pairs in the benchmark, not by authoring 1,000 distinct pairs.
- Keep NCD out of any serialized struct hashed by AmbiguousHash (or, if surfaced as a display field, a single controlled `UPDATE_GOLDEN=1` run suffices and does not recur across zstd upgrades).
