# Epic Plan 34.0: Enterprise Immutable Audit Logging

- **Tasks/Components**: TBD
- **Execution**: init-plan

## Objective

Implement strict, SIEM-compatible JSON Lines (JSONL) audit logging for every `atcr` execution. This satisfies "Layer 5: Governance and Audit" enterprise requirements, allowing organizations in highly regulated environments (e.g., FedRAMP, HIPAA) to adopt `atcr` by guaranteeing a tamper-evident record of all AI interactions.

## Context

Enterprises evaluating the "Build vs. Buy" path for AI code review often reject "Buy" solutions because commercial vendors cannot provide the level of granular audit logging required for compliance. When engineers run AI models against proprietary codebases, compliance teams need to know exactly what was sent, to which model, by whom, and what the model returned.

Currently, `atcr` outputs developer-friendly artifacts (`review.md`, `td-stream.txt`) but lacks a structured, machine-readable audit trail. By generating a definitive JSONL audit log, `atcr` provides the necessary observability for enterprise SIEM ingestion (e.g., Splunk, Datadog), making it enterprise-ready out of the box.

## Proposed Solution

1. **Audit Logger Package:** Create `internal/audit` to handle structured JSON logging.
2. **Execution Context Capture:** For every model invocation, log a JSON payload containing:
   - Timestamp and Execution ID
   - User/Runner Identity (from environment/git context)
   - Code Context Sent (file paths, block hashes, or exact snippets)
   - Model Provider and Model Name
   - Exact Prompt Provided
   - Exact Model Response
   - Token Usage and Estimated Cost
3. **Log Sinks:** Configure the audit logger to write to a dedicated, append-only file (e.g., `.atcr/audit.log`) or stream to a configurable endpoint.
4. **Log Masking:** Ensure sensitive environment variables or explicit secrets (though already filtered by `atcrignore`) are categorically excluded from the audit log payload.

## Acceptance Criteria

- [ ] Every model invocation generates a discrete, structured JSONL entry.
- [ ] Log entries include execution identity, model metadata, full prompt/response payloads, and token telemetry.
- [ ] The audit log can be successfully ingested and parsed by standard SIEM tools (e.g., Splunk).
- [ ] Sensitive secrets are structurally omitted or masked from the log payloads.
