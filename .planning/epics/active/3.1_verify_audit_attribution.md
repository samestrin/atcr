  1. Epic 3.0 (adversarial verification) **merged** — this work edits the verify pipeline 3.0 introduces.
  2. TD-009 (pipeline.go:159 parallelism) **landed** — both touch the `verifyFinding` / `pipeline.go` code paths.
- **Do not start before both gates are met.** See Sequencing.

## Current State

The `VerificationResult` record written to `verification.json` has three structured audit fields that are never populated correctly:

- `TrippedBudgets []string` — always empty; `invokeSkeptic` folds tripped budgets into free-text `Notes`, never into the slice
- `Model string` — hard-coded to `skeptics[0].Config.Model` regardless of which skeptic produced the winning verdict in a multi-vote run
- `DurationMs int` / `TrippedBudgets` — lost entirely on the skip-already-verified rebuild path (pipeline.go:199), which re-synthesizes a compact record from the on-disk Verification block

Additionally, `Model` is empty on the `no_eligible_skeptic` and `tool_harness_unavailable` early-return paths in `verifyFinding`.

## Pain Points

### Degraded audit trail
A human reviewing `verification.json` cannot determine:
- Which skeptic's model was actually responsible for the winning verdict (multi-vote runs)
- Whether a budget was tripped during verification (tripped budgets buried in prose Notes)
- What the original DurationMs/Model was for a finding that was skipped on a re-run

### Misattribution risk
In a 3-skeptic Thorough run where skeptic[0] is refuted and skeptics[1]+[2] confirm, the recorded `Model` is skeptic[0]'s — the loser's, not the majority winners'.

### Extends TD-011
This item is explicitly tracked as "[intent: deferred per sprint-plan TD-011]" in the TD README. The Phase 5 Clarifications confirmed deferral pending TD-009 sequencing.

## Proposed Solution

Thread the tripped-budget slice and the winning skeptic's model up from `invokeSkeptic` into the `VerificationResult` struct, and preserve `Model`/`DurationMs`/`TrippedBudgets` on the skip-already-verified rebuild path.

### Key changes

1. **`internal/verify/invoke.go`** — change `invokeSkeptic` return to surface `trippedBudgets []string` alongside the verdict (currently only folded into Notes)
2. **`internal/verify/pipeline.go` / `verifyFinding`** — after aggregating votes via `aggregateVerdicts`, identify the winning skeptic by matching the aggregated verdict against per-skeptic results (mirroring `joinSkeptics`); set `base.Model` to the winner's model, join models when multiple voters agree
3. **`internal/verify/pipeline.go` / skip-already-verified path** — carry `Model`/`DurationMs`/`TrippedBudgets` forward from the on-disk `VerificationResult` rather than synthesizing a lossy record
4. **`internal/verify/pipeline.go` / early-return paths** — set `base.Model` to the skeptic's model on the `no_eligible_skeptic` (empty) and `tool_harness_unavailable` (empty) paths
