# Sprint Plan 13.4: Deterministic PageRank Signal

**Estimated Duration**: 4 days
**Status**: Draft

## Objective

Evolve the confidence scoring system from a flat "vote counting" metric to a dynamic, graph-based authority scoring model using Deterministic PageRank.

## Problem Statement

"3 models agree" currently produces a flat confidence score (e.g., HIGH), regardless of whether those 3 models frequently hallucinate or are highly authoritative for that specific codebase or language.

## Proposed Solution

-   Build an agreement graph for the current review run. Nodes are models; edges represent "Model X agreed with Model Y on a finding".
-   Run deterministic PageRank over this graph to calculate the relative "authority" of each model within the context of the current run.
-   If models agree frequently on high-confidence issues, their isolated findings gain inherited weight.

## Acceptance Criteria

- [ ] AC1: Agreement graph generation implemented during the reconcile resolution phase.
- [ ] AC2: Deterministic PageRank calculates authority scores per model.
- [ ] AC3: Confidence scoring logic updated to factor in PageRank weights, resulting in more nuanced and accurate `Confidence` labels in the final `Result`.
