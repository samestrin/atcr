## Current State

Two divergent `writeRadarSection` implementations render the disagreements radar:

1. `internal/reconcile/disagree.go:413` — `writeRadarSection`, used by the reconciled report.md path
2. `internal/report/disagree.go:49` — `writeRadarSection` (delegates to `writeRadarItems` at :58), used by the archival/display path

Both render the same `DisagreementsFile` structure but with different escaping helpers (`esc`/`codeSpan` in reconcile vs. `escTrunc`/`joinReviewers`/`reviewerOrUnknown` in report). The duplication risks divergent escaping and behavioral drift.

## Pain Points

### Divergent Escaping
- reconcile uses `esc()` + `codeSpan()` from emit.go
- report uses `escTrunc()` + `joinOrNone()`/`joinReviewers()`
- Same data structure, two rendering pipelines — a bug fix in one won't reach the other

### Import Constraint
- `internal/report` already imports `internal/reconcile`
- `internal/reconcile` → `internal/report` would be circular
- The shared renderer must live in `reconcile` (or a new shared package), not in `report`

### Intentional Differences
- archival `esc` vs display `escTrunc` (verbatim vs truncated)
- `joinOrNone` vs `joinReviewers`/`reviewerOrUnknown` (different reviewer formatting)
- These must be captured as explicit parameters, not silently erased

## Proposed Solution

Extract a shared `WriteRadarSection` function in `internal/reconcile` parameterized by:
- A truncate/verbatim text renderer (function parameter)
- A heading prefix format (function parameter)
- Reviewer join strategy (function parameter)

`internal/report` calls the shared renderer with its display-oriented parameters. `internal/reconcile` continues to use it with its archival parameters.

## Scope

### In Scope
- Shared renderer in `internal/reconcile` with parameterized escaping/formatting
- `internal/report/disagree.go` calls the shared renderer
- Preserve intentional differences via parameters
- Update tests for both call sites

### Out of Scope
- Changes to the DisagreementsFile data model
- Changes to call sites in `cmd/atcr/report.go` or `internal/mcp/handlers.go` (separate TD rows)
