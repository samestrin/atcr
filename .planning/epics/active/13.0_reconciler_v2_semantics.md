# Epic Plan 13.0: Reconciler v2 Semantic Algorithms

**Estimated Durations**: 4 weeks
**Tasks/Components**: 10/4
**Status**: Draft

## Objective

Evolve the extracted `github.com/atcr/reconcile` module to "v2" by replacing legacy string-matching and greedy clustering with highly advanced, LLM-free deterministic algorithms. This maximizes accuracy and structural deduplication without adding Python/GPU dependencies or sacrificing execution speed.

## Context

Epic 8.0 extracts the existing v1 reconciler into a standalone module. While the v1 reconciler is robust (using basic text similarity and deduplication), it fails in edge cases where models use vastly different phrasing for the exact same underlying structural issue. To maintain our architectural moat, the next iteration of the reconciler must "understand" code semantics deterministically.

## Problem Statement

1.  **Lexical limitations**: Standard text-overlap algorithms (like Jaccard/TF-IDF) miss findings where models identify the same issue using entirely different vocabularies.
2.  **Line number drift**: Models frequently hallucinate or skew line numbers by ±3 lines. Current proximity grouping is brittle.
3.  **Sub-optimal grouping**: Greedy clustering algorithms often group the wrong things or leave stragglers when matching multiple reviewers' outputs.
4.  **Flat vote counting**: "3 models agree" is currently a flat confidence score, regardless of whether those 3 models frequently hallucinate or are highly authoritative for that specific file type.

## Proposed Solution

Introduce four advanced algorithmic phases to the reconciler pipeline:

1.  **Semantic Phase: Normalized Compression Distance (NCD)**
    -   Use Go's highly optimized `zstd` compression to calculate `NCD(x, y) = (C(x+y) - min(C(x), C(y))) / max(C(x), C(y))`.
    -   Concatenating and compressing two review findings provides a zero-LLM semantic similarity score that is language-agnostic and incredibly fast.
2.  **Structural Phase: AST Subgraph Isomorphism**
    -   Map the raw line numbers flagged by models to their smallest covering Abstract Syntax Tree (AST) node.
    -   Compare Merkle hashes of the AST nodes. If two models flag different lines but map to the same logical `for` loop or `if` statement block, group them deterministically.
3.  **Resolution Phase: Bipartite Graph Matching**
    -   Implement the Kuhn-Munkres (Hungarian) algorithm to match Model A's findings to Model B's findings optimally, replacing the legacy greedy-match heuristic.
    -   Implement DBSCAN (Density-Based Clustering) to isolate uncorroborated, single-model "hallucinations" cleanly into the `ambiguous.json` sidecar.
4.  **Signal Extraction: Deterministic PageRank**
    -   Build an agreement graph for the current review run. If models agree frequently on high-confidence issues, their isolated findings gain inherited weight.

## Acceptance Criteria

- [ ] AC1: NCD algorithm implemented and bench-marked to process 1,000 finding comparisons in <50ms using `zstd`.
- [ ] AC2: AST parsing (using `go/ast` for Go, or a fast Tree-sitter binding for polyglot) implemented for line-to-node resolution.
- [ ] AC3: Reconciler deduplication pipeline rewritten to use Bipartite Matching; tests prove it resolves known greedy-clustering failure edge cases.
- [ ] AC4: DBSCAN successfully isolates known model hallucinations into `ambiguous.json` without requiring arbitrary `k` thresholds.
- [ ] AC5: The `atcr reconcile` benchmark suite shows a >20% reduction in duplicate "gray zone" findings without increasing false-merges.

## Success Criteria

### Functional
- [ ] Zero reliance on external Python ML/NLP packages or local embeddings.
- [ ] End-to-end reconcile execution time remains under 200ms per file.

### Business
- [ ] Solidifies ATCR's reconciler as the absolute state-of-the-art reference implementation.
- [ ] Reduces human-review fatigue by drastically cutting down redundant findings that survived v1 deduplication.

## Technical Constraints

-   **Zero LLM in the loop**: The reconciler MUST remain 100% deterministic and offline.
-   **Pure Go**: Avoid CGO dependencies if possible to maintain cross-compilation simplicity, though a static Tree-sitter C binding is acceptable if strictly necessary for polyglot ASTs.

## Dependencies

-   **Epic 8.0**: MUST be fully completed. The `github.com/atcr/reconcile` module must be stable, extracted, and versioned before these algorithms are introduced as a v2.0 feature branch.

## Risks

| Risk | Likelihood | Impact | Mitigation |
|------|------------|--------|------------|
| NCD compression overhead is too slow | Low | High | Benchmark `zstd` early; pre-allocate byte buffers to avoid GC pressure. |
| AST parsing is too slow or complex for polyglot | Medium | High | Start with just Go AST support as a proof-of-concept before adding Tree-sitter. |
| Bipartite matching is overkill | Medium | Low | O(V³) is negligible when V is typically <20 findings per file. |
