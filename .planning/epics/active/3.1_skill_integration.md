# Epic Plan 3.1: Skill Integration

**Estimated Durations**: 2-3 weeks (one sprint)

## Objective

Replace the `llm-support` CLI backend in the existing `execute-code-review` skill's multi-agent mode with `atcr`, so the skill's multi-agent fan-out uses atcr's heterogeneous reviewer panel and deterministic reconciler instead of the legacy `review_direct` / `multi_review` commands.

## Context

The existing `execute-code-review` skill (in `~/Documents/GitHub/claude-prompts/.claude/skills/execute-code-review/`) has two modes:

1. **Default mode** (phases 1-8): Sprint verification pipeline — work-item review, test execution, coverage gates, adversarial risk analysis. This is sprint-coupled and stays as-is.
2. **Multi-agent mode** (`--multi-agent`): Thin orchestration over `llm-support review_direct` or `llm-support multi_review` CLI calls. The CLI handles range resolution, fan-out, and merge.

ATCR replaces the reviewer panel — heterogeneous personas, deterministic reconcile with clustering/dedupe/confidence, payload modes, and a clean findings format. Epic 1.8 adds `--output-dir` so atcr's output can be directed to the skill's expected location.

The end state: the skill's multi-agent mode calls `atcr review --output-dir ${CR_OUT_DIR}` + `atcr reconcile`, reads atcr's actual output format (not aliases), and the downstream `/reconcile-code-review` skill accepts atcr's output structure.

## Problem Statement

1. The existing multi-agent mode shells out to `llm-support review_direct` / `multi_review`, which are single-backend (openclaw or direct API) with no cross-reviewer agreement scoring.
2. ATCR provides the differentiated capability (heterogeneous panel, deterministic reconcile, confidence scoring) but the existing skill cannot consume it.
3. The two systems produce different output structures: the skill expects `td-stream.txt` + `multi-review-summary.json`; atcr writes `sources/pool/findings.txt` + `sources/pool/summary.json` + `reconciled/` artifacts.

## Proposed Solution

### Epic 1.8 (ATCR side — prerequisite)

`--output-dir` flag on `atcr review` so the review tree is created at the skill's expected location. No output format changes — atcr's output is clean and consistent.

### This epic (skill side)

1. **Modify `multi-agent.md`** to invoke `atcr review --output-dir ${CR_OUT_DIR}` (background, polled) followed by `atcr reconcile`, replacing the `llm-support review_direct` / `multi_review` invocation.
2. **Update output verification** (Step 8) to check for atcr's actual output files: `sources/pool/findings.txt`, `sources/pool/summary.json`, `reconciled/findings.txt`, `reconciled/summary.json`.
3. **Update summary reporting** (Step 9) to read atcr's `reconciled/summary.json` and extract the relevant fields (source count, finding count, partial flag). The skill's display format is unchanged; only the data source changes.
4. **Update `/reconcile-code-review`** (in `claude-prompts`) to accept atcr's output structure as a source. The reconcile skill currently auto-discovers sources by looking for `td-stream.txt` files; it gains atcr's `reconciled/findings.txt` as an additional source pattern.
5. **Findings format compatibility**: atcr's per-source `findings.txt` is 8-column pipe-delimited (same as the skill's `td-stream.txt`). The reconciled `findings.txt` is 9-column (adds REVIEWERS + CONFIDENCE). The reconcile skill must handle both formats.

## Scope

This epic is one plan (3.2) because the work is tightly coupled — modifying the skill without updating reconcile leaves a broken pipeline, and the validation is inseparable from the changes.

## Success Criteria

- [ ] The existing skill's multi-agent mode invokes `atcr review --output-dir ${CR_OUT_DIR}` + `atcr reconcile` instead of `llm-support review_direct` / `multi_review`.
- [ ] Output verification checks atcr's actual output files and reports success.
- [ ] Summary reporting reads atcr's `reconciled/summary.json` and displays finding count, source count, partial flag.
- [ ] `/reconcile-code-review` accepts atcr's `reconciled/findings.txt` as a source (in addition to the legacy `td-stream.txt`).
- [ ] A sprint review run through the modified skill produces correct output that `/reconcile-code-review` can consume.
- [ ] The `llm-support` backend is decommissioned from the skill (no dead code paths).

## Out of Scope

- Modifying the skill's default (host Claude) review mode — it stays as-is.
- Changing atcr's output format for compatibility — the skill adapts to atcr's format.
- Sprint verification logic (phases 1-5, 7-8) — unchanged.

## Dependencies

- Epic 1.8 (`--output-dir` support) — must land first.
- ATCR binary built and installed from the `atcr` repo.
- The `claude-prompts` repo (where the skill and reconcile skill live).

## Risks

| Risk | Likelihood | Impact | Mitigation |
|------|------------|--------|------------|
| Atcr's summary.json fields don't map cleanly to the skill's expected fields | Medium | Medium | Read atcr's actual schema; the skill's display is simple (counts + partial flag) and easy to adapt |
| Reconcile skill's source discovery breaks with atcr's directory structure | Medium | High | Add atcr's `reconciled/findings.txt` as an explicit source pattern; keep `td-stream.txt` for legacy compatibility |
| Findings format column mismatch (8-col vs 9-col) | Low | High | Atcr's per-source is 8-col (same as legacy); reconciled is 9-col. The reconcile skill must handle both — document the format difference |
| Parallel-run validation reveals behavioral differences | Medium | Medium | Run both backends on the same sprint, compare output; differences in findings count/severity are expected (different reconcilers) but the pipeline must not break |
