# Epic Plan 28.0: Telemetry Expansion & Cloud Sync

- **Estimated time**: TBD
- **Tasks/Components**: TBD
- **Execution**: init-plan

## Objective

Introduce opt-in, lightweight telemetry to track actual product adoption, gather anonymous data for a crowdsourced Persona Leaderboard, and provide a `--sync-cloud` payload mechanism for the upcoming enterprise SaaS dashboard.

## Context

Epic 10.0 successfully built a model-evaluation leaderboard based on explicit, opt-in benchmark runs. However, the product is currently blind to daily usage metrics (MAU, run success rates, language distribution) and cannot measure which community personas (from Epic 19.6) perform best in the real world. Furthermore, the commercial roadmap requires a mechanism to push local run data to a cloud dashboard to demonstrate ROI to engineering managers.

## Proposed Solution

1. **Usage Telemetry**: Implement an anonymous background ping on `atcr` commands (e.g., `{"event": "review_run", "lang": "go", "lines": 450, "status": "success"}`). This helps measure the top-of-funnel conversion without capturing source code.
2. **Persona Ranking Data**: Expand the local scorecard export to explicitly hash and include the Persona ID used during the run. This allows the backend to aggregate which prompts are empirically the most effective.
3. **Cloud Sync Flag**: Add a `--sync-cloud` flag to push the complete, anonymized local scorecard data (including time/credits saved metrics) to a user's `atcr.dev/dashboard` account.
4. **Opt-Out Mechanism**: Provide a strict, well-documented `ATCR_TELEMETRY=0` environment variable and `atcr config set telemetry false` command to completely disable the usage ping for privacy-conscious teams.

## Acceptance Criteria

- [ ] A background telemetry client exists and safely fails open (does not block CLI execution if the network drops).
- [ ] The CLI securely sends anonymous usage events on run completion.
- [ ] `ATCR_TELEMETRY=0` strictly disables all background telemetry.
- [ ] The exported scorecard schema includes Persona ID hashing for the Persona Leaderboard.
- [ ] The `--sync-cloud` flag successfully authenticates and pushes run history to a designated endpoint.
