# Epic 14.3: Diff Chunking & Context Management

- **Estimated time**: 2 weeks
- **Tasks/Components**: 3/2
- **Execution**: execute-epic

## Context

LLMs suffer from attention degradation when fed excessively large inputs. If `atcr` attempts to review a massive git range (e.g., thousands of lines) in a single prompt, the LLM often loses its place and enters a "runaway generation" loop, outputting hundreds of hallucinated TD items.

## Problem Statement

We currently lack guardrails on the size of the context we provide to individual reviewer personas. Massive diffs directly correlate with hallucination rates.

## Proposed Solution

1. **Max Diff Size Limits**: Implement hard limits on the size of the diff sent to any single LLM prompt. If a diff exceeds the token limit or line limit, it must be chunked.
2. **Per-File Fan-Out**: Instead of sending an entire PR to a single persona prompt, fan out the review at the file level. For example, if a PR touches 10 files, create 10 separate review jobs per persona. This keeps the context window extremely focused and minimizes hallucinations.

## Acceptance Criteria

- [ ] AC1: Implement a configuration or hardcoded limit for maximum lines/tokens per prompt.
- [ ] AC2: Implement per-file (or logical chunk) splitting in the fan-out dispatcher before sending code to the reviewer personas.

## Technical Constraints

- Fanning out per file will drastically increase the number of API calls made.
- We must handle rate limiting and concurrency carefully.

## Non-Functional Requirements

### Scalability
- The system must comfortably handle fanning out 100+ files to 3 personas without hitting standard API rate limits (requires concurrency controls and backoff).

## Risks

| Risk | Likelihood | Impact | Mitigation |
|------|------------|--------|------------|
| Loss of cross-file context | High | High | The Reconciler will still have access to the global context. If a bug requires seeing two files, the file-level reviewer might miss it. We may need to provide "import" context alongside the chunk. |
| API Cost | High | Medium | Warn the user if a PR will generate >50 API calls before execution. |
