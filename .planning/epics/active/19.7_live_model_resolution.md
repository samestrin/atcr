# Epic Plan 19.7: Live Model Resolution — Aliases, Lockfile & Drift Detection

- **Estimated time**: TBD
- **Tasks/Components**: TBD
- **Execution**: init-plan

## Objective

Layer live, auto-updating model resolution over the persona `model` bindings that Epic 19.6 shipped, so a user rides each vendor family's capability curve as models evolve **without hand-editing slugs** — while keeping reviews **reproducible by default**. A persona binds to a model *family/channel*; a resolved **lock** records the concrete slug that actually runs; and the model only changes on an explicit `atcr personas upgrade`, never silently mid-review. This is a resolution layer *on top of* 19.6's static slugs, not a replacement — 19.6's pinned `model` becomes the initial lock.

## Context

**19.6 is the substrate, verified correct.** 19.6 stored fully-qualified slugs (`anthony → anthropic/claude-opus-4.8`) and deliberately made the **vendor token** in that field the load-bearing grouping key for discovery (Phase 5 clarification). 19.7 promotes that same token to a *resolution* key. 19.6's fetch-and-pin already ships an `atcr personas upgrade` command this epic advances.

**Catalog spike (OpenRouter `/api/v1/models`, 344 models, run 2026-07-08):**

- **`-latest` aliases exist for 7 of 10 personas** — Anthropic (`~anthropic/claude-opus-latest`, `claude-sonnet-latest`), OpenAI (`~openai/gpt-latest`, `gpt-mini-latest`, `gpt-chat-latest`), Google (`~google/gemini-pro-latest`, `gemini-flash-latest`), Moonshot (`~moonshotai/kimi-latest`). For these the **provider owns resolution** → near-zero maintenance.
- **No `-latest` alias for DeepSeek, Qwen, GLM** (delia/quinn/glenna). These need a `created`-timestamp "newest-in-vendor-prefix" resolver or an explicit pin.
- **Schema:** each model carries `id`, `canonical_slug`, `created` (timestamp), and `expiration_date` (deprecation signal — only 8 models today), but **no explicit GA/preview/stable flag** → the "stable channel" needs a heuristic (exclude preview/beta/exp tokens + honor `expiration_date`) or light curation.
- **All 10 of 19.6's pinned slugs still resolve verbatim** — nothing is broken today; this epic is additive.
- **Open question to close first:** most aliases carry a `~` prefix (OpenRouter alias/meta marker). Their routability in an actual completion call needs one authenticated request to confirm before the design bets on aliases.

**Reproducibility posture (19.6 C3).** 19.6 locked *fetch-and-pin freezes the version; upgrade is explicit*. This epic honors that: resolution happens at `upgrade` time (and hits the endpoint there), never on the review hot path. Silent runtime auto-latest is explicitly rejected — it would make the code-review gate non-deterministic (a clean PR sprouting findings with an unchanged diff) and add a hot-path network dependency.

## Problem Statement

1. **Manual slug churn.** 19.6's frozen slugs must be hand-edited every vendor release, ×10 personas, kept in lockstep with `index.json` (the AC7 gate forces YAML↔index match). Unsustainable as models change monthly.
2. **No drift or deprecation detection.** A sunset model (nonzero `expiration_date`) silently 404s a persona at review time; nothing warns.
3. **Deferred 19.6 HIGH — roster/index disjoint.** `init`/`quickstart` fetch-and-pin the roster `builtins.Names()` (9 model-agnostic built-ins), but the shipped `personas/community/index.json` publishes only the 10 model-indexed library personas — a disjoint set. Against the real index, online `init`/`quickstart` pin **zero** community personas and emit 9 `not found in community index — skipping` warnings. This epic rebuilds init/quickstart model enablement and closes the gap.

## Proposed Solution

1. **Spike (first task):** one authenticated completion against a `~…-latest` alias to confirm routability; enumerate preview/beta/exp token patterns to define the `@stable` heuristic. The result decides whether alias-bind covers 7 personas cleanly or needs a fallback.
2. **Two-layer binding.** A persona declares a *logical* binding (family + channel, e.g. `anthropic/claude-opus@stable` or a token); atcr records the *resolved* slug as a lock. Reviews run the lock. 19.6's `claude-opus-4.8` seeds the first lock — zero migration.
3. **Hybrid resolver.** (a) Alias-bind the 7 alias-covered personas (provider resolves). (b) `created`-timestamp newest-in-vendor-prefix resolver for DeepSeek/Qwen/GLM. (c) Explicit-slug pin as an always-available escape hatch.
4. **Channels.** `@stable` (default — newest GA member; excludes preview tokens + expiring models); `@latest` (includes preview); an explicit slug never floats.
5. **Reproducible by default + explicit upgrade.** `atcr personas upgrade` re-resolves and advances the lock, printing exactly what changed (`otto: opus-4.8 → 5.0`). The models endpoint is touched only here.
6. **`atcr models check [--json]`.** A deterministic drift report: newer family member available, model expiring (`expiration_date`), or slug missing — with exit codes and machine-readable output. This is both a user command and the seam Epic 19.8's mechanical agent wraps.
7. **Major-bump re-validation gate.** A minor advance (4.8→4.9) auto-locks. A major jump (4.x→5.x) requires the persona's fixture to still pass **and** flags "prompt tuned for the prior major — verify" before the lock advances. (The fixture only proves the template renders; the tuning judgment stays human — see 19.8.)
8. **Rebuild init/quickstart model enablement** to close the deferred 19.6 HIGH — either publish the built-in lenses into the community channel behind a model-agnostic gate carve-out (keeps C2: built-ins stay model-agnostic), or align the fetch-and-pin roster with what the index actually publishes and drop the misleading skip-warning. The exact reconciliation is an `init-plan`/`design-sprint` decision; the requirement is: online `init`/`quickstart` deliver a working, non-noisy persona set.
9. **CI stays zero-live-network.** A checked-in catalog snapshot fixture backs all resolver tests (same discipline as 19.6's `httptest` pattern), with a refresh command.

## Acceptance Criteria

- [ ] AC1: `~…-latest` alias routability is confirmed (or refuted) by a real completion call, and the finding is recorded; the `@stable` preview-exclusion heuristic is defined against the live schema.
- [ ] AC2: A persona binds to a family/channel and resolves to a concrete slug recorded in a lock; reviews run the locked slug deterministically (no endpoint call on the review path).
- [ ] AC3: The 7 alias-covered personas bind to provider `-latest` aliases; DeepSeek/Qwen/GLM resolve via a `created`-based newest-in-vendor resolver or an explicit pin; an explicit-slug pin never floats.
- [ ] AC4: `atcr personas upgrade` advances the lock and reports the before→after slug per persona; no silent runtime model change ever occurs.
- [ ] AC5: `atcr models check [--json]` reports drift (newer member), deprecation (`expiration_date`), and missing-slug conditions with machine-readable output and exit codes.
- [ ] AC6: A major-version model jump gates on the persona fixture re-passing and surfaces a re-tune flag; a minor jump auto-locks.
- [ ] AC7: Online `init`/`quickstart` deliver a working community persona set with no misleading "not found in community index" warnings (19.6 roster/index HIGH closed), backward-compatible with existing on-disk personas.
- [ ] AC8: `go test ./...` passes with all resolver/catalog tests backed by a checked-in snapshot (zero live network in CI); docs document the family/channel/lock model and the reproducible-vs-upgrade behavior.

## Components Touched

- `internal/personas` (resolver, catalog client, `models check`, upgrade/lock)
- `internal/registry` (persona binding schema, lock read on the resolve path)
- `cmd/atcr` (`models` command, `personas upgrade`, `init`/`quickstart` rework)
- `personas/community` (binding format, index reconciliation)
- `docs/`

## Out of Scope

- **The LLM prompt-drafting / monitoring agent** — Epic 19.8 (this epic ships the deterministic `atcr models check` primitives the agent consumes; it does not ship the agent).
- **Community prompt submissions / feedback collection** — a later epic (under discussion).
- **Silent runtime auto-latest** — rejected on purpose; reproducible-lock is the default, `@latest`/floating is opt-in.
- **A second hosted surface / registry site** — resolution reads the provider endpoint read-only; no new hosting (keeps 19.6's "one repo, one maintenance surface").
- **OAuth / new auth model** — unchanged from 19.6.

## Dependencies

- **Epic 19.6 (Community Registry Hub)** — provides the persona units, `index.json`, fetch-and-pin, `upgrade`, and the C3 guardrails this epic builds on.
- **OpenRouter `/api/v1/models`** — the live catalog; alias routability is confirmed by AC1 before the design commits to aliases.
