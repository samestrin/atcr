# Epic Plan 42.0: Review Intelligence Analytics Dashboard

- **Estimated time**: 4 weeks
- **Tasks/Components**: 4 tasks / 3 components
- **Execution**: init-plan

## Objective

Build a telemetry aggregation and analytics dashboard that provides engineering leaders with macro-level insights into codebase health, common vulnerabilities, and team performance across hundreds of repositories.

## Context

Engineering leaders (Directors/VPs) are the buyers for the Enterprise Tier ($15k/yr), but they don't look at individual PR comments. They need high-level visibility. A Review Intelligence dashboard aggregates the findings from ATCR runs to answer questions like, "Are we introducing more concurrency bugs this month?" or "Which repositories have the highest density of unresolved technical debt?"

## Proposed Solution

1. **Data Ingestion:**
   - Define a separate `EnterpriseAnalyticsPayload` structure containing `Severity`, `Category`, and `PersonaIDHash` to keep enterprise data isolated from the standard usage ping's strict privacy schema.
   - Expand the telemetry hooks (Epic 28.0 / Epic 35.0) to securely forward this metadata payload to the enterprise backend when authenticated.
2. **Analytics Dashboard:**
   - Build a web-based dashboard (React/Next.js) visualizing trend lines, heatmaps of vulnerabilities by repository, and persona performance.
3. **Automated Reporting:**
   - Generate automated weekly/monthly "State of Code Quality" PDF/Email reports for engineering leadership.

## Tasks

- **T1: Enterprise Telemetry Payload Definition**
  Define `EnterpriseAnalyticsPayload` containing `Severity`, `Category`, and `PersonaIDHash` in a new file `internal/telemetry/enterprise.go`. Keep this separate from `Event` to preserve the standard usage-ping privacy model.
- **T2: Enterprise Client Extensions**
  Extend `telemetry.Client` to support custom ingestion endpoints and authorization headers (reusing `ATCR_API_KEY`) for enterprise payload transmission.
- **T3: Integrate Enterprise Forwarding Hooks**
  Wire the new enterprise client to dispatch the metadata payload on review/reconcile completion when authenticated.
- **T4: Gating and Regression Verification**
  Write tests in `cmd/atcr/telemetry_test.go` and `internal/telemetry/client_test.go` ensuring `TestClient_Send_PayloadHasExactlyFourAllowlistedKeys` passes and that enterprise payloads are never dispatched without explicit opt-in and authentication.

## Acceptance Criteria

- [ ] ATCR enterprise clients can opt-in to forward finding metadata to the central analytics store.
- [ ] A web dashboard visualizes finding trends over time grouped by severity and category.
- [ ] The dashboard can drill down into specific repositories or teams.
