# Epic Plan 50.0: Enterprise Agent Runtime Controls (AgentControl)

- **Estimated time**: 3 weeks
- **Tasks/Components**: 4 tasks / 3 components
- **Execution**: init-plan

## Objective

Build a "kill-switch" and granular runtime governance layer for ATCR agents, allowing Enterprise Engineering VPs to dynamically toggle, throttle, or configure specific agent personas (e.g., Fixer, Skeptic, QA) globally or per-repository without modifying code.

## Context

Enterprise companies are wary of autonomous AI agents running unchecked in their CI/CD pipelines. Inspired by enterprise feature-flag patterns (e.g., LaunchDarkly's AgentControl), ATCR needs an embedded governance layer. If an agent starts hallucinating, producing degraded code, or running up API costs, engineering leadership needs a way to instantly disable or constrain that agent across the organization. 

## Proposed Solution

1. **Global & Repo-level Governance**: Add runtime configuration flags (via `.atcr/config.yaml` or a remote Enterprise Dashboard API) to enable/disable specific personas.
2. **Directory-Level Scoping**: Allow administrators to enforce rules like "The Auto-Fixer agent is banned from modifying `auth/` or `payments/` directories."
3. **FinOps & Throttle Limits**: Implement budget caps per agent or per repository to prevent runaway LLM API costs during infinite loops or massive automated PR spams.

## Acceptance Criteria

- [ ] Administrators can disable specific agent personas (e.g., Skeptic, Fixer) at runtime without altering source code.
- [ ] Administrators can enforce directory-level blocklists for the Fixer agent.
- [ ] FinOps budget caps can be applied to stop agent execution if LLM token spend exceeds a daily threshold.

## Tasks

- **T1: Configuration Schema**: Add `AgentControl` settings to `ProjectConfig` for toggles and blocklists.
- **T2: Execution Gate**: Modify `internal/personas/runner.go` to check the `AgentControl` state before invoking any LLM API.
- **T3: Token Tracking (FinOps)**: Add an accumulator for token usage that triggers a circuit breaker if limits are exceeded.
- **T4: CLI Output**: Update the CLI to visibly report when an agent was skipped due to Runtime Controls or FinOps limits.
