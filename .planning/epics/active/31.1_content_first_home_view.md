# Feature Request: AXI Content-First Home View

- **Estimated time**: 1-2 days
- **Tasks**: 4
- **Components**: `cmd/atcr`

## Context

Epic 31.0 (AXI Compliance) implements `--axi` token-dense output, pagination, exit-code reconciliation, and stderr isolation for `atcr review`/`atcr report`, but explicitly scoped out axi.md's Principle 8 ("Content First"). This epic delivers Principle 8 as a small, self-contained follow-on — sequenced as `31.1` (not `31.0`'s own scope, and not folded into the later `48.0` epic) specifically so it lands **before** the `33.0`→`33.1`→`33.2` public-launch/atcr.dev sequence, since it is cheap, low-risk, and directly strengthens an eventual AXI-catalog listing.

## Problem Statement

Bare `atcr` (no arguments, no flags) currently calls `cmd.Help()` (`cmd/atcr/main.go`), showing static usage text. axi.md's Principle 8 argues a no-args invocation should show live, actionable data instead: an orchestrator or agent often runs a bare command just to sanity-check state before deciding what to do next, and every such invocation currently costs a wasted discovery turn returning nothing but help text it likely already has cached.

## Proposed Solution

Add a home-view branch to the root command's `RunE` for the true no-args case only. Cobra's `-h`/`--help` flags and `--version` already short-circuit before `RunE` runs, and every subcommand's own `Args`/`RunE` is untouched — this only changes what happens when `atcr` is invoked with literally nothing.

The home view shows:
1. The executable path, with the home-directory prefix rendered as `~` (per axi.md's example).
2. atcr's existing one-line description (already defined on the root command: "Agent Team Code Review — a review panel, not a reviewer").
3. Live state: the most recent review's id/status, read via the same `fanout.ReadReviewStatus` reader `cmd/atcr/status.go` already uses against `.atcr/latest` — or an explicit "no reviews yet, run `atcr review`" message when no review exists yet (never an empty/error output).

The view respects `--axi` (bare `atcr --axi`) by reusing Epic 31.0's `FormatAXI`/pagination/context-propagation plumbing — this is not a second, parallel output mode.

## Tasks

- **T1**: Add a no-args home-view branch to the root command's `RunE` in `cmd/atcr/main.go`, guarded so it only fires when zero args and zero flags are passed; `-h`/`--help`/`--version` and every subcommand remain on their existing paths.
- **T2**: Implement the home-view renderer: `~`-relativized executable path (`os.Executable()` + home-dir substitution), the existing one-line description, and live state via `fanout.ReadReviewStatus` against `.atcr/latest`, with an explicit no-reviews-yet state when absent.
- **T3**: Wire `--axi` (bare) through Epic 31.0's format-dispatch/context-propagation mechanism so the home view's `--axi` output is one more consumer of that plumbing, not a new mode switch.
- **T4**: Add a golden/snapshot test pinning the non-`--axi` home-view output, plus regression tests confirming `--help`/`-h`/`--version` and all existing subcommands are unaffected.

## Acceptance Criteria

- [ ] AC1: `atcr` with no arguments and no flags shows a live home view (not help text) — `~`-relativized executable path, one-line description, and current review state.
- [ ] AC2: `atcr --help`, `atcr -h`, `atcr --version`, and every existing subcommand's behavior is unchanged.
- [ ] AC3: When no review has ever been run (`.atcr/latest` missing), the home view shows an explicit "no reviews yet" state — never an error, never empty/zero-byte output.
- [ ] AC4: `atcr --axi` (bare, no subcommand) renders the home-view data through Epic 31.0's `FormatAXI`/pagination context plumbing, not a parallel ad hoc renderer.
- [ ] AC5: A golden/snapshot test pins the non-`--axi` home-view output; full existing test suite, `go vet`, and lint pass unmodified.

## Out of Scope

- axi.md Principle 9 (Contextual Disclosure / `help[]` next-step suggestions) — scoped separately as Epic `48.0`, deliberately deferred until the `--axi` output/command surface is more locked down.
- Session-hook / plugin auto-install (Principle 7's primary recommendation) — atcr already satisfies Principle 7's secondary recommendation via the existing installable `skill/SKILL.md` Agent Skill; no new work scoped here.
- Any change to `atcr status`'s existing behavior when explicitly invoked with an id/path argument.

## Dependencies

- Epic 31.0 (AXI Compliance) — reuses its `--axi` format-dispatch and context-propagation plumbing. Should be sequenced after 31.0's foundational format-dispatch work lands (at minimum Phase 1/2 of 31.0's sprint design), and before the `33.0`/`33.1`/`33.2` public-launch sequence.
