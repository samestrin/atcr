# Feature Request: Human Names for Built-in Reviewer Personas

- **Estimated time**: 1-2 days
- **Tasks/Components**: 4/2
- **Execution**: execute-epic

## Context

Most of the built-in reviewer personas in `atcr` have natural human names (like `bruce`, `greta`, `kai`, `mira`, `dax`, `otto`). However, we have three legacy "role-based" names remaining: `sentinel`, `tracer`, and `idiomatic`. 

## Problem Statement

Having a mix of human names and role-based names is inconsistent and breaks the "panel of human-like reviewers" metaphor. The remaining stragglers should be renamed to human names that personify their specific activities.

## Proposed Solution

 Rename the three straggler personas to natural human names and generalize their scope where appropriate:
1.  **`sentinel`** (security/injection) $\rightarrow$ **`sasha`** (starts with S like security/sentinel).
2.  **`tracer`** (performance/resources) $\rightarrow$ **`penny`** (personifying a "penny pincher" of cpu/memory/allocations).
3.  **`idiomatic`** (currently Go idioms/conventions) $\rightarrow$ **`ingrid`** (or **`ian`**). We will generalize this persona's prompt from Go-only to cover language-specific idioms, style guidelines, and conventions dynamically based on the target files (Go, Python, TypeScript, etc.).

This rename and generalization requires updating:
*   The `.md` files under `personas/` (`sentinel.md` $\rightarrow$ `sasha.md`, etc.).
*   The system prompt in `idiomatic.md` (renamed to `ingrid.md` or `ian.md`) to instruct the LLM to apply conventions matching the active target language rather than hardcoded Go rules.
*   The `names` slice in `personas/personas.go`.
*   The embedded patch fixture files in `personas/testdata/` (e.g. `sentinel_fixture.patch` $\rightarrow$ `sasha_fixture.patch`).
*   Any references in tests (`personas_test.go`, etc.), configurations, and user guides.

## Acceptance Criteria

- [ ] AC1: All built-in personas have lowercase human names (no references to `sentinel`, `tracer`, or `idiomatic` remain in the active registry).
- [ ] AC2: Embedded templates, fixtures, and Go source code are updated to reflect the new names.
- [ ] AC3: The `idiomatic` persona is generalized to dynamically analyze idiomatic patterns for Go, Python, and other supported target languages.
- [ ] AC4: `go test ./...` passes successfully with all updated names.

## Technical Constraints

- Must maintain the underlying role/behaviors for the security and performance personas (only the names and fixtures change).
- The generalized idioms persona (`ingrid` / `ian`) must dynamically evaluate code patterns based on the file extension/language.
