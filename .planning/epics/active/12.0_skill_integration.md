# Epic Plan 12.0: Skill Integration (atcr Backend Swap)

**Estimated Durations**: 2-3 weeks (one sprint)

> Consolidated plan. This merges the former skill-integration strategy and
> backend-swap execution plans, which described the same tightly-coupled work.
> **Off the atcr-core critical path** — this modifies the external `claude-prompts`
> repo, not atcr itself, so it can run in parallel with any core epic once its
> prerequisite (Epic 1.8) is satisfied. **Epic 1.8 is complete** (`--output-dir`
> ships in `cmd/atcr/review.go`), so this plan is unblocked.

## Objective

Replace the `llm-support` CLI backend in the existing `execute-code-review` skill's
multi-agent mode with `atcr`, so the skill's multi-agent fan-out uses atcr's
heterogeneous reviewer panel and deterministic reconciler instead of the legacy
`review_direct` / `multi_review` commands. Update `/reconcile-code-review` to accept
atcr's output structure as a source.

## Context

The existing `execute-code-review` skill (in
`~/Documents/GitHub/claude-prompts/.claude/skills/execute-code-review/`) has two modes:

1. **Default mode** (phases 1-8): Sprint verification pipeline — work-item review, test
   execution, coverage gates, adversarial risk analysis. Sprint-coupled; stays as-is.
2. **Multi-agent mode** (`--multi-agent`): Thin orchestration over `llm-support
   review_direct` / `multi_review`. The CLI handles range resolution, fan-out, and merge.

ATCR replaces the reviewer panel — heterogeneous personas, deterministic reconcile with
clustering/dedupe/confidence, payload modes, and a clean findings format. The end state:
the skill's multi-agent mode calls `atcr review --output-dir ${CR_OUT_DIR}` + `atcr
reconcile`, reads atcr's actual output format, and `/reconcile-code-review` accepts
atcr's output structure.

The skill lives in
`~/Documents/GitHub/claude-prompts/.claude/skills/execute-code-review/multi-agent.md`.
The reconcile skill lives in
`~/Documents/GitHub/claude-prompts/.claude/skills/reconcile-code-review/`.

## Problem Statement

1. `multi-agent.md` Phase 2 Step 6 invokes `llm-support review_direct` / `multi_review`
   — single-backend (openclaw or direct API), no cross-reviewer agreement scoring.
2. ATCR provides the differentiated capability (heterogeneous panel, deterministic
   reconcile, confidence scoring) but the existing skill cannot consume it.
3. The two systems produce different output structures: the skill expects `td-stream.txt`
   + `multi-review-summary.json`; atcr writes `sources/pool/findings.txt` +
   `sources/pool/summary.json` + `reconciled/` artifacts.
4. `/reconcile-code-review` auto-discovers sources by `td-stream.txt` filename; it must
   also discover atcr's `reconciled/findings.txt`.

## Proposed Solution

### Changes to `multi-agent.md`

**Phase 2, Step 6** — replace the `llm-support` invocation:

Before:
```
llm-support review_direct \
  --config "${CONFIG_PATH}" \
  --repo "${REPO_ROOT}" \
  ${RANGE_FLAGS} \
  ${SCOPE_FLAG} \
  --output-dir "${CR_OUT_DIR}"
```

After:
```
atcr review \
  --output-dir "${CR_OUT_DIR}" \
  ${RANGE_FLAGS}
```

Followed by:
```
atcr reconcile "${CR_OUT_DIR}"
```

Notes:
- `--config` and `--repo` are not needed — atcr reads config from `.atcr/config.yaml` and
  `~/.config/atcr/registry.yaml`, and the repo is the CWD.
- `${SCOPE_FLAG}` (`--sprint-plan`) is dropped — atcr has no sprint-plan concept.
  Reviewers fall back to diff-touched-line scope.
- `${RANGE_FLAGS}` maps directly: `--base`, `--head`, `--merge-commit` are the same.

**Phase 3, Step 8** — update output verification:

Before:
```
paths: ["${CR_OUT_DIR}td-stream.txt", "${CR_OUT_DIR}multi-review-summary.json"]
```

After:
```
paths: [
  "${CR_OUT_DIR}sources/pool/findings.txt",
  "${CR_OUT_DIR}sources/pool/summary.json",
  "${CR_OUT_DIR}reconciled/findings.txt",
  "${CR_OUT_DIR}reconciled/summary.json"
]
```

**Phase 3, Step 9** — update summary reporting: read `reconciled/summary.json` for
`sources` (count), `total_findings`, `partial`. Display format unchanged; only the data
source changes.

**Phase 1, Step 3** — add an `atcr --version` pre-flight check (backend selection). If the
binary is not found, display an actionable error and HARD STOP.

### Changes to `/reconcile-code-review`

1. Add `reconciled/findings.txt` as an additional source pattern (alongside
   `td-stream.txt`).
2. When a source is `reconciled/findings.txt`, read it as 9-column format
   (SEVERITY|FILE:LINE|PROBLEM|FIX|CATEGORY|EST_MINUTES|EVIDENCE|REVIEWERS|CONFIDENCE)
   instead of 8-column.
3. Sources under `.atcr/reviews/` are NOT auto-discovered — only sources under the
   sprint's `code-review/` directory (where `--output-dir` points).

## Out of Scope

- Modifying the skill's default (host Claude) review mode — it stays as-is.
- Changing atcr's output format for compatibility — the skill adapts to atcr's format.
- Sprint verification logic (phases 1-5, 7-8) — unchanged.
- Updating the `llm-support` CLI itself (it continues to exist for other consumers).

## Dependencies

- Epic 1.8 (`--output-dir` support) — **complete**.
- ATCR binary built and installed from the `atcr` repo.
- Access to `~/Documents/GitHub/claude-prompts/` to modify the skill and reconcile skill.

## Risks

| Risk | Likelihood | Impact | Mitigation |
|------|------------|--------|------------|
| Atcr's summary.json fields don't map cleanly to the skill's expected fields | Medium | Medium | Read atcr's actual schema; the skill's display needs only source count, finding count, partial flag — easy to map |
| Reconcile skill's 9-column parser is not implemented | Medium | High | The 9-column format is documented in atcr's findings-format spec; implement the parser as part of this plan |
| Reconcile skill's source discovery breaks with atcr's directory structure | Medium | High | Add atcr's `reconciled/findings.txt` as an explicit source pattern; keep `td-stream.txt` for legacy compatibility |
| Findings format column mismatch (8-col vs 9-col) | Low | High | Atcr's per-source is 8-col (same as legacy); reconciled is 9-col. The reconcile skill must handle both — document the format difference |
| Parallel-run validation reveals finding count discrepancies | Medium | Low | Expected — atcr's reconciler uses different dedupe/confidence logic. The pipeline must not break, but finding counts may differ. Document this in the skill |
| `--sprint-plan` scope flag is lost | Low | Medium | Atcr has no sprint-plan concept. Reviewers fall back to diff-touched-line scope. Document this behavioral change in the skill |
