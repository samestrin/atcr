# Epic Plan 36.0: Enterprise Protected Path Overrides (Code Owners Enforcement)

- **Estimated time**: TBD
- **Tasks/Components**: TBD
- **Execution**: init-plan

## Objective

Implement a `.atcr-protected` (or support native `.codeowners`) configuration that identifies high-risk paths within a repository. Changes touching these paths will trigger an enforced "human gate" mode where AI autonomous approval is bypassed and explicit human sign-off is mandated.

## Context

While AI scales the checking of code, it does not own the judgment or accountability. Enterprise organizations have highly sensitive files (e.g., CI/CD workflows, authentication logic, infra-as-code, and billing controllers) where an AI hallucination or a bypass of a "low-risk" assessment could be catastrophic. ATCR needs a mechanism to acknowledge that AI is a sensor, but humans are the accountable owners.

## Proposed Solution

1. **Configuration File:**
   - Introduce `.atcr-protected` schema allowing teams to define glob patterns (e.g., `.github/workflows/**`, `src/auth/**`).
   - Alternatively, parse standard `CODEOWNERS` files.
2. **Review Execution Logic:**
   - When ATCR analyzes the git diff, it checks for intersections with protected paths.
   - If an intersection is found, the Reconciler explicitly flags the PR as `REQUIRES_HUMAN_APPROVAL`, bypassing any automatic merge behavior, regardless of how clean the AI panel's review is.
3. **Integration with CI/CD:**
   - Output specific exit codes or GitHub Check Run statuses (e.g., `action_required`) to halt the merge queue until a human intervenes.

## Acceptance Criteria

- [ ] ATCR correctly parses `.atcr-protected` glob patterns.
- [ ] Diffs containing protected files result in a mandatory human review block.
- [ ] AI review panel still runs on protected files to provide context to the human reviewer, but cannot autonomously approve the PR.

## Components Touched

- `internal/config/` (Schema for protected paths)
- `internal/git/` (Diff analysis against protected patterns)
- `internal/reconciler/` (Emitting the human-gate requirement)
