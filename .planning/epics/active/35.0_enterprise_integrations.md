# Epic Plan 35.0: Enterprise Ticketing and PR Integrations

- **Estimated time**: TBD
- **Tasks/Components**: TBD
- **Execution**: init-plan

## Objective

Build native integrations with enterprise ticketing systems (Jira, Linear) and CI/CD version control platforms (GitHub, GitLab). This fulfills "Layer 4: Enterprise Integrations", allowing `atcr` findings to seamlessly enter existing enterprise workflows without manual copy-pasting.

## Context

A major blocker for adopting custom or CLI-based AI code review tools is the lack of integration with developer workflows. While `atcr` deduplicates findings and scores their confidence brilliantly (Epic 13.0 / Reconciler), developers still have to manually translate the `td-stream-merged.txt` output into actionable Jira tickets or PR comments.

To compete with "Buy" platforms that cost millions of dollars, `atcr` needs lightweight but powerful hooks to push its reconciled findings directly into the systems where developers already live.

## Proposed Solution

1. **Ticketing Integration:**
   - Create `internal/integrations/tickets`.
   - Add functionality to parse `td-stream-merged.txt` and automatically create Linear or Jira issues for findings above a certain confidence threshold.
   - Include links back to the git commit and file/line number in the ticket body.
2. **PR Commenting Hooks:**
   - Create `internal/integrations/vcs`.
   - Build GitHub Action and GitLab CI templates/wrappers that use the GitHub/GitLab APIs to post findings as inline PR/MR comments.
3. **Configuration Driven:**
   - Allow teams to map `atcr` personas or severity levels to specific Jira labels, Linear projects, or PR comment behaviors via `.atcr/config.yaml`.

## Acceptance Criteria

- [ ] `atcr` can authenticate and create tickets in Jira and Linear based on config settings.
- [ ] High-confidence technical debt findings are automatically pushed to the configured ticketing board.
- [ ] Templates exist for GitHub Actions and GitLab CI to post findings as inline comments on Pull Requests.
- [ ] Integration behavior (thresholds, labels, target projects) is fully configurable.

## Components Touched

- `internal/integrations/` (New packages for tickets and vcs)
- `.atcr/config.yaml` (New schema for integration targets)
- `.github/workflows/` (Example templates)
