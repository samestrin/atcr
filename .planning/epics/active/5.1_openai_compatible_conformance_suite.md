# Epic 5.1: OpenAI-Compatible Conformance Suite

**Status:** Planned  
**Source:** TD chat.go:1 (deferred from /resolve-td 2026-06-14)  
**Scope:** OpenAI + Ollama (both /v1/chat/completions surface); Anthropic native API excluded

## Problem

`internal/llmclient` deliberately absorbs real wire divergence (string-encoded vs
raw-object tool_call `arguments`, lenient finish_reason) but is tested only against
synthetic fixtures. A regression in how a specific provider shapes its `tool_calls`
response (`ToolCallArguments`, `chatToolResponse` decode, `finish_reason` handling)
would not be caught.

## Goal

Record real wire fixtures from OpenAI and Ollama endpoints and assert the llmclient
parser produces identical engine-facing results across both providers. This gives the
same protection an official SDK would provide, without adopting one.

## Acceptance Criteria

1. Fixture capture script records a live `tool_calls` response from OpenAI `/v1/chat/completions`
2. Fixture capture script records a live `tool_calls` response from Ollama `/v1/chat/completions`
3. Both fixtures stored under `internal/llmclient/testdata/conformance/`
4. Table-driven conformance test asserts `ToolCallArguments`, `chatToolResponse` decode,
   and `finish_reason` handling produce identical engine-facing `ChatResponse` from each fixture
5. Test runs offline (fixtures are static files, no live calls at test time)

## Out of Scope

- Anthropic native API (uses a different wire format; separate concern) — now tracked as **Epic 5.3 (Anthropic-Native Conformance)** so the gap is visible rather than dropped.
- litellm, vLLM, Together — lower priority, can be added incrementally
- Load/performance testing

## Notes

- Fixture capture requires live provider access (OpenAI API key, running Ollama instance)
- Fixtures must be redacted of any sensitive content before committing
- Est: 2-3 days (fixture capture + harness + test authoring)
