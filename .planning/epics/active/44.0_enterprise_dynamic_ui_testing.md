# Epic Plan 44.0: Enterprise Dynamic UI Testing (Browser MCPs)

- **Estimated time**: TBD
- **Tasks/Components**: 5 tasks / 4 components
- **Execution**: init-plan

## Objective

Evolve ATCR from a purely static code reviewer into a dynamic QA agent for Enterprise customers by integrating browser automation MCPs (Playwright, Chrome DevTools). This enables ATCR to visually verify frontend changes, run E2E flows, and catch UI regressions that static analysis cannot detect.

## Context

Currently, ATCR evaluates code syntax, logic, and ASTs statically. When reviewing frontend pull requests (React, Vue, HTML/CSS), it cannot determine if a CSS change breaks a layout or if a complex DOM interaction actually works in the browser. Large enterprise engineering organizations struggle with visual regressions and brittle E2E tests. By giving ATCR agents the ability to "click around" a live sandbox of the PR using MCPs, we provide a massive value-add: an AI reviewer that performs automated visual QA and dynamic testing before approving a PR.

## Proposed Solution

1. **Sandbox Provisioning**: Tie into Epic 32.0 (Sandbox Execution Environment) to build and serve the frontend PR code in an isolated live environment.
2. **Browser MCP Integration**: Integrate a Playwright or Chrome DevTools MCP server into the ATCR agent's toolset.
3. **Frontend QA Persona**: Create a dedicated persona whose system prompt specializes in writing and executing UI automation scripts against the sandbox URL.
4. **Visual Regression Reporting**: Enable the agent to capture screenshots, detect console errors, and attach visual evidence of UI breakages directly to PR comments.

## Acceptance Criteria

- [ ] ATCR can detect when a PR contains frontend/UI changes and automatically provision a sandbox environment.
- [ ] Agents are equipped with browser automation MCP tools (Playwright/Chrome DevTools).
- [ ] A Frontend QA persona successfully navigates the sandbox DOM, executes a basic E2E flow, and captures screenshots.
- [ ] Console errors and visual regressions are reported back to the user as actionable PR comments with attached evidence.
