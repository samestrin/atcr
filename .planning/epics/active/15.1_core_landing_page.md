# Sprint Plan 15.1: Core Landing Page & Design System

- **Estimated time**: 5 days
- **Tasks**: 6
- **Components**: 2
- **Execution**: execute-epic

## Context

The primary landing page of `atcr.dev` represents the first impression developers have of the tool. It needs to reflect the design system specified in [design-system.md](file:///Users/samestrin/Documents/GitHub/atcr.dev/the-artifact/design-system.md): dual-theme, paper-like colors, IBM Plex typography, and a mechanical, high-trust documentation aesthetic. The core visual asset is "The Artifact Viewer" itself.

## Problem Statement

We need to construct the foundation of the site:
1. Build a responsive layout supporting device sizes from mobile viewports (320px) up to widescreen.
2. Implement theme switching automatically using `prefers-color-scheme`.
3. Support the interactive file tab switching of the artifact view panel (`report.md` / `findings.txt` / `findings.json`) without layout shifts or text wrapping.
4. Establish copy-to-clipboard command pill logic.

## Proposed Solution

1. Port mockup styles to `styles.css` matching colors, typography rules, layout widths, and key components.
2. Port index HTML structure to `index.html` ensuring semantic element tags (`<nav>`, `<section>`, `<footer >`).
3. Port Javascript animations and events (tab toggles, clipboard copy, scroll reveals) to `script.js`.
4. Package assets (such as `favicon.svg` and brand logo marks).

## User Personas

### Primary: Software Engineer (Devin)
- Wants to understand what ATCR does in 5 seconds.
- Looks for command-line instructions (`go install`).
- Evaluates the readability of output file formats.

## User Journeys

### Visitor Landing & Command Copy
1. Visits `atcr.dev`.
2. Scans the hero message ("A review panel, not a reviewer") and watches the scroll reveal animations.
3. Clicks through the "Artifact Viewer" tabs (`report.md`, `findings.txt`, `findings.json`) to see differences.
4. Clicks the "copy" button on the command pill to copy the installation command.

## Acceptance Criteria

- [ ] AC1: Page structure utilizes semantic HTML5 tags and has a single H1.
- [ ] AC2: All colors (backgrounds, borders, text, severe warnings) strictly map to HSL/Hex specs in the design system.
- [ ] AC3: The Artifact Viewer tabs are keyboard-focusable and switch active panes instantly without layout reflows.
- [ ] AC4: Clicking the copy button copies `$ go install github.com/samestrin/atcr/cmd/atcr@latest` to the system clipboard and displays a visual check/state change.
- [ ] AC5: The design changes themes automatically between light/dark depending on the user's OS settings.

## Success Criteria

### Functional
- [ ] Initial load time of the landing page is under 1.0 second on standard connections.
- [ ] Javascript payload is under 5KB and requires zero dependencies.
- [ ] Page layout displays correctly on Safari, Chrome, and Firefox.

### Business
- [ ] Provide clear, accessible CTA buttons leading directly to Docs and Team Edition validation program signups.

## Technical Constraints

- Vanilla HTML, CSS, and JS only (no React, Svelte, or tailwind utility pollution).
- Responsive breakpoint configurations must prevent code wrap in the artifact view panels (using horizontal scrolls).

## Non-Functional Requirements

### Performance
- Total size of HTML/CSS/JS/SVG asset files must be less than 150KB.
- Lighthouse performance score must be > 95.

### Security
- All scripts are inline or local; no external script calls (e.g. CDNs) to protect visitor privacy.

## Out of Scope

- Model Leaderboard and Personas pages details (covered in Sprint 14.1 and 14.2).
- Integration of backend lead tracking databases.
