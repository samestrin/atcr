# Epic Plan 25.0: SARIF Output Integration

- **Estimated time**: TBD
- **Tasks/Components**: TBD
- **Execution**: init-plan

## Objective

Support exporting ATCR review findings in standard SARIF (Static Analysis Results Interchange Format) to natively integrate with GitHub Advanced Security, GitLab CI, and IDEs.

## Context

Currently, ATCR writes rich markdown files locally. For enterprise adoption and CI/CD integration, developers expect AI review findings to appear as inline annotations on their Pull Requests ("Files Changed" tab), just like SonarQube or CodeQL. SARIF is the industry standard for this.

## Proposed Solution

1. **SARIF Formatter**: Add a new render target in `internal/report` that maps ATCR's reconciled findings (file, line, message, severity) to the SARIF JSON schema.
2. **CLI Flag**: Add `--format=sarif` to `atcr review` and `atcr reconcile`.
3. **CI Example**: Document a GitHub Actions workflow step demonstrating how to pipe `atcr review --format=sarif` to GitHub's `codeql-action/upload-sarif`.

## Acceptance Criteria

- [ ] `atcr review --format=sarif` produces valid SARIF JSON.
- [ ] The SARIF output correctly maps ATCR severities (CRITICAL, HIGH, etc.) to SARIF levels.
- [ ] File paths and line numbers correctly anchor to the git diff.
- [ ] A documentation example exists showing the CI integration.

## Components Touched

- `internal/report`
- `cmd/atcr`
- `docs/`

## Out of Scope

- Direct API calls to GitHub/GitLab to post comments (SARIF ingestion handles this automatically without ATCR needing auth tokens).

## Dependencies

- None.
