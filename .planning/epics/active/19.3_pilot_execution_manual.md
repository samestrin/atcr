# Epic Plan 19.3: Pilot Execution (MANUAL — do not run a skill)

- **Estimated time**: 1-1.5 weeks (elapsed, mostly async)
- **Tasks/Components**: manual / none
- **Execution**: MANUAL — this is human process (recruitment, interviews, synthesis). No execute-epic / execute-sprint.

## Objective

Recruit 3-5 teams from the atcr.dev signups, run a 2-week pilot on the 19.0-19.2 features, capture exit interviews, and publish the go/no-go memo.

## Checklist

**Recruit (from Netlify signups)**
- [ ] Screen signups by team size + stack fit; select 3-5.
- [ ] Send pilot invite: "2-week pilot, no obligation, free during pilot only."

**Onboard (Week 0)**
- [ ] Onboarding call; install atcr + configure shared registry (`ATCR_REGISTRY_URL`).
- [ ] Confirm `atcr history` and `atcr audit-report` run in their repo. Keep setup < 30 min.

**Run (Weeks 1-2)**
- [ ] Week 1 async check-in: blockers + first impressions.
- [ ] Week 2 exit interview (30 min): force-rank the features; capture WTP.

**Exit interview script**
- Which feature would you miss most if removed?
- How do you solve this today?
- What would make you pay? At what monthly price is it a no-brainer / too expensive?
- Minimum feature set for a paid launch?

**Synthesize**
- [ ] Store transcripts in `.planning/product/validation/team-edition/`.
- [ ] Publish `memo.md`: counts, force-ranked priorities with evidence quotes, price summary, go/no-go + flip conditions.

## Success Criteria

- [ ] 3+ teams complete the full pilot + exit interview.
- [ ] ≥2 teams express willingness to pay for a feature subset.
- [ ] Memo published with a clear ranking and a documented go/no-go decision.
