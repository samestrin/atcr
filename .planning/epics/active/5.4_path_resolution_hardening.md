# Epic 5.4: Path-Resolution Hardening

**Status:** Draft
**Created:** 2026-06-20
**Priority:** Medium (correctness + reviewer UX; not blocking)
**Estimated time:** ~3 days
**Source:** Deferred AC7 from Epic 5.0 + adversarial findings from the 5.0 code review (2026-06-20)

## Current State

Epic 5.0 (File Path Validation) added existence-only validation: a finding whose
`File` does not resolve under the repo root is flagged `PathValid=false` /
`PathWarning="file not found"`, preserved (never discarded), and surfaced in the
reports. That shipped AC1ŌĆōAC6.

What it deliberately did **not** do ŌĆö and what this epic picks up ŌĆö is make the
existence check *smart*. Today the check is a single `os.Stat` with three known
gaps, all of which share one root cause: there is no listing of the repo's real
files to reason against, only a yes/no stat of the exact path a reviewer cited.

## Pain Points

### 1. A flagged path gives no path forward (deferred 5.0 AC7)
When a reviewer hallucinates `internal/auth/validator.go` (the real file is
`internal/auth/validate.go`), the user sees `ŌÜĀ’ĖÅ File not found` but must hunt for
the real file by hand. The finding is preserved but effectively un-actionable
until the path is corrected manually.

### 2. Case-typos pass silently on the primary dev platform
`os.Stat` is case-insensitive on the default macOS (APFS) and Windows
filesystems. `Parser.go` resolves as present for `parser.go` and is never flagged
ŌĆö on darwin, the maintainer's own platform and where most reviews run. Case
divergence is one of the most common hallucination forms, and the feature misses
it exactly where it matters most.

### 3. Existence check can be an oracle outside the repo
`os.Stat` follows symlinks. A repo symlink (`link -> /etc`) plus a
reviewer-controlled `File="link/passwd"` passes the lexical `..` containment guard
(the cleaned path has no `..`), then stats through the symlink and reports a file
outside the repo as present ŌĆö leaking one bit about the host filesystem into
findings.json.

## Proposed Solution

Build a **candidate file index once per reconcile run** and use it for all three
improvements. The index is the missing primitive: with the repo's real file list
in hand, case-exact matching, symlink-safety, and fuzzy suggestion all become
straightforward.

### Candidate source: `git ls-files`, not `filepath.Walk`
`atcr` reviews a git repo. `git ls-files` is fast, respects `.gitignore`, skips
`vendor/`, `node_modules/`, `.git/`, and lists only tracked files ŌĆö exactly the
set a reviewer could legitimately cite. Run it once, build:
- `basenameIndex: map[basename] -> []relpath`
- `dirIndex: map[dir] -> []basename`
- a case-folded variant of each for the case-exact check.

### Tiered matching (strategy matters more than the distance metric)
Hallucinated paths fail in predictable shapes, so resolve in tiers, cheapest and
most certain first:

- **Tier 1 ŌĆö exact basename, wrong directory (no fuzzy math).** Cited file
  missing but exactly one file with that basename exists elsewhere ŌåÆ near-certain
  match. Covers 5.0's "wrong directory" case deterministically, no threshold. If
  multiple matches, rank by path-segment overlap; if still tied, report ambiguity
  and emit no suggestion.
- **Tier 2 ŌĆö basename typo, same directory (the only tier needing edit distance).**
  Directory exists, file does not ŌåÆ compare against the actual files in that dir by
  Levenshtein, pick the single closest above threshold. Covers `validator.go` ŌåÆ
  `validate.go`.
- **Tier 3 ŌĆö case-only difference.** Folds in pain point #2 using the case-folded
  index.

### Suggest, never silently auto-correct
A wrong auto-correction is worse than an obviously-broken path: it points a real
finding at the wrong real file and looks authoritative. Default behavior is
suggestion only ŌĆö add `PathSuggestion string` and render:
> ŌÜĀ’ĖÅ File not found: `internal/auth/validator.go` (did you mean `internal/auth/validate.go`?)

A `--correct-paths` flag MAY later promote suggestions to actual `File` rewrites
for users who opt in, but the safe default stays suggest-only and the original
hallucinated path is always preserved in the artifacts.

### Symlink safety (pain point #3)
Switch the existence stat to `os.Lstat` (so a symlinked segment is not traversed)
or `filepath.EvalSymlinks` + re-check containment before stat.

## Scope

### In Scope
- `git ls-files`-backed candidate index built once per reconcile run.
- Tier 1 (exact-basename-elsewhere) and Tier 3 (case-exact) ŌĆö deterministic.
- Tier 2 (Levenshtein typo match) behind a tuned threshold.
- New `PathSuggestion` field threaded through `stream.Finding` ŌåÆ `reconcile.JSONFinding`
  (omitempty) ŌåÆ report.md (`internal/reconcile/emit.go`) and `atcr report`
  markdown/checklist/refuted views (`internal/report/render.go`).
- `os.Lstat`/EvalSymlinks symlink-safe existence check.
- Unit tests per tier + an end-to-end test (hallucinated path ŌåÆ suggestion in report).

### Out of Scope
- Auto-rewriting `finding.File` by default (suggestion only; `--correct-paths` is a
  possible future opt-in, not part of this epic).
- Line-number validation and file-content validation (out of scope in 5.0; still out).
- Cross-repo / non-tracked-file matching (untracked files are not review targets).

## Acceptance Criteria

- [ ] AC1: A candidate file index is built once per reconcile run from `git ls-files` (not per-finding, not via `filepath.Walk`).
- [ ] AC2: Tier 1 ŌĆö a missing path whose basename matches exactly one tracked file elsewhere yields that file as `PathSuggestion`, with no edit-distance threshold applied.
- [ ] AC3: Tier 3 ŌĆö a missing path differing from a real file only by case (e.g. `Parser.go` vs `parser.go`) is flagged invalid AND suggests the correctly-cased path, including on case-insensitive filesystems.
- [ ] AC4: Tier 2 ŌĆö a basename typo in an existing directory (e.g. `validator.go` ŌåÆ `validate.go`) yields the closest tracked file as `PathSuggestion` when above the configured similarity threshold; no suggestion when ambiguous or below threshold.
- [ ] AC5: The existence check does not traverse symlinks out of the repo root (verified by a test with `link -> /tmp` and `File="link/known"` staying flagged invalid with no suggestion).
- [ ] AC6: `PathSuggestion` flows through `JSONFinding` (omitempty ŌĆö byte-identical findings.json when absent) and renders as a "(did you mean ŌĆ”)" line in report.md and all `atcr report` views; the original hallucinated path is preserved.
- [ ] AC7: The default never rewrites `finding.File` ŌĆö suggestion only.
- [ ] AC8: Unit tests cover each tier, the ambiguous (multi-match) case, the below-threshold case, and the symlink case; plus one end-to-end test from a hallucinated finding to a suggestion in report.md.

## Success Criteria

- [ ] The 5.0 motivating examples (`validator.go`ŌåÆ`validate.go`, wrong-directory, case-typo) all produce a correct suggestion.
- [ ] Zero false auto-corrections (default is suggest-only).

## Risks

| Risk | Mitigation |
|------|------------|
| Fuzzy match suggests the wrong file | Suggest-only (never rewrite); show the original path; require a single clear winner, else no suggestion |
| Similarity threshold too strict/loose (see Open Questions) | Don't gate Tier 1 on distance at all; tune Tier 2 against the 5.0 examples |
| `git ls-files` unavailable (non-git dir) | Degrade gracefully to existence-only (current 5.0 behavior); no suggestion |
| Large repos make the index expensive | Build once per run; `git ls-files` is fast and gitignore-pruned |

## Open Questions

1. **Tier 2 similarity threshold.** The epic 5.0 plan recommended >90%, but the
   canonical example fails it: `validator.go` vs `validate.go` is edit distance 1
   on the stem ŌåÆ similarity Ōēł 89%. **Recommendation:** do not apply a distance
   threshold to Tier 1 (exact basename needs none); for Tier 2 use ~85% on the
   stem, validated against the 5.0 examples rather than a round number.
2. **Multiple equal candidates.** Report ambiguity (no suggestion) vs list top-N.
   **Recommendation:** no suggestion when ambiguous ŌĆö a wrong guess is worse than
   none; keep the plain `file not found` warning.
3. **`--correct-paths` opt-in rewrite.** In this epic or deferred? **Recommendation:**
   deferred; ship suggest-only first.
