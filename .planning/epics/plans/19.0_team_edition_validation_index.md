# Epic 19.0: Team Edition Validation

**Status:** Active
**Created:** 2026-07-04
**Priority:** Medium (revenue de-risking)
**Target:** Decide go/no-go on a paid Team Edition by validating demand on top of real, keepable features — not throwaway prototypes.

## Overview

Team Edition has the highest revenue ceiling but the longest build and least moat. This epic de-risks it by shipping three **real** features every atcr user benefits from, running a lightweight pilot with volunteer teams, and producing a go/no-go memo with a ranked feature list. Nothing built here is throwaway: finding-history and audit-trail are valuable to solo users too, and shared-registry is a thin extension of the existing `internal/registry` loader.

The pilot recruitment channel already exists — the signup form shipped in atcr.dev Epic 1.2 (`team-edition.html` + `pilot-form.js` → Netlify Forms), so this epic needs no website work.

## Deliverable

A one-page go/no-go memo in `.planning/product/validation/team-edition/memo.md` with: signup/completion counts, a force-ranked feature priority (registry / history / audit / dashboard), price-sensitivity summary, and a go/no-go recommendation with the conditions that would flip it.
