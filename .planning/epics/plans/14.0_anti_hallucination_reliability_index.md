# Epic 14.0: Anti-Hallucination & Reliability

**Status:** Planned
**Created:** 2026-06-22
**Priority:** High (Addresses severe false positives in TD generation)
**Target:** Decrease hallucinated technical debt items by 95% in multi-review scenarios.

## Overview

A core problem with fanning out code review across multiple LLM personas is "runaway generation," where a persona loses grounding and hallucinates technical debt (TD) items that do not exist in the source code. This results in overwhelming noise for the end user (e.g., hundreds of false-positive TD items).

This Epic introduces three primary defense layers to protect against hallucinated TD items.

## Strategy Plans

| ID | Title | Status | Description |
|---|---|---|---|
| 14.1 | [Verification & Grounding](./../active/14.1_verification_grounding.md) | Planned | Require exact quotes/lines, validate against git patch programmatically, and limit scope to changed lines. |
| 14.2 | [Reconciliation Layer](./../active/14.2_reconciliation_layer.md) | Planned | Implement the "Two-Persona Rule" for consensus and strict anti-hallucination prompting for the Host. |
| 14.3 | [Diff Chunking & Context](./../active/14.3_diff_chunking_context.md) | Planned | Fan-out by file and limit maximum diff size to protect attention windows. |
