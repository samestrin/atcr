+# Epic Plan 27.0: Local Model / Ollama Persona Integration
+
+- **Estimated time**: TBD
+- **Tasks/Components**: TBD
+- **Execution**: init-plan
+
+## Objective
+
+Ship a "Zero Data Egress" persona pack (using Ollama or llama.cpp) in the `personas/community/` registry, tuned for local hardware tiers, to resolve enterprise privacy objections during onboarding. The local-endpoint provider wiring itself already exists (`docs/providers.md`'s `local:` example) — the gap this epic closes is the tuned persona content and the walkthrough connecting it end-to-end.
+
+## Context
+
+While LiteLLM and proxy configurations are supported, privacy-conscious enterprise teams often refuse to send proprietary code to external APIs like OpenAI or Anthropic. `docs/providers.md` already documents a `local:` provider block (`base_url: http://localhost:11434/v1`) for wiring atcr at a local Ollama/llama.cpp endpoint — the missing piece is a first-class, officially supported persona tuned for local models (e.g., a highly optimized prompt for `Ornith 1.0` or `Qwen 3.6`), which makes the "privacy first" adoption path friction-free.
+
+## Proposed Solution
+
+1. **Targeted Persona Pack (The 3-Tier Strategy)**: Author a focused pack of personas — each a `<name>.yaml` + `<name>.md` + `community/testdata/<name>_fixture.patch` triple under `personas/community/` (with a matching `index.json` entry), following the established bare-human-first-name convention (no parenthetical qualifiers, no model name in the display name — model identity lives only in the `provider`/`model` fields) and passing `internal/personas/community_fixture_test.go`'s fixture gate. New files are auto-embedded via the existing `//go:embed` globs — no Go code changes needed. Using mnemonic human names that hint at the underlying model (e.g., Quinn for Qwen, Gerald for Gemma). Tuned for specific hardware tiers:
+   - **Tier 1: 32GB Dense** (e.g. `Gerald` for Gemma 4 or `Quinn` for Qwen 3.6): High reasoning, standard context (32k). Best for file-by-file review.
+   - **Tier 2: 32GB Long-Context** (e.g. `Orson` for Ornith 1.0): Optimized for massive context (256k on 32GB VRAM). Best for full-repo architectural reviews.
+   - **Tier 3: 64GB+ Heavyweight** (e.g. `Liam` for Llama 4 70B or `Dexter` for DeepSeek V4 Pro): For Mac Mini M4 Pro or dual-GPU setups. Fits 70B+ quants with massive context, rivaling frontier cloud APIs.
+2. **Documentation**: Extend `docs/personas-install.md`'s existing "Provider tiers beyond Synthetic" section with a Local / Privacy-First subsection explaining how to `brew install ollama`, pull a model, and run `atcr` entirely offline — including a reminder to `export OLLAMA_API_KEY=<placeholder>` (the provider config's `api_key_env` field is required even though a local endpoint needs no real key).
+
+## Acceptance Criteria
+
+- [ ] `personas/community/` includes a ready-to-use local persona for each of the 3 hardware tiers, each passing its `internal/personas/community_fixture_test.go` fixture.
+- [ ] A local model can successfully execute a review pipeline using one of these personas without external network calls (once the user's own `registry.yaml` defines a matching `local` provider, per `docs/providers.md`).
+- [ ] Documentation clearly outlines the zero-egress setup, including the `api_key_env` placeholder-value requirement.
+
+## Components Touched
+
+- `personas/community/` (persona content: yaml + md + fixture patch + index.json)
+- `docs/` (providers.md + personas-install.md)
+
+## Out of Scope
+
+- Managing or installing the local model server itself (user is expected to run Ollama/llama.cpp themselves).
+
+## Dependencies
+
+- Epic 19.6 (Community Registry Hub)

+## Role

+## Focus
+1. Broken precondition: a function now called before the state it requires is
+   established, or with an argument that violates a documented assumption
+2. Violated postcondition: a change that returns or stores a value the callers
+   were guaranteed never to see (a nil where non-nil was promised, an unclosed
+   handle, an unsorted result a binary search depends on)

+   but are now updated on divergent paths, leaving a reachable inconsistent state
+4. Lost idempotency or ordering guarantee: a retry, replay, or re-entry that the
+   surrounding contract assumed was safe and this change makes unsafe
+5. Unchecked invariant boundary: a monotonic counter, balance, or index that can
+   now underflow, overflow, or run past the bound the rest of the code trusts
