## Overview
`go-gitdiff` is a Go library designed for parsing and applying patches generated by Git tools (such as `git diff`, `git show`, and `git format-patch`) as well as unified diffs created by GNU `diff`. It is a standard choice in Go projects that need to manipulate or inspect patch files programmatically without relying on an external git binary.

## Quick Start
To install the package, run:
```bash
go get github.com/bluekeyes/go-gitdiff
```

### Basic Parsing Example
```go
package main

import (
	"os"
	"github.com/bluekeyes/go-gitdiff/gitdiff"
)

func main() {
	f, _ := os.Open("changes.patch")
	defer f.Close()

	files, preamble, err := gitdiff.Parse(f)
	if err != nil {
		panic(err)
	}
	// files is a slice of *gitdiff.File
}
```

### Basic Apply Example
```go
package main

import (
	"bytes"
	"os"
	"github.com/bluekeyes/go-gitdiff/gitdiff"

func main() {
```

## Key APIs
- **`gitdiff.Parse(r io.Reader)`**: Reads patch data and returns a slice of `*gitdiff.File` structures representing the modified files and hunks.
- **`gitdiff.Apply(w io.Writer, r io.Reader, f *gitdiff.File)`**: Applies the changes described in a parsed `*gitdiff.File` structure to the source content from `r`, writing the patched result to `w`.
- **`gitdiff.File`**: A structure representing a file modification, containing the old/new paths, metadata, and a slice of hunks (`gitdiff.Hunk`).
