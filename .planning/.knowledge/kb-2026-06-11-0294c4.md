# Why must the fan-out engine set Finding.Reviewer itself rath

## Decision

The fan-out engine must set `Finding.Reviewer` = agent name itself, ignoring any model-supplied 8th column. The findings stream contract specifies 7 columns from the model (SEVERITY|FILE:LINE|PROBLEM|FIX|CATEGORY|EST_MINUTES|EVIDENCE) with REVIEWER appended by the engine. The parser is lenient (pads short rows), so a misbehaving model that self-emits an 8th column could have that value land in `Finding.Reviewer` — forging its own attribution. This is TD-016 remediation: trust the engine's agent identity, not the model's output, for reviewer attribution.
