# Why must a two-tier auto-fix run use the same executor name 

## Decision

generateFixes (internal/verify/executor.go) prevents a second tier from re-attempting a finding a prior tier already fixed via hasFixAttribution(f.Evidence, ex.Name), scoped to the executor config's Name field, not the fix content. Distinct name values across tiers let tier 2 re-dispatch an already-fixed finding, breaking the "exactly one tier" partition invariant (AC 03-02). Not fixable by keying the guard off f.Fix != "" (the executor's job is to REFINE a reviewer-suggested Fix). Documented precondition: use the same name (default "executor") across all tiers, with non-decreasing ceilings across tier order.
