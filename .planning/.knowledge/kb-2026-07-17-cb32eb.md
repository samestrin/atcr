# How do you add a new independent opt-in privacy gate without

## Decision

Mirror the existing gate's shape (enabled(envEnabled bool, cfg *bool) bool pure/total, plus a Gate() bool I/O wrapper) but never call into or read the old gate's env var/config key/cache. Reuse shared hardened config-persistence helpers (atomic write, lock, symlink rejection) via delegation rather than copy-pasting them per setting. Both gates should fail-safe to disabled on malformed config, with an OR-shaped opt-in truth table. Concrete case: Sprint 30.0's quality_signal gate was built independent from the pre-existing telemetry gate — verified by adversarial review that neither gate calls the other or shares state, while both reuse the same atomic config-write helpers.
