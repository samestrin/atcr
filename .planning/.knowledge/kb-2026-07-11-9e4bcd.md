# Should install-script integration tests use a real go instal

## Decision

Happy-path install-script tests should run against a real `go install github.com/samestrin/atcr/cmd/atcr@latest` (real network egress, installing the actual published @latest module) gated behind `//go:build integration` — no new env-var gate needed. Stubbing `go` on a temp PATH is reserved only for failure-path scenarios: go missing (fatal, non-zero + stderr), go install failing (non-zero propagated), GOPATH/bin absent from PATH (non-fatal warning, exit 0). General pattern: don't stub a script's core external command for the happy path just for hermeticity — reserve stubbing for failure branches that are unsafe/impractical to trigger for real; the happy path's value is proving the real command works end-to-end.
