# Why must a structural path allow-list filter explicitly reje

## Decision

A CI gate that allow-lists changed file paths structurally (e.g. every file must match personas/community/*.yaml) can be bypassed by renaming a disallowed file (prompt.md) to an allowed one (prompt.yaml) — the diff shows only the new path. Reject any file with diff status "renamed" (or check previous_filename for a disallowed extension) rather than trusting the post-rename path alone.
