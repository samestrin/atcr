# How do you reliably detect all bare git exec.Command call si

## Decision

A string-literal grep for exec.Command("git",...) misses variable-indirected invocations (e.g. gitPath, _ := exec.LookPath("git"); exec.Command(gitPath, ...)). Sprint 32.4 hit this exact gap in internal/tools/snapshot.go, escaping both planning grep and the first AC4 regression test draft. Fix: implement the "no bare git exec outside gitexec" regression test as an AST classifier (go/ast) flagging any non-literal os/exec command-name argument as an offender, plus resolving import aliases, rather than a plain string grep.
