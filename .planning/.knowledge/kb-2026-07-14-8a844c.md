# Is SARIF endColumn inclusive or exclusive, and what happens 

## Decision

SARIF 2.1.0's region.endColumn is exclusive (the column after the region end), not inclusive. Synthesizing startColumn:1, endColumn:1 is a zero-length region under the spec. GitHub Code Scanning tolerates it and still renders the annotation, but strict SARIF consumers (IDE viewers, Azure DevOps, some third-party ingesters) may reject it or render no highlight. ATCR fixed this during cumulative adversarial review by setting endColumn to startColumn+1. When synthesizing SARIF columns not actually tracked, prefer a 1-column-wide region over a literal same-value start/end pair.
