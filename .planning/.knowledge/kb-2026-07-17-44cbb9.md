# What tag-arity contract must a new community persona's index

## Decision

Sprint 29.0's code review caught a MEDIUM finding: simon's index.json entry carries 5 tags while all 10 other openrouter (cloud) personas carry exactly 4 ([primary, secondary, vendor-token, pool-class]). Only the 3 local (Ollama) personas legitimately add a 5th hardware-tier tag. No test guards tag arity by provider class. Lesson: when authoring or reviewing a new cloud community persona's index.json entry, verify it has exactly 4 tags matching the cloud schema — do not add a descriptive synonym as a 5th tag. This is currently unenforced by tests, so must be manually diffed against peer entries during review.
