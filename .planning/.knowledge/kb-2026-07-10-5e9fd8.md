# How to close a TOCTOU gap in a gh-based auto-merge CI workfl

## Decision

Pin the triggering head SHA once at the start of the job, evaluate every subsequent check (changed files, check-run status) against that pinned SHA, and merge with `gh pr merge --match-head-commit "$SHA"` so the merge itself aborts if HEAD moved between evaluation and merge. Found via a fresh-subagent adversarial review of .github/workflows/hermes-auto-merge.yml (a multi-step filter → check-status → merge pipeline is vulnerable if each step independently re-resolves HEAD).
