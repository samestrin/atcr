# lgtmaybe Security Architecture

## Decision

lgtmaybe is an open-source, provider-agnostic AI Pull Request code reviewer created by Matt Coles (GitHub: MattJColes). It implements prompt injection hardening through multiple defense layers:

**Core Hardening Measures:**
1. **Boundary enclosure with token alteration** - Wraps code diffs in special markers and instructs the LLM that enclosed content is unsafe. Alters markers (e.g., swapping underscores for hyphens) to prevent attackers from replicating closing delimiters to escape the restricted zone
2. **Task reiteration** - Repeats primary instructions after the code snippet to maintain safety constraints prominence
3. **Credential scrubbing** - Strips passwords, cloud access keys, and sensitive information before transmission to LLM
4. **No code execution** - Fetches PR diffs via GitHub API only; never checks out or executes the reviewed code

**Architecture:**
- Provider-agnostic: Works with OpenAI, Anthropic, Bedrock, Vertex, or local Ollama models
- Diff-only review: Analyzes only changed lines, limiting untrusted content volume
- Keyless cloud auth: No hardcoded credentials required
- Eval harness: Formal evaluation framework using intentionally flawed code samples to benchmark model resilience against prompt injection

**Pricing:**
- Free with local models (Ollama)
- Cloud API costs depend on provider billing

**Sources:**
- GitHub: github.com/MattJColes/lgtmaybe
- Blog: coles.codes/posts/building-lgtmaybe/ (2026-06-07)
- Primary documentation claims hardening measures; no independent security audit found

## Rationale

- The tool treats code diffs as untrusted input by design, implementing defense-in-depth against indirect prompt injection attacks via malicious PR descriptions or code comments
- Architectural approach (no code checkout, diff-only) reduces attack surface compared to tools that execute or clone code
- Token alteration technique specifically addresses delimiter break-out attacks where malicious content tries to escape safety boundaries
- Open source with specific technical documentation enables community verification
