# When does skill/CONVENTIONS.md get extracted from skill/SKIL

## Decision

Epic 20.0 deliberately deferred extracting shared skill boilerplate (binary-on-PATH check, git-worktree check, .atcr/ path-safety rules) into skill/CONVENTIONS.md until "a second public skill is actually added." Sprint 20.1 added skill/debt-resolve/SKILL.md as that second skill, so the extraction shipped as Story 4 here. Pattern: don't extract shared conventions speculatively with only one skill — wait for the second skill, move content verbatim, point both SKILL.md files at CONVENTIONS.md via a pointer line, verify byte-identical extraction, and add new embedded content to TestSkill_NoAbsoluteOrClaudePaths.
