# Why must a required check-run gate filter by app.slug, not j

## Decision

A merge gate checking only "does a check-run named X report success" can be satisfied by any actor able to create a same-named check-run via the API, even if the real CI workflow hasn't started or never runs. Filter check-runs by app.slug == 'github-actions' (or verify check_suite.id matches the actual CI run) in addition to name matching. Found and fixed as a code-review TD item on .github/workflows/hermes-auto-merge.yml — "check-run name collision allows gate bypass if real CI hasn't started".
