# Why must a fetched community persona's {{ }} guardrail use a

## Decision

A fetched community persona's prompt is untrusted input (C3), so the install-time and resolve-time guardrails initially rejected ANY {{ / }} in the prompt body. But every persona template legitimately needs {{.AgentName}}, {{.ScopeRule}}, {{if .ToolsEnabled}}, etc. — reject-all broke the exact feature it protected: every community persona failed to install or resolve (TD-010), masked in CI because tests exercised the embedded bypass path. Fix: registry.ValidateFetchedPersonaPrompt, an exact-match allowlist of known required template actions, rejecting anything else. General pattern: enumerate what's required and allow exactly that; verify such guardrails via a real end-to-end fixture, not just synthetic unit tests, since mock paths can silently bypass the real guard.
