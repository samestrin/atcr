# How should language-aware skeptic routing be integrated into

## Decision

Use a two-partition reorder on the already-collected eligible names slice — language-matching skeptic names first, non-matching after — so the existing `n`-cap naturally favors language-matched skeptics. "general-purpose" is conceptual (= a skeptic with no `Language` declared), not a literal field. Tie-break = highest corroboration score then alphabetical. Within the matched partition, sort by `scores[name]` descending then name ascending; ties and absent keys (zero value) fall through to alphabetical, preserving determinism. Do NOT add a new function overload for the scoreless path — the nil map is a valid "no score data" signal.
