# wasip1-only guest ABI packages: substitute compile-time verification for unit tests

`internal/astgroup/parsers/src/guestabi` (and the three parser `main.go` files that import it) are `//go:build wasip1`-tagged and live in isolated nested Go modules excluded from the parent module's `go build ./...`/`go test ./...`. Under host GOOS these packages have zero buildable files, so a conventional `_test.go` unit-test harness cannot exercise them directly — there is no host-side entry point to call `guestabi.Alloc`/`Emit`/`Lookup` from.

Sprint 22.2 substituted two checks in place of unit tests, run per task rather than once at the end:
1. `GOOS=wasip1 GOARCH=wasm go build ./...` and `go vet ./...` run from inside each affected module — catches compile errors and vet issues the wasip1 tag would otherwise hide from a default `go build ./...`.
2. The existing host-side `internal/astgroup` regression suite (`go test ./internal/astgroup/...`), which loads the compiled `.wasm` binaries via wazero and exercises the guest ABI indirectly through real parse calls — this is the correctness oracle, not a mock.

This combination (isolated per-task compile+vet gate, then a full host-side regression run as the closing correctness check) is the reusable pattern for any future change to wasip1-only guest code in this repo, since a "test-first" RED/GREEN cycle isn't reachable for code the host toolchain can't compile.
