# What is the configuration structure for ATCR's --auto-fix se

## Decision

ATCR's --auto-fix configuration is defined in .atcr/config.yaml under a top-level auto_fix: block, which maps to registry.ProjectConfig.AutoFix (an *AutoFixConfig struct). It supports three keys: (1) apply_target: the directory path where patches are applied, defaulting to the repository root `.`. (2) validate_command: the custom validation command arguments, defaulting to go build ./... or similar verify defaults. (3) validate_timeout: the duration string (e.g., 2m) after which the validation command times out, defaulting to 2 minutes.
