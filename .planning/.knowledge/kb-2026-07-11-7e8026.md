# How do you add a new dimension to a fan-out cache key withou

## Decision

Fold the new dimension in as a NUL-separated token with a neutral-value sentinel, so old keys collapse unchanged. Sprint 19.10's F7 folded a sizingToken ("%d:%d" of effectiveByteBudget:maxLines) into diffCacheKey's tuning token via NUL-separator, but only appended it when sizing != "" && sizing != "0:0" — the sentinel meaning "no per-agent sizing applied". Every pre-existing cache entry and exact-key regression test stayed byte-identical. Generalizable rule: when adding a dimension to a composite cache/identity key, (1) reserve a genuinely-unreachable no-op sentinel value, (2) skip appending when the value equals that sentinel, (3) use an unambiguous separator (NUL byte) between components so the new dimension takes effect only where it matters.
