# Why is llm_support_group_td unusable for a 'From Review:' TD

## Decision

Sprint 33.0's Task 07 tried to shard 23 findings into .planning/technical-debt/README.md via the llm_support_group_td binary and it emitted an outdated 8-column "### [date] From Sprint:" shape with every cell dropped — it does not correctly handle a "### [date] From Review:" section (the shape /execute-code-review-produced TD captures use, as opposed to /execute-sprint-produced ones). The section had to be hand-formatted byte-consistent with the adjacent 32.4 sprint's 11-column section instead. Any skill sharding TD rows for a code-review (not sprint) source should verify group_td's output shape before trusting it, or hand-format to match an existing dated section.
