# How should persona language scope be stored and matched in A

## Decision

Add a new `Language []string` field to `AgentConfig` (`internal/registry/config.go`). Do NOT reuse the existing `Scope []string` field — `Scope` means "soft prompt-injection focus categories" injected via `payload.ScopeFocus()`; repurposing it is a semantic collision. Canonical form: without leading dot, lowercased (e.g. `["go", "ts"]`). Accept either `go` or `.go` in YAML and canonicalize at load in `applyDefaults` — trim space, strip single leading dot, lowercase — mirroring the existing `MinSeverity`/`Role` load-time canonicalization. `validateAgent` rejects empty entries and control characters but does NOT enforce a known-language allow-list (forgiving for third-party persona authors). Matching strips dot + lowercases `filepath.Ext(finding.File)` via `normalizeExt(ext string) string` helper so both sides compare in canonical form.
