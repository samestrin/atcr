# How does ATCR orchestrate --auto-fix validation, reverts, an

## Decision

In ATCR, the --auto-fix flow runs at the CLI command level (in cmd/atcr/autofix.go via runAutoFix). It follows a strict sequence: (1) Apply the patch to the working directory using autofix.ApplyPatch. (2) Run the configured local validation command (e.g. go build ./...) using verify.RunConfiguredValidation. (3) If validation fails, immediately revert changes using autofix.RevertPatch and exit with a non-zero code without making any remote GitHub calls. (4) If validation passes, clean up backup files with autofix.CleanupBackups, then orchestrate the remote Git operations via the GitHub API (create branch, commit changes, and open/update the PR).
