# Why must --force never overwrite existing persona files duri

## Decision

AC 01-05's Related Files line said persona files are "never overwritten unless --force," while the same AC's own DoD and the parent user story's SMART criteria said personas are preserved unconditionally, force or not. Phase 3 adjudicated by authority rank: user-story SMART criteria + the AC's own DoD (higher authority) beat a Related-Files prose note (lower authority, easily stale). Resolution: --force keeps only its original job of bypassing the top-level "config already exists" gate, and never triggers persona-file removal; runInit's unconditional os.Remove on force=true became persona-aware. Lesson: when planning artifacts disagree, resolve by document authority (user story/DoD > AC prose > incidental Related-Files notes), not by reading order or implementation convenience. Record the adjudication explicitly as a Clarification so later phases don't re-litigate it.
