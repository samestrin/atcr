# Which payload mode is most compact for frontier models vs wh

## Decision

`diff` mode (unified diff) is the most compact and token-friendly payload mode — right for frontier models and large ranges. `blocks` mode (changed hunks expanded to enclosing function/block with real line numbers via git --function-context) is the default and produces better quality from small/low-active-parameter MoE models. `files` mode (full head-version content with changed regions marked) is highest token cost, suited for small ranges or audit-style review. Documentation must explicitly state the diff-vs-blocks token guidance so users can choose appropriately.
