# gh fork+PR automation: does a shallow-cloned fork need to be

## Decision

Yes. Before branching off a reused fork for an automated PR, sync the fork's default branch with upstream (e.g. `gh repo sync <owner>/<repo> --source <upstream>`) or fetch and branch off upstream's default SHA directly. A "fork already exists" non-fatal check is not the same guarantee as "fork is in sync" — a stale fork default produces a PR whose diff includes unrelated upstream commits, effectively unmergeable, even though fork/push/PR-create all report success.
