# How does ATCR's --auto-fix handle the GitHub API PR create-v

## Decision

Under the --auto-fix flow, ATCR checks for an existing open PR from the same head branch using ghaction.Client.FindOpenPullRequest. If an open PR is found, ATCR updates it via UpdatePullRequest (using a PATCH request). Otherwise, a new PR is created via CreatePullRequest. The live CLI adapter by default mints a unique branch suffix atcr/auto-fix/<UTC-timestamp> per run, resulting in a new PR for each run, while the update path is structurally verified in unit tests and integration tests via mock entries.
