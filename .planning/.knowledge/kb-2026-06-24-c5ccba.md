# How should a persona fixture runner be structured when the f

## Decision

Ship the CLI surface, exit-code-mirroring contract, and an injectable `FixtureRunner` interface as the seam. Set a `noFixtureRunner` stub as the default that reports `HasFixture: false`. The seam allows wiring the real LLM-backed runner additively in a follow-up with no API change — just set `personasFixtureRunner` default. Exercise the seam via a stub in tests so the contract is locked. Deferred runner is tracked as technical debt (TD-012 in sprint 9.0). This pattern avoids delivering a non-functional default while keeping the extension point clean.
