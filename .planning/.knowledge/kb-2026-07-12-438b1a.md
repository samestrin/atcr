# Why does the local TD store (internal/localdebt) treat resol

## Decision

internal/localdebt is append-only, dedup'd by history.FindingID (excludes severity). selectOpenDebt folds an id closed if any record ever carried a terminal status ("resolved wins forever"), and persistLocalDebt seeds its append-time dedup seen set from ReadAll (which includes terminal records), so a re-detected identical finding is skipped on append too. Flagged twice (TD-001, TD-004) and explicitly ratified as accepted-by-design rather than fixed in-sprint — a real re-open path needs both the append-side dedup and the read-side fold changed together, deferred to a follow-on epic.
