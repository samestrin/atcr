# Does GitHub Code Scanning require a SARIF region block for f

## Decision

The base SARIF 2.1.0 spec allows omitting `region` entirely for a file-level (no-line) result. GitHub Code Scanning's ingestion does not honor that — a result without startLine/startColumn/endLine/endColumn fails to display in the Security tab. ATCR's sarifLocation (internal/report/sarif.go) synthesizes a {1,1,1,1} fallback region for any Line<=0 finding specifically to satisfy this GitHub-specific display requirement. Confirmed live via a manual smoke test.
