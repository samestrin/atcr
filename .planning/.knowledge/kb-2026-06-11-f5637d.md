# How does leaf-preference source discovery prevent double-cou

## Decision

A directory's `findings.txt` is a reconcile input only when no subdirectory beneath it also contains a `findings.txt` (leaf-preference). This resolves the conflict between nested structures like `pool/raw/agent/*/findings.txt` (per-agent raw files) and `pool/findings.txt` (merged pool output). Per-agent raw files become the pool inputs; the merged `pool/findings.txt` is written for downstream convenience but is NOT re-discovered (no double-counting). `host/findings.txt` is read directly. `reconciled/` is never an input source. Confidence counts distinct REVIEWER values across all discovered rows.
