# Should transport-level errors (connection reset, EOF, DNS) r

## Decision

Yes. Transport-level errors (connection resets, EOF, DNS failures) retry on the same backoff schedule as HTTP 429/5xx responses. The original contract specified "429/500/502/503/504 only" for HTTP responses, but transport errors are not HTTP responses and fall outside that enumeration. This was adopted during /resolve-td as a test-driven robustness fix. Context cancellation and deadline-exceeded still return immediately — only transport-level failures trigger retry. The LLM Client Contract, documentation/llm-client-fanout.md, and AC 01-04's spec-alignment note were all updated to reflect this.
