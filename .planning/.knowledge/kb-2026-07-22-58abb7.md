# Does atcr's full-tree dogfood review need --byte-budget 0?

## Decision

The default 512 KB payload budget (internal/payload/budget.go:46) drops ~90% of a multi-MB review tree before any reviewer persona sees it. Sprint 33.0 needed a single full-tree `atcr review --base <empty-tree-sha> --head HEAD --byte-budget 0` invocation to dogfood the tool against its own ~6.1 MB cmd/+internal/+reconcile/+skill/ tree. Scoped to that one invocation only (not a .atcr/config.yaml change), keeping a single review id so downstream triage still gets one reconciled artifact instead of fragmenting across multiple review ids.
