# Why does atcr's persona ID hashing use plain SHA-256 instead

## Decision

HashPersonaID (internal/scorecard/telemetry.go:18) is a bare, unsalted hex SHA-256 of the persona name — dictionary/rainbow-table reversible against the small, often-public set of persona names (TD-007, HIGH). Kept as plain SHA-256 rather than escalated to HMAC+pepper because cross-install leaderboard aggregation requires every independent CLI install to hash the same persona name to the same digest; an HMAC pepper strong enough to matter would have to be a shared secret baked into the distributed binary, trivially extractable by disassembly — marginal protection at the cost of breaking the plaintext-equivalence aggregation depends on. Correct fix (deferred, backend-owned): a server-side keyed hash or a properly distributed/rotatable secret managed outside the binary.
