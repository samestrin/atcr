# Why must internal/personas.PersonasDir() match the resolver'

## Decision

PersonasDir() (where install/init/quickstart write fetched personas) and ResolvePersona's Registry dir (where review-time resolution looks for pinned community personas) were two independently-defined paths. On darwin they diverged: PersonasDir() used os.UserConfigDir()/atcr/personas (~/Library/Application Support/...) while the resolver expected filepath.Dir(DefaultRegistryPath())/personas (~/.config/atcr/personas). A persona could install successfully and never resolve at review time because writer and reader disagreed on the directory. Lesson: when a feature has separate write and read paths for the same directory concept, define the directory once and have both sides reference it — don't let platform-specific stdlib helpers silently diverge from a resolver's own convention. Each side's tests pass in isolation; only an end-to-end install→resolve test catches the seam.
