# Why must skeptic selection enforce the different-model rule 

## Decision

Skeptic selection must enforce the different-model rule at the engine level, never left to configuration discipline alone. A skeptic sharing a model with any crediting reviewer is never selected; if no eligible skeptic exists, the finding is unverifiable with reason no_eligible_skeptic. Why: Agreement between models is correlated, not independent, evidence. A skeptic using the same model as the finder shares blind spots — plausible-but-wrong findings that several models produce for the same reason cannot be caught by a same-model skeptic. Additional considerations: deep-copy reference fields (Scope []string, *Temperature, etc.) when returning AgentConfig to prevent callers from corrupting the shared registry; flag skeptics with undefined providers at selection time (not invocation time) so misconfiguration is attributed early. Context: Epic 3.0 adversarial verification. The different-model rule is necessary-not-sufficient; Epic 4.0 debate adds pressure.
