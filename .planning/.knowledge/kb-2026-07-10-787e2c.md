# how should a docs-accuracy adversarial review handle a pre-e

## Decision

Reclassify it as out-of-scope-for-this-phase with a citation to the recorded user decision, rather than fixing it inline just because the severity rubric says fix CRITICAL/HIGH. A fresh-subagent adversarial reviewer has no visibility into earlier scope negotiations in the same sprint (it runs with deliberately no implementation memory), so a correct-on-its-face finding can still be out of scope if the user already explicitly deferred it (with its own tracked tech-debt item). Re-running the same context-free reviewer only reproduces the same out-of-context finding — document the reclassification rationale instead of looping the review.
