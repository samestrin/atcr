## TD-002 — `go install .../atcr@latest` blocked by replace directive; Story 3 paused (HIGH)
**Origin:** Phase 2, task 2.4 pre-flight (blocker; Story 3 paused), 2026-07-11
**File:** go.mod:41
**Issue:** `go install github.com/samestrin/atcr/cmd/atcr@latest` fails for every external user with "The go.mod file for the module providing named packages contains one or more replace directives." Root cause is `replace github.com/samestrin/atcr/reconcile => ./reconcile` (the in-tree nested module from Epic 8.0). Go refuses to `go install <module>@<version>` any module whose published go.mod carries a replace directive, so the remote install form is broken at every version for everyone. Local `go build ./...` / `go install ./cmd/atcr` from a checkout still work, so CI and the pre-commit hook stay green and the failure is invisible until a real remote install is attempted. This makes AC1 (install via `go install`) false today, makes install.sh (AC4) a wrapper over a currently-unusable command, and makes the `go install ...@latest` lines in README.md + docs/skill-usage.md inaccurate.
**Mitigation this sprint:** Story 3 (install.sh + install-script test) paused before any code was written — no knowingly-broken script or permanently-red integration test committed. Story 2 (backend-contract test) is complete and unaffected. Verified independently by the operator (reproduced the identical error in a scratch GOPATH).
**Fix in:** Epic 8.0 / Epic 21.0 (release & packaging) — publish `github.com/samestrin/atcr/reconcile` as a real fetchable module and drop go.mod:41's replace directive (the `reconcile/vX.Y.Z` release gate at `.github/workflows/reconcile-module.yml` already exists but has never been tagged). Once `go install ...@latest` resolves, re-sequence Story 3 (install.sh + real-install integration test per the Phase 2 clarification) and Story 4's doc-accuracy pass. Publishing a fetchable module is explicitly Epic 21.0 scope per original-requirements.md Out of Scope, not this sprint's install.sh concern.
