# Completed Sprints

Archive of completed sprint folders and their outcomes.

## Structure

- Each completed sprint maintains its original folder structure:
  - `{SPRINT_NUMBER}_{feature-name}/README.md` - Final sprint plan with completion status
  - `research/` - All research files and documentation
  - `notes/` - Development notes and observations
  - `artifacts/` - Final code artifacts and prototypes
- Add completion notes and lessons learned to the main README.md
- Reference actual implementation files in the project

## Review Process

- Document what went well in the sprint's README.md
- Note deviations from plan and reasons
- Capture lessons for future sprints
- Preserve all research and artifacts for future reference

## Example Structure
```
completed/
├── 6.0_dashboard_redesign/
│   ├── README.md (with completion status)
│   ├── research/
│   ├── notes/
│   └── artifacts/
└── 5.0_user_authentication/
    ├── README.md
    └── ...
```
